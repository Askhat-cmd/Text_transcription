PRD-034.1 — StateAnalyzer JSON hardening + Admin runtime badge fix

Контекст:
После PRD-034 бот снова работает, Writer успешно отвечает через выбранную модель gpt-5, hot-swap моделей из админки работает. Но state_analyzer иногда даёт JSONDecodeError, а в админке сверху может отображаться legacy_adaptive, хотя фактически trace показывает Pipeline NEO | multiagent_v1.

Задачи:

1. Усилить StateAnalyzer._parse_json:
   - убирать markdown fences;
   - извлекать JSON между первой "{" и последней "}";
   - логировать raw_response preview при JSONDecodeError;
   - сохранять raw_response и parse_error в last_debug;
   - не терять state_analyzer_error в orchestrator debug.

2. Усилить JSON-инструкцию в agent_llm_client для require_json=True:
   Return ONLY valid JSON.
   Use double quotes for all keys and string values.
   No markdown.
   No comments.
   No trailing commas.

3. Добавить тесты:
   - pure JSON парсится;
   - ```json ... ``` парсится;
   - текст + JSON + текст парсится;
   - невалидный JSON сохраняет parse_error в last_debug.

4. Починить runtime badge в Admin:
   - если active pipeline реально multiagent_v1, не показывать legacy_adaptive;
   - корректно понимать env значения: "on", "true", "1", "yes";
   - добавить в status active_runtime: "multiagent".

Acceptance:
- State Analyzer не падает шумно на markdown-wrapped JSON.
- JSONDecodeError не ломает ответ пользователю.
- В trace видно raw_response preview/parse_error при проблеме.
- В админке runtime badge соответствует фактическому runtime.
- Writer gpt-5 продолжает работать.
- tests/multiagent проходят.
- tests/admin проходят.
