# PRD-045.1.1 — Доставка quality_trace до API, debug endpoints и baseline report

**Проект:** Bot Psychologist / Neo MindBot  
**Репозиторий:** `Askhat-cmd/Text_transcription`  
**Рабочая зона:** `bot_psychologist`  
**Приоритет:** высокий  
**Риск:** низкий  
**Тип:** bugfix / observability / trace propagation  
**Влияние на ответы бота:** не должно менять финальный текст ответа  
**Предыдущий этап:** PRD-045.1 — Расширение trace quality fields  
**Следующий этап после успешной проверки:** PRD-045.2 — Writer Prompt v2

---

## 0. Краткое резюме

PRD-045.1 добавил deterministic модуль `quality_trace.py` и интегрировал `quality_trace` в `MultiAgentOrchestrator.debug`.

Однако live baseline после PRD-045.1 показал:

```text
Quality trace: null
```

для всех live-кейсов.

Это означает, что `quality_trace` создаётся внутри orchestrator, но не доходит до внешнего API response `/api/v1/questions/adaptive` и/или совместимой модели `DebugTrace`.

Этот PRD должен исправить именно доставку trace-полей через API/debug слой.

Важно: не менять Writer prompt, не менять логику генерации ответа, не улучшать качество ответов на этом шаге. Только сделать так, чтобы уже созданный `quality_trace` был видим в API, baseline runner и debug endpoints.

---

## 1. Контекст проблемы

После PRD-045.1:

- `bot_agent/multiagent/quality_trace.py` создан.
- `orchestrator.py` импортирует `QUALITY_TRACE_VERSION` и `build_quality_trace`.
- `orchestrator.debug` содержит:
  - `quality_trace_version`
  - `quality_trace`
  - `quality_trace_error`
- Unit tests и orchestrator e2e tests проходят.

Но live baseline report показывает:

```markdown
- Quality trace: null
```

Это говорит о разрыве между внутренним `orchestrator.debug` и внешним API/trace response.

Вероятная причина:

1. `api/routes/chat.py` вручную собирает `trace_payload`.
2. В whitelist/extensions списка `chat.py` нет `quality_trace_version`, `quality_trace`, `quality_trace_error`.
3. `api/models.py::DebugTrace` не содержит этих полей.
4. Pydantic-модель может отбрасывать extra fields.
5. Baseline runner читает `trace.quality_trace`, но поле уже потеряно на стороне API.

---

## 2. Цель PRD

Сделать так, чтобы `quality_trace` был видим во всех нужных местах:

1. В raw multiagent debug storage.
2. В `/api/v1/questions/adaptive` при `debug=true`.
3. В `response.trace`.
4. В `/api/debug/session/{session_id}/traces`.
5. В `/api/debug/session/{session_id}/multiagent-trace`, если этот endpoint используется Web Trace.
6. В `run_quality_baseline.py` live report.

```bash

```

в markdown report должно быть не:


а примерно:

```markdown

```

или:

```markdown

```

В JSON report у каждого live turn должно быть:

```json
{
  "debug": {
    "quality_trace_version": "quality_trace_v1",
    "quality_trace": {
      "version": "quality_trace_v1",
      "answer": {},
      "state": {},
      "thread": {},
      "memory": {},
      "continuity": {},
      "response_mode": {},
      "validator": {},
      "summary_flags": []
    },
    "quality_trace_error": null
  }
}
```

Если `quality_trace` по какой-то причине не может быть построен, должно быть:

```json
{
  "quality_trace_version": "quality_trace_v1",
  "quality_trace": {
    "version": "quality_trace_v1",
    "error": "quality_trace_failed"
  },
  "quality_trace_error": "quality_trace_failed:<ExceptionClass>"
}
```

Но API не должен падать.

### Task 3 — Добавить quality_trace поля в API model DebugTrace

Файл:

```text
bot_psychologist/api/models.py
```

В `DebugTrace` добавить:

```python
quality_trace_version: Optional[str] = None
quality_trace: Optional[Dict[str, Any]] = None
quality_trace_error: Optional[str] = None
```

Убедиться, что `Dict` и `Any` уже импортированы. Если импорт есть — не менять лишнего.

---

### Task 4 — Пробросить поля в chat.py trace_payload

Файл:

```text
bot_psychologist/api/routes/chat.py
```

В месте, где расширения из `raw_dict` переносятся в `trace_payload`, добавить ключи:

```python
"quality_trace_version",
"quality_trace",
"quality_trace_error",
```

Или добавить явный блок:

```python
for key in ("quality_trace_version", "quality_trace", "quality_trace_error"):
    if key in raw_dict and raw_dict.get(key) is not None:
        trace_payload[key] = raw_dict.get(key)
```

Важно:

- `quality_trace` должен оставаться dict.
- Не сериализовать его в строку.
- Не добавлять raw prompts/user text.
- Не трогать existing trace fields.

### Task 6 — Добавить quality_trace в multiagent-trace endpoint

Файлы:

```text
bot_psychologist/api/models.py
bot_psychologist/api/debug_routes.py
```

Сейчас `/api/debug/session/{session_id}/multiagent-trace` возвращает `MultiAgentTraceResponse`.

Добавить в модель `MultiAgentTraceResponse` опциональные поля:

```python
quality_trace_version: Optional[str] = None
quality_trace: Optional[Dict[str, Any]] = None
quality_trace_error: Optional[str] = None
```

В `debug_routes.py::get_multiagent_trace(...)` передать:

```python
quality_trace_version=debug.get("quality_trace_version"),
quality_trace=debug.get("quality_trace") if isinstance(debug.get("quality_trace"), dict) else None,
quality_trace_error=debug.get("quality_trace_error"),
```

Важно: это не должно ломать существующий UI. Поля опциональные.
