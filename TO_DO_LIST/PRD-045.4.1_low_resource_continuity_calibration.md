**Проект:** Bot Psychologist / Neo MindBot  
**Репозиторий:** `Askhat-cmd/Text_transcription`

**Риск:** средний  
**Тип:** targeted runtime calibration / continuity fix  
**Влияние на ответы бота:** да, ожидается улучшение continuity для low-resource multi-turn диалогов  
**Предыдущий этап:** PRD-045.4 — Thread Manager Diagnostics v1  
**Следующий этап после аудита:** PRD-045.5 Pattern Core / Active Frame preparation  
**Главный принцип:** простота, модульность, гибкость, быстрые настройки, понятный trace-аудит.

## 0. Краткое резюме

PRD-045.4 успешно добавил `thread_diagnostics_v1`.

Direct smoke сразу показал ценную диагностику:

```text
QB-002:
response_mode=validate
mode_reason=explicit_contact_validate

QB-005:
response_mode=practice
mode_reason=solution_practice

QB-010:
relation_reason=continuity_below_threshold
phase_reason=new_thread_default_clarify
mode_reason=phase_clarify_reflect
thread_action=new_thread
```

Это означает:

- `QB-002` и `QB-005` уже исправлены в свежем direct runtime;
- `QB-010` всё ещё ломает линию разговора;
- причина не скрыта: Thread Manager создаёт новый thread из-за низкого token overlap;
- после new thread система уходит в `clarify/reflect`, хотя пользователь находится в low-resource продолжении и просит “что-то совсем простое”.

Этот PRD делает маленькую точечную правку:

1. Добавляет low-resource continuation hints в State Analyzer.
2. Добавляет low-resource continuation guard в Thread Manager relation logic.
3. Расширяет diagnostics reason codes.
4. Добавляет regression tests на `QB-010`-подобный multi-turn сценарий.
5. Проверяет, что `QB-002` и `QB-005` не регрессируют.

Это не Pattern Core и не Response Planner. Это минимальный continuity fix до большого смыслового слоя.

---

## 1. Почему это нужно

В психологическом диалоге пользователь часто продолжает ту же линию не теми же словами.

Пример:

```text
1. Сил почти нет.
2. Даже отвечать тяжело.
3. Можно что-то совсем простое на сейчас?
```

Token overlap между третьим сообщением и `core_direction` может быть низким, но по смыслу это очевидное продолжение low-resource состояния.

Текущая логика Thread Manager слишком легко создаёт `new_thread`, потому что опирается на token overlap. Diagnostics теперь это показывает явно.

Нам нужен маленький guard:

```text
Если есть current_thread и сообщение выглядит как low-resource продолжение,
не создавать new_thread только из-за низкого overlap.
```

### 7.3 Intent precedence

Не менять порядок радикально, но убедиться, что:

```text

```

Если пользователь просит “что-то совсем простое” в состоянии low-resource, это не должно превращаться в `explore`.

Допустимые варианты для финального turn:

- `intent=contact` + `nervous_state=hypo` → `validate`;
- `intent=solution` + `nervous_state=hypo` → по Thread Manager должно стать `regulate`, потому nervous_state выше solution;
- `intent=contact` предпочтительнее для “не дави / просто рядом / совсем простое”.

### 8.1 Relation rule

В `_resolve_relation_with_debug(...)` добавить правило до `continuity_below_threshold`, но после explicit return marker и после obvious branch marker, если branch marker должен оставаться сильнее.

Рекомендуемая логика:

```python
low_resource_continue_marker_hit = any(marker in lowered for marker in _LOW_RESOURCE_CONTINUE_MARKERS)

if archived_threads and return_marker_hit:
    ...
elif followup_continue_marker_hit:
    ...
elif low_resource_continue_marker_hit and current_thread is not None and not branch_marker_hit:
    relation = "continue"
    continuity = max(continuity_raw, 0.25)
    relation_reason = "low_resource_continuation_marker"
elif continuity_raw < _NEW_THREAD_THRESHOLD:
    ...
elif branch_marker_hit:
    ...
```

Если решишь, что branch marker должен идти раньше low-resource guard, зафиксируй это в implementation report.

### 8.2 Diagnostics fields

В relation diagnostics добавить:

```python
"low_resource_continue_marker_hit": bool
```

В possible `relation_reason` добавить:

```text
low_resource_continuation_marker
```

В summary flags добавить:

```text
low_resource_followup_continued
```

Если marker поднял continuity выше threshold:

```text
low_resource_marker_overrode_low_overlap
```

Пример:

```json
{
  "relation_reason": "low_resource_continuation_marker",
  "continuity_raw": 0.0,
  "continuity_score": 0.25,
  "low_resource_continue_marker_hit": true
}
```
