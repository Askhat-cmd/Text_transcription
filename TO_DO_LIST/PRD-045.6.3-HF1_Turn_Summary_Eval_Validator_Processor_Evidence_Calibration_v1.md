---

## 1. Контекст

`PRD-045.6.3 — Async Turn LLM Summary Store v1` в целом реализовал важный слой:

```text
TurnLLMSummary contract
→ pending record on memory write
→ mock/openai summary service
→ processor CLI
→ Context Assembly selection:
   ready+valid+hash-match → llm_abstractive_v1
   pending/failed/hash mismatch → deterministic fallback
→ trace reasons
```

Но audit показал несколько acceptance gaps:

1. Eval fixture содержит только `3` кейса, хотя PRD требовал минимум `5`.
2. `summary_processor_smoke.json` прошёл по пользователям, но фактически обработал `0` pending items, поэтому нет сильного evidence pending → ready.
3. Validator в отчёте описан как базовый readiness/hash validator, но acceptance требовал также conservative guard против diagnosis language, direct advice/action commands, raw full turn dump, overly long quote/summary.
4. Перед переходом к retrieval eval важно убедиться, что async summary слой не только “существует”, но и имеет проверяемые quality gates.

Этот PRD не добавляет новую архитектуру, а закрывает acceptance долги.

### Workstream C — Strengthen summary validator

Обновить:

```text
bot_psychologist/bot_agent/multiagent/contracts/turn_llm_summary.py
```

или добавить отдельный helper module, если так чище.

Validator `validate_turn_llm_summary(...)` должен проверять не только ready/hash/version, но и content safety.

Минимальные checks:

#### C1. Diagnosis language guard

Reject summary if it contains unsafe diagnosis assertions, например:

```text
"у пользователя депрессия"
"это ПТСР"
"у него травма"
"у неё расстройство"
"это клиническая"
```

Важно: guard должен быть conservative, но не ломать нейтральные формулировки вроде:

```text
"пользователь описывает тревогу"
"звучит истощённо"
"просит поддержки без анализа"
```

#### C2. Direct advice/action guard

Reject summary-model voice if summary превращается в совет:

```text
"тебе нужно..."
"ты должен..."
"обязательно сделай..."
"надо срочно..."
```

Нейтральное описание действий ассистента допустимо:

```text
"ассистент предложил сделать один маленький шаг"
"бот уже дал дыхательную практику"
```

#### C3. Raw dump / overlong quote guard

Reject if:

```text
summary length > TURN_LLM_SUMMARY_MAX_SUMMARY_CHARS
important_quote length > 160
summary contains huge repeated user text
summary looks like raw user+assistant transcript
```

Heuristic examples:

```text
"USER:"
"ASSISTANT:"
"Пользователь: ... Бот: ..." если это длинный transcript-style dump
```

#### C4. Empty semantic fields are okay

Do not require every optional field to be filled. `open_loop`, `important_quote`, `assistant_move` may be empty.

## 6. Acceptance criteria

HF1 считается выполненным, если:

1. Eval fixture has `cases_total >= 5`.
2. Required 5 semantic case types are present.
3. Validator rejects diagnosis assertion summaries.
4. Validator rejects direct-advice summary voice.
5. Validator rejects raw transcript-style dump / overlong quote.
6. Ready valid summary still passes.
7. Pending summary still falls back to deterministic.
8. Failed summary still falls back to deterministic.
9. Hash mismatch still falls back to deterministic.
10. Unsafe invalid summary falls back to deterministic.
11. Processor smoke proves actual pending → ready with `processed_count >= 1`.
12. No raw full turn dumps in reports/logs.
13. Existing context assembly tests pass.
14. Multiagent tests pass.
15. Docs updated.
16. Commit and push to `main`.
17. Implementation report updated after final push with actual commit hash and push status.
