# PRD-045.7.1 — Writer Move Compliance / Diagnostic Card Alignment v1

## 0. Краткое резюме

**Цель PRD-045.7.1:** довести связку `DiagnosticCard v1 -> Writer` до практического качества.

PRD-045.7 успешно добавил `DiagnosticCard`, deterministic `build_diagnostic_card_v1(...)`, интеграцию в `WriterContract`, вывод diagnostic card в Writer prompt, debug и baseline. Но аудит показал новый слабый шов:

```text
Diagnostic Card уже присутствует,
но Writer не всегда достаточно жёстко исполняет suggested_writer_move / avoid_list / risk_flags.
```

Baseline после PRD-045.7 показывает:

```text
Generic-risk flags: 3
QB-002: generic_phrase_risk
QB-010: generic_phrase_risk
QB-015: generic_phrase_risk
```

То есть карточка есть, но её управляющая сила пока недостаточна.

Главный принцип:

```text
Не менять DiagnosticCard contract радикально.
Не менять State Analyzer / Thread Manager / Context Assembly.
Не делать PRD-046.x, пока Writer не научился стабильно исполнять diagnostic card.
```

---

## 1. Почему нужно

PRD-045.7 сделал важный архитектурный шаг: Writer теперь получает компактную рабочую ориентацию.

Но в текущей реализации Writer получает в prompt только краткий summary:

```text
situation_label
current_need
suggested_writer_move
confidence
avoid_list
risk_flags
```

Этого оказалось достаточно для технической проводки, но недостаточно для устойчивого качества ответов.

### 1.1. Проблема на baseline

После PRD-045.7:

- `QB-002` получает `low_resource_contact / regulate_first`, но ответ всё ещё может звучать общо и задавать лишний контактный вопрос.
- `QB-010` получает `low_resource_contact / regulate_first`, но ответ может включать вопрос/обратную связь, хотя low-resource режим требует минимального давления.
- `QB-015` получает `semantic_continuation / reflect_pattern_once`, но Writer может уходить в несколько гипотез и общий психологический разбор вместо одного точного отражения паттерна.

### 1.2. Архитектурная причина

`DiagnosticCard` сейчас описывает ориентацию, но не превращается в достаточно конкретные output constraints.

Например:

```text
suggested_writer_move = regulate_first
```

нужно разворачивать для Writer в операциональные ограничения:

```text
- 2–4 коротких предложения
- одно телесное действие
- без анализа
- без вопроса, если пользователь просит поддержку / low-resource contact
```

А:

```text
suggested_writer_move = reflect_pattern_once
```

нужно разворачивать в:

```text
- назвать один повторяющийся паттерн
- не давать несколько гипотез
- не разгонять глубину
- максимум один вопрос
```

---

## 2. Цель

### 2.1. Основная цель

Добавить слой:

```text
Writer Move Compliance v1
```

который:

1. Превращает `DiagnosticCard.suggested_writer_move`, `situation_label`, `avoid_list`, `risk_flags` в конкретные prompt constraints.
2. Делает Writer prompt более управляемым без радикального переписывания.
3. Добавляет deterministic compliance checks в `quality_trace` или отдельный `writer_move_compliance_trace`.
4. Снижает generic-risk и over-exploration в smoke cases.
5. Не ломает текущую архитектуру PRD-045.7.

### 2.2. Цель по baseline

Direct smoke:

```text
QB-002
QB-005
QB-010
QB-015
```

после PRD-045.7.1 должен показать:

```text
diagnostic_card_present = true
writer_move_compliance_trace_present = true
generic-risk flags <= 1 total across four cases
QB-010 remains continue / clarify / validate
QB-015 remains continue / active_frame_semantic_continuity
```

Для `QB-010` желательно:

```text
no user-facing question
answer <= 4 short sentences
one simple regulation action
```

Для `QB-015` желательно:

```text
one repeated pattern reflected
no broad psychological lecture
at most one question
no internal labels
```

## Task 2 — Add Writer Move Instruction mapping

Добавить deterministic helper:

```python
build_writer_move_instructions_v1(diagnostic_card: DiagnosticCard | None) -> dict[str, Any]
```

Рекомендуемый файл:

```text
bot_psychologist/bot_agent/multiagent/writer_move_compliance.py
```

или локально рядом с Writer.

### 2.1. Output schema

```python
{
    "version": "writer_move_instructions_v1",
    "move": "regulate_first",
    "max_sentences": 4,
    "max_questions": 0,
    "style": "brief_low_pressure",
    "must_do": [
        "offer_one_simple_body_action",
        "keep_language_short"
    ],
    "must_not_do": [
        "do_not_analyze",
        "do_not_ask_followup_question",
        "do_not_offer_multiple_steps"
    ],
}
```

### 2.2. Mapping rules

#### `safe_override`

```text
max_sentences: 3
max_questions: 0
must_do:
- prioritize_safety_protocol
- use_short_direct_support
must_not_do:
- do_not_explore
- do_not_analyze
```

#### `regulate_first`

```text
max_sentences: 4
max_questions: 0
must_do:
- offer_one_simple_body_action
- keep_language_short
- reduce_pressure
must_not_do:
- do_not_analyze
- do_not_explain_pattern
- do_not_ask_followup_question
- do_not_offer_multiple_steps
```

#### `validate_briefly`

```text
max_sentences: 4
max_questions: 0 или 1 по context
must_do:
- validate_current_state
- stay_close_to_user_words
must_not_do:
- do_not_analyze_deeply
- do_not_give_advice
```

Если `risk_flags` содержит `low_resource`, тогда:

```text
max_questions = 0
```

#### `offer_one_micro_step`

```text
max_sentences: 5
max_questions: 0
must_do:
- offer_one_executable_micro_step
- include_time_or_start_condition
must_not_do:
- do_not_offer_list
- do_not_expand_theory
```

#### `reflect_pattern_once`

```text
max_sentences: 6
max_questions: 1
must_do:
- name_one_repeated_pattern_gently
- connect_to_recent_context
- return_authority_to_user
must_not_do:
- do_not_list_multiple_hypotheses
- do_not_rewrite_full_history
- do_not_over_explain
```

#### `clarify_one_point`

```text
max_sentences: 5
max_questions: 1
must_do:
- reflect_one_key_point
- ask_one_specific_question
must_not_do:
- do_not_open_multiple_threads
```

#### `explore_carefully`

```text
max_sentences: 6
max_questions: 1
must_do:
- offer_one_hypothesis_as_hypothesis
- avoid_strong_interpretation
must_not_do:
- do_not_turn_into_lecture
- do_not_add_second_hypothesis
```

## Task 3 — Integrate move instructions into WriterContract / prompt

Update `WriterContract.to_prompt_context()` to include:

```python
"writer_move_instructions": {...},
"writer_move_instruction_summary": "move=...; max_sentences=...; max_questions=..."
```

If diagnostic card is missing, use safe default:

```text
move=validate_briefly
max_sentences=5
max_questions=1
```

Update `WRITER_USER_TEMPLATE` with a compact block:

```text
WRITER MOVE INSTRUCTIONS:
{writer_move_instruction_summary}

MUST DO:
{writer_move_must_do}

MUST NOT DO:
{writer_move_must_not_do}
```

Do not radically rewrite the whole prompt.

### 3.1. Important prompt rule

Add explicit instruction:

```text
Если Writer Move Instructions конфликтуют с привычным response_mode, следуй более безопасному ограничению.
Например, low_resource + regulate_first запрещает вопрос, даже если обычный validate допускает контактный вопрос.
```

## Task 4 — Add compliance trace

Add deterministic trace after prompt-context construction or in `quality_trace.py`:

```text
writer_move_compliance_trace_v1
```

Recommended fields:

```json
{
  "version": "writer_move_compliance_trace_v1",
  "move": "regulate_first",
  "max_sentences": 4,
  "max_questions": 0,
  "sentence_count": 3,
  "question_count": 0,
  "violations": [],
  "risk_flags": ["low_resource"]
}
```

### 4.1. What to check

Use simple deterministic checks:

- sentence count approximate;
- question count;
- numbered list presence;
- if `move=regulate_first`, detect if answer contains more than one question or too many sentences;
- if `move=offer_one_micro_step`, detect numbered list / multiple step markers;
- if `move=reflect_pattern_once`, detect too many questions or too long answer.

### 4.2. No raw private logs

Compliance trace may inspect final answer locally, but debug summary/sanitized logs should avoid dumping long raw content.

## 6. Acceptance criteria

PRD считается выполненным, если:

1. Added deterministic `writer_move_instructions_v1`.
2. `WriterContract.to_prompt_context()` exposes move instructions.
3. Writer prompt includes move constraints without radical rewrite.
4. `regulate_first + low_resource` sets `max_questions=0`.
5. `offer_one_micro_step` discourages lists/multiple steps.
6. `reflect_pattern_once` discourages multiple hypotheses and over-explanation.
7. `writer_move_compliance_trace_v1` exists in debug or quality trace.
8. Baseline markdown prints writer move compliance summary.
9. Targeted tests pass.
10. Multiagent regression passes.
11. Direct smoke `QB-002/QB-005/QB-010/QB-015` runs with `errors_count=0`.
12. `QB-010` remains `continue / clarify / validate`.
13. `QB-015` remains `continue / active_frame_semantic_continuity`.
14. Total generic-risk flags across the four smoke cases is reduced compared to PRD-045.7 baseline, ideally `<=1`.
15. No raw private text is added to sanitized logs.
16. Task list, implementation report, compliance report and logs are created.
17. Commit and push to `main` are completed.
18. Implementation report updated after push with commit hash and push status.
