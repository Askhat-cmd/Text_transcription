# PRD-046.0.10-HF1 — BotDB SD Config / Encoding Finalization v1

## 0. Краткий смысл

`PRD-046.0.10` в целом выполнен успешно: legacy SD отключён через explicit-gate, BotDB docs обновлены, runtime smoke зелёный, production KB не мутировалась.

Но перед переходом к `PRD-046.1 Diagnostic Center` нужно закрыть два мелких, но важных хвоста:

1. В `Bot_data_base/config.yaml` всё ещё визуально указано `sd_labeling.enabled: true`, хотя фактический runtime выключает SD через `legacy_sd_labeling.enabled=false` и `explicit_legacy_mode=false`.
2. Runtime smoke artifacts содержат mojibake/encoding corruption в русских строках (`Ð`, `Ñ`, `\u0085`), что делает отчёты менее надёжными для human review и будущих quality gates.

Этот HF не должен менять базу знаний, Chroma, registry, Writer или Diagnostic Center.

## 3. Scope

### 3.1. SD config normalization

Нужно привести `Bot_data_base/config.yaml` к недвусмысленному виду:

```yaml
sd_labeling:
  enabled: false
  explicit_legacy_mode: false
  deprecated: true
  model: gpt-4o-mini
  temperature: 0.1
  max_tokens: 1500
  min_confidence: 0.5
  batch_size: 5
legacy_sd_labeling:
  enabled: false
  explicit_legacy_mode: false
```

Допустимо сохранить параметры модели как legacy reference, но `enabled` должен быть `false`.

### 3.2. SD runtime guard verification

Проверить, что `PipelineRunner` не вызывает `SDLabeler.label_blocks()` без явного двойного разрешения:

```text
enabled == true
AND explicit_legacy_mode == true
```

Требование:

- default path не создаёт LLM calls;
- upload/reprocess не вызывает SD provider;
- `sd_distribution` для новых default ingestion остаётся `{}`;
- `sd_level` в query API остаётся backward-compatible input и игнорируется как фильтр.

### 3.3. Optional improvement: lazy SDLabeler construction

Если безопасно и быстро: не создавать `SDLabeler` в default path.

Возможный вариант:

```python
self.sd_labeler = SDLabeler(self._legacy_sd_cfg) if self.legacy_sd_enabled else None
```

И в местах вызова:

```python
if self.legacy_sd_enabled and self.sd_labeler is not None:
    blocks = self.sd_labeler.label_blocks(blocks)
```

Это уменьшит зависимость default ingestion от legacy OpenAI SD class.

Если изменение окажется рискованным — оставить текущую конструкцию, но обязательно покрыть тестом отсутствие `label_blocks` calls в default path.

### 3.4. UTF-8 artifact fix

Исправить генерацию PRD/runtime smoke artifacts так, чтобы русские строки сохранялись корректно:

- `Кузница Духа`;
- `Саламат Сарсекенов`;
- `123__кузница_духа`;
- `Основной источник базы`.

Добавить проверку против mojibake markers:

```text
Ð
Ñ
\u0085
```

Тест/validator должен падать, если runtime smoke содержит такие маркеры в JSON output.

### 3.5. Docs final sync

Проверить и при необходимости обновить:

- `Bot_data_base/README.md`;
- `Bot_data_base/docs/LEGACY_SD.md`;
- `Bot_data_base/docs/PROJECT_STATE.md`;
- `docs/PROJECT_STATE.md`;
- `docs/PRD_INDEX.md`.

Docs должны говорить:

```text
SD is deprecated legacy metadata.
Default ingestion does not run SD labeling.
Explicit legacy SD mode requires two flags and is not part of production runtime.
```

## 4. Strict Non-Goals

Запрещено:

1. Не менять `Bot_data_base/data/processed/all_blocks_merged.json`.
2. Не менять содержимое `Кузница Духа`.
3. Не менять governance authority fields:
   - `chunk_type`;
   - `allowed_use`;
   - `safety_flags`;
   - `source_id`;
   - `block_id`;
   - `text`.
4. Не делать Chroma reindex.
5. Не вызывать LLM provider.
6. Не запускать Diagnostic Center.
7. Не удалять исторические `sd_level` / `sd_distribution` поля из старых артефактов насильно.
8. Не удалять backward compatibility из query schema, если это может сломать UI/API.
