## 1. Контекст

Проект прошёл большой технический цикл:

```text
KB governance
→ Chroma/API retrieval
→ LLM enrichment
→ review workflow
→ auto-decisions
→ controlled apply
→ admin live gate
→ Chroma reconciliation
→ source hygiene
→ legacy SD cleanup
→ BotDB docs alignment
→ UTF-8/config finalization
```

Сейчас важно не начинать Diagnostic Center сразу “вслепую”, а сделать короткий финальный readiness PRD, который:

1. собирает текущее состояние системы в один machine-readable snapshot;
2. создаёт human-readable итоговый отчёт;
3. проверяет, что readiness gates не деградировали после `PRD-046.0.10-HF1`;
4. нормализует docs/roadmap порядок следующих PRD;
5. не мутирует production KB, registry, Chroma и runtime-функциональность.

## 3. Non-goals

В рамках этого PRD строго не делать:

```text
- не реализовывать Diagnostic Center;
- не менять Writer prompt;
- не менять State Analyzer;
- не менять Thread Manager;
- не менять Context Assembly;
- не менять Diagnostic Card;
- не менять Writer Move Compliance;
- не менять retrieval policy;
- не запускать reindex Chroma;
- не запускать source reprocess;
- не запускать provider / LLM enrichment calls;
- не применять overlay;
- не менять production block text;
- не менять chunk_type / allowed_use / safety_flags;
- не включать legacy SD;
- не индексировать Neo MindBot КОНСПЕКТ как user-facing KB;
- не добавлять новые sources в production KB;
- не превращать КУЗНИЦУ ДУХА в quote source;
- не делать большие refactor;
- не скрывать blocker через warning.
```

Если любой readiness check не проходит, PRD должен завершиться честным статусом:

```text
done_with_readiness_blocker
```

а не `passed`.

## 4. Архитектурные правила, которые нельзя ломать

Сохраняются все текущие ADR/правила проекта:

```text
Multiagent-only runtime.
Writer is not the whole diagnostic system.
Context Assembly вместо everything-in-prompt.
KB is an internal lens library, not a quote source.
Deterministic governance is authority; LLM enrichment is advisory.
Raw history preserved; summaries are additive.
Turn LLM summary async with deterministic fallback.
Diagnostic Center deferred until readiness gates pass.
Applied enrichment stored as advisory metadata only.
Turn summary validator includes safety-content guards.
Retrieval eval gate mandatory before Diagnostic Center/Admin Review scale-up.
Internal-only retrieval hits suppressed for non-safety user contexts.
Post-reprocess enrichment must be rebuilt against current block ids.
BotDB Admin Dashboard uses explicit read-only summary contract.
Admin UI acceptance requires runtime/browser-visible smoke, not only TestClient.
Writer KB snippets must be boundary-aware and never cut Cyrillic words mid-token.
Admin source registry must be row-isolated and never fail silently.
Historical Chroma proof cannot override live dashboard mismatch.
Legacy SD is deprecated/default-disabled and cannot be runtime authority.
UTF-8 artifacts must be clean from mojibake markers.
```

Если в ходе проверки будет найдено нарушение этих правил, оно должно стать blocker.
