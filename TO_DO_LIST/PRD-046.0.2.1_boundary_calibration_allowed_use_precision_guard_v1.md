# PRD-046.0.2.1 — Boundary Calibration / Allowed-Use Precision Guard v1

## 0. Краткое резюме

**Цель PRD-046.0.2.1:** довести слой `Bot_data_base` structure-aware chunking + governance до состояния, в котором governed chunks можно будет безопасно использовать в следующем этапе runtime filtering.

PRD-046.0.2 успешно исправил главный провал PRD-046.0.1:

```text
до: synthetic practice_manual -> theory=3
после: synthetic practice_manual -> lens=1, practice=1, safety=1
```

Но аудит после PRD-046.0.2 выявил три хвоста:

1. `PRD-046.0.2_IMPLEMENTATION_REPORT.md` остался с `Commit hash: pending / Push: pending`.
2. `mixed_intent_risk` всё ещё срабатывает на chunks, где это может быть не реальное смешение, а допустимое упоминание соседней роли.
3. В `governance_adapter.py` профиль `practice_manual` всё ещё задаёт `practice_suggestion` как default `allowed_use`, из-за чего есть риск, что lens/safety chunks из практического материала будут ошибочно доступны как практика.

Главный принцип:

```text
Не переходить к runtime filtering, пока allowed_use и mixed_intent_risk не стали достаточно точными.
```

### Почему нужен этот PRD

Следующий логичный этап после governance bridge — runtime filtering. Но включать runtime filtering сейчас рано, потому что:

```text
Если allowed_use слишком широкий,
runtime начнёт считать lens/safety chunks практиками.

Если mixed_intent_risk слишком шумный,
система будет либо чрезмерно блокировать полезные chunks,
либо игнорировать важный warning.

Если тестовый контур Bot_data_base не полностью зелёный,
дальнейшие PRD будут строиться на нестабильном основании.
```

## Task 2 — Tighten allowed_use policy

Изменить правила так, чтобы `allowed_use` строился в два слоя:

### 2.1. Source/profile-level base use

Пример:

```python
practice_manual:
  base_allowed_use = ["writer_context", "diagnostic_lens"]
  base_safety_flags = ["requires_grounding", "not_for_direct_quote"]

architecture_notes:
  base_allowed_use = ["internal_only", "style_guidance", "diagnostic_lens"]
  base_safety_flags = ["source_style_not_user_facing", "not_for_direct_quote"]

general_book:
  base_allowed_use = ["writer_context"]
  base_safety_flags = ["not_for_direct_quote"]

transcript:
  base_allowed_use = ["writer_context", "diagnostic_lens"]
  base_safety_flags = ["requires_grounding", "not_for_direct_quote"]
```

### 2.2. Chunk-type-specific allowed use

Добавлять только по реальному `chunk_type`:

```text
chunk_type=practice:
  add practice_suggestion

chunk_type=safety:
  add safety_protocol
  add writer_context

chunk_type=style:
  add style_guidance
  if architecture profile: add internal_only

chunk_type=lens:
  add diagnostic_lens

chunk_type=quote:
  add writer_context only if not_for_direct_quote is present
  never add practice_suggestion

chunk_type=case:
  writer_context, diagnostic_lens if safe

chunk_type=excluded:
  allowed_use = ["do_not_use"]
```

### 2.3. Required assertions

After this PRD:

```text
lens block from practice_manual:
  must NOT have practice_suggestion

safety block from practice_manual:
  must NOT have practice_suggestion
  must have safety_protocol

practice block:
  must have practice_suggestion

architecture non-practice block:
  must NOT have practice_suggestion
  must have internal_only/style_guidance
```

## Task 3 — Calibrate mixed_intent_risk

Current behavior reports `mixed_intent_risk count: 2` for synthetic practice manual after PRD-046.0.2.

This may be partly false positive because:
- practice title can mention a lens, e.g. "Стоп-кадр избегания";
- safety block may say "эта практика не заменяет специалиста";
- a lens section may mention "практика" as context.

Add a more nuanced trace.

### 3.1. New fields in `chunking_quality`

Add or update:

```json
{
  "mixed_intent_risk": true,
  "mixed_intent_severity": "none|low|medium|high",
  "primary_role": "practice|safety|lens|architecture|case|quote|theory|unknown",
  "secondary_role_markers": ["practice", "lens"],
  "role_marker_counts": {...},
  "mixed_intent_reason": "..."
}
```

### 3.2. Calibration rules

Recommended:

```text
If section_role_hint is strong and secondary marker count is only 1:
  severity = low
  mixed_intent_risk = false or low only

If safety section contains phrase "эта практика не заменяет специалиста":
  do not count "практика" as competing practice intent

If practice title/body contains one lens word like "избегание":
  mark secondary_role_markers=["lens"], severity=low, not high risk

If body contains substantial markers from 2+ roles:
  severity=medium/high
  mixed_intent_risk=true

If chunk_type=safety and body includes crisis/safety + practice steps:
  severity=medium/high
  governance_notes include "safety_practice_mixed"
```

### 3.3. Acceptance target

Synthetic practice manual from PRD-046.0.2 should become:

```text
lens=1, practice=1, safety=1
high_or_medium_mixed_intent_risk <= 0 or <=1
false-positive mixed intent reduced
```

Do not hide real mixed chunks. Mixed-risk synthetic should still produce a warning.
