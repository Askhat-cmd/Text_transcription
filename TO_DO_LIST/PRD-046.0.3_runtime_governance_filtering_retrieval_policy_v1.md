# PRD-046.0.3 — Runtime Governance Filtering / Retrieval Policy v1

## 0. Краткое резюме

**Цель:** подключить governance metadata к runtime-пути `bot_psychologist`, но безопасно: не усиливать Writer, не запускать Diagnostic Center, не цитировать базу напрямую.

После PRD-046.0.1 и PRD-046.0.2.x `Bot_data_base` уже умеет:

- принимать `governance_profile` / `source_kind` при загрузке;
- сохранять `governance` и `chunking_quality` в `UniversalBlock`;
- отдавать governance summary через `/api/query`;
- прокидывать `heading_path`, `section_role_hint`, `mixed_intent_*`, `allowed_use`, `safety_flags`.

Теперь нужно научить `bot_psychologist` применять эти поля как **runtime policy filter** перед попаданием knowledge chunks в `ContextAssemblyPackage` и Writer prompt.

Главный принцип:

```text
retrieved chunks ≠ automatically usable chunks
```

RAG может найти похожий фрагмент, но runtime должен решить:

```text
можно ли этот chunk показывать Writer;
можно ли использовать его как diagnostic lens;
можно ли предлагать практику;
нужно ли скрыть direct quote;
нужно ли отбросить chunk из-за mixed/high risk;
нужно ли оставить его только как internal signal.
```

---

## 1. Почему это нужно

Сейчас `MemoryRetrievalAgent._load_rag(...)` превращает результаты retriever в `SemanticHit`, а затем `knowledge_rag_hits` формируются только с:

```python
chunk_id, source, score, content
```

Проблема: governance metadata уже есть в `Block` / `RetrievedChunk`, но на уровне `MemoryRetrievalAgent` она почти не влияет на то, что попадёт в Writer.

Это опасно для материалов типа:

- `КУЗНИЦА ДУХА`;
- `Neo MindBot КОНСПЕКТ`;
- будущие практические / диагностические / архитектурные материалы.

Потому что база знаний должна быть библиотекой линз и практик, а не источником прямых авторитетных цитат.

---

## 2. Цель PRD

Добавить deterministic runtime governance filtering layer в `bot_psychologist`:

```text
SemanticHit/raw Block governance
    ↓
KnowledgePolicyDecision
    ↓
filtered knowledge_rag_hits
    ↓
ContextAssemblyPackage + trace
    ↓
Writer sees only policy-allowed, sanitized knowledge context
```

---

## 3. Non-goals

В рамках этого PRD нельзя:

1. Делать Diagnostic Center v1.
2. Менять Writer prompt по смыслу.
3. Подключать `КУЗНИЦУ` как user-facing источник цитат.
4. Делать LLM-based filtering.
5. Делать новый retrieval ranking model.
6. Переделывать `Bot_data_base` chunker.
7. Менять Web Admin UI.
8. Убирать существующий fallback retrieval.
9. Ломать `SemanticHit` backward compatibility.
10. Давать Writer raw chunks с `internal_only`, `do_not_use`, `source_style_not_user_facing`, `not_for_direct_quote` без policy handling.

```python
@dataclass
class KnowledgePolicyDecision:
```

Методы:


### 5.2. KnowledgePolicyTrace

Создать dataclass или plain dict builder:

```python
{
  "version": "knowledge_policy_trace_v1",
  "input_hits_count": 0,

  "included_diagnostic_count": 0,
```

## 6. Policy rules v1

### 6.1. Inputs

Policy принимает list of raw hits. Hit может быть:

- `SemanticHit` with governance field, если расширите contract;
- dict из `knowledge_rag_hits`;
- block-like object from retriever;
- legacy hit без governance.

Нужно поддержать backward compatibility.

### 6.2. Legacy fallback

Если governance отсутствует:

```text
allowed_use = [writer_context]
safety_flags = [legacy_no_governance]
action = include_writer_context
risk_flags += legacy_no_governance
```

Но такой chunk должен получить низкий приоритет после governed chunks.

### 6.3. do_not_use / excluded

Если:

```text
allowed_use contains do_not_use
or chunk_type == excluded
```

то:

```text
action = drop
reason = do_not_use_or_excluded
```

### 6.4. internal_only / architecture notes

Если:

```text
allowed_use contains internal_only
or safety_flags contains source_style_not_user_facing
```

то нельзя отдавать content в Writer context.

Возможное решение v1:

```text
action = internal_only
allowed_for_writer = false
allowed_for_diagnostic = true only if diagnostic_lens present and no high risk
sanitized_content = ""
```

### 6.5. not_for_direct_quote

Если:

```text
safety_flags contains not_for_direct_quote
```

то Writer не должен получать длинный raw text.

V1 rule:

- если chunk allowed as `writer_context`, дать только короткий sanitized preview max 240 chars;
- добавить reason `content_sanitized_not_for_direct_quote`;
- не передавать кавычки/длинные фрагменты как будто это цитата.

### 6.6. practice_suggestion

Если:

```text
allowed_use contains practice_suggestion
and chunk_type == practice
```

то:

```text
allowed_for_practice = true
```

Но:

- если `practice_metadata.low_resource_safe == false`, добавить reason `practice_requires_low_resource_check`;
- если current thread/state is low-resource/hypo, не отдавать как Writer context, только как candidate/internal;
- если нет доступа к state здесь, оставить flag для будущего planner:
  - `risk_flags += practice_requires_low_resource_check`.

### 6.7. diagnostic_lens

Если:

```text
allowed_use contains diagnostic_lens
```

то:

```text
allowed_for_diagnostic = true
```

Но Writer v1 не должен получать diagnostic lens как прямую теорию, если одновременно:

- `source_style_not_user_facing`;
- `too_strong_claim`;
- `spiritual_authority_risk`;
- `clinical_risk`;
- `mixed_intent_severity in {medium, high}`.

### 6.8. mixed_intent

Если:

```text
chunking_quality.mixed_intent_risk == true
and mixed_intent_severity in {medium, high}
```

то:

```text
action = internal_only or drop
```

Recommended:

- `high` → drop from Writer context;
- `medium` → internal_only unless chunk_type == safety.

### 6.9. safety chunks

If chunk_type is `safety` and `allowed_use` contains `safety_protocol`:

- allowed_for_writer can be true only as short safety context;
- max sanitized content 240 chars;
- never treat as practice suggestion.

## 7. Integration points

### 7.1. Extend SemanticHit

Update:

```text
bot_psychologist/bot_agent/multiagent/contracts/memory_bundle.py
```

Add optional fields:

```python
governance: dict[str, Any] = field(default_factory=dict)
chunking_quality: dict[str, Any] = field(default_factory=dict)
```

Update:

```text
to_dict/from_dict
```

Backward compatible: old payloads without fields must still load.

### 7.2. Preserve governance in MemoryRetrievalAgent._load_rag

Update:

```text
bot_psychologist/bot_agent/multiagent/agents/memory_retrieval.py
```

When block has:

```python
block.governance
block.chunking_quality
```

put them into `SemanticHit`.

### 7.3. Apply runtime policy before knowledge_rag_hits

In `MemoryRetrievalAgent.assemble(...)`, after score filtering and sorting:

```text
filtered_hits -> apply_knowledge_policy_v1(...) -> policy decisions
```

Build:

```python
knowledge_rag_hits = [decision.to_writer_hit_dict() for included writer decisions]
```

Minimum writer hit dict:

```python
{
  "chunk_id": ...,
  "source": ...,
  "score": ...,
  "content": decision.sanitized_content,
  "governance_summary": {
    "chunk_type": ...,
    "allowed_use": ...,
    "safety_flags": ...,
    "policy_action": ...,
    "policy_reasons": [...]
  }
}
```

Important: `content` must be sanitized according to policy.

### 7.4. Add policy trace to MemoryBundle

Extend `MemoryBundle` with:

```python
knowledge_policy_trace: dict[str, Any] = field(default_factory=dict)
```

Update `to_dict/from_dict`.

### 7.5. Propagate trace into ContextAssemblyPackage

Update context assembly trace to include safe summary:

Option A, minimal:

- keep trace in `memory_bundle.knowledge_policy_trace` only;
- add into orchestrator debug.

Option B, preferred:

- add `knowledge_policy_trace` field to `ContextAssemblyTrace` or package.

Do the minimal safe option if contract changes become too large.

### 7.6. Orchestrator debug

Update orchestrator debug to include:

```text
knowledge_policy_trace summary
```

without full raw content.

## 11. Acceptance criteria

PRD is accepted only if:

1. Governance metadata is preserved in `SemanticHit`.
2. Runtime policy filters `knowledge_rag_hits` before Context Assembly.
3. `internal_only` chunks do not expose raw content to Writer.
4. `not_for_direct_quote` chunks are sanitized/truncated.
5. `practice_suggestion` is only allowed for `chunk_type=practice`.
6. `mixed_intent_severity=high` does not enter Writer context.
7. Legacy chunks still work with trace warning.
8. `knowledge_policy_trace_v1` appears in debug/report.
9. Existing multiagent tests pass.
10. Writer prompt / DiagnosticCard behavior remains unchanged.
