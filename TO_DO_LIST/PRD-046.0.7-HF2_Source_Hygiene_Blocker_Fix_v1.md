# PRD-046.0.7-HF2 — Source Hygiene Blocker Fix v1

## 0. Краткое назначение

`PRD-046.0.7-HF2` закрывает readiness-blocker, найденный после `PRD-046.0.7-HF1`:

```text
multiple_active_sources_without_allowlist
```

Цель PRD — безопасно перевести тестовые / пустые / зависшие источники BotDB из active-состояния в `archived`, не трогая основной источник `КУЗНИЦА ДУХА`, не изменяя knowledge blocks, не переиндексируя Chroma и не запуская reprocess.

После HF2 система должна быть готова к следующему этапу:

```text
PRD-046.0.8 — Clean Source Reprocess from Single Active Source v1
```

---

## 1. Контекст

После `PRD-046.0.7-HF1` состояние проекта такое:

- BotDB Admin/API legacy SD active path отключён.
- В UI/Admin SD-визуалы должны быть удалены.
- Source hygiene audit/plan/readiness gate внедрены.
- Registry не мутировался, потому что HF1 был dry-run/readiness этапом.
- Reprocess пока заблокирован.

Ключевые результаты HF1:

```text
total_sources: 7
unique_source_ids: 3
sources_with_blocks: 1
sources_zero_blocks: 6
focus_source_id: 123__кузница_духа
focus_source_blocks: 229
planned_archive_count: 6
mutation_mode: dry_run
registry_mutated: false
legacy_sd_filter_still_active: false
ready_for_clean_reprocess: false
blocker: multiple_active_sources_without_allowlist
```

В registry остаются active/test/stale источники:

```text
test123
avtor__книга
```

Они имеют `0` blocks и могут иметь статусы `failed` / `processing`, но пока всё ещё считаются active/stale и мешают clean reprocess.

## 2. Главная цель

Закрыть blocker `multiple_active_sources_without_allowlist` через controlled archive/apply workflow.

После реализации:

```text
active_source_count == 1
active_source_id == 123__кузница_духа
focus_source_protected == true
legacy_sd_active == false
ready_for_clean_reprocess == true
```

Если readiness gate всё ещё не `ready`, PRD должен честно зафиксировать оставшиеся blockers/warnings и не переходить к reprocess.

## 4. Архитектурное решение

### 4.1. Cleanup через archive, не delete

Тестовые источники должны быть не физически удалены, а переведены в статус:

```text
archived
```

Причина:

- сохраняется audit trail;
- проще восстановить/проверить историю;
- безопаснее перед clean reprocess;
- меньше риск случайного удаления нужного файла.

Hard delete может остаться как отдельная возможность для будущего Admin UI, но не является целью HF2.

### 4.2. Source of truth

Главный рабочий источник сейчас:

```text
source_id: 123__кузница_духа
title: Кузница Духа
blocks: 229
```

Он должен быть защищён на всех уровнях:

- plan;
- apply;
- tests;
- readiness gate;
- reports.

### 4.3. Граница PRD

HF2 завершает только hygiene blocker fix:

```text
dry-run evidence from HF1
→ controlled apply archive
→ after-audit
→ readiness gate
→ docs update
```

Полный reprocess будет отдельным PRD.

## 5. Требуемые изменения

### A. Controlled archive apply

Использовать существующий workflow `source_hygiene_apply.py`, но адаптировать его под PRD-046.0.7-HF2 outputs.

Ожидаемая команда:

```bash
python Bot_data_base/tools/source_hygiene_audit.py \
  --source-prd PRD-046.0.7-HF2 \
  --output-json TO_DO_LIST/logs/PRD-046.0.7-HF2/source_hygiene_audit_before.json \
  --output-md TO_DO_LIST/reports/PRD-046.0.7-HF2_SOURCE_HYGIENE_AUDIT_BEFORE_REPORT.md

python Bot_data_base/tools/source_hygiene_apply.py \
  --source-prd PRD-046.0.7-HF2 \
  --audit-json TO_DO_LIST/logs/PRD-046.0.7-HF2/source_hygiene_audit_before.json \
  --plan-json TO_DO_LIST/logs/PRD-046.0.7-HF2/source_hygiene_plan.json \
  --apply-result-json TO_DO_LIST/logs/PRD-046.0.7-HF2/source_hygiene_apply_result.json \
  --output-md TO_DO_LIST/reports/PRD-046.0.7-HF2_SOURCE_HYGIENE_APPLY_REPORT.md \
  --apply \
  --confirm
```

Требования к apply:

- `mode=apply`;
- `confirm=true` обязателен;
- `registry_mutated=true`, если реально архивированы stale/test sources;
- `focus_source_protected=true`;
- `applied_mutation_count == planned_archive_count`, если нет новых обстоятельств;
- `blocked_actions_count == 0`;
- `errors == []`;
- registry snapshot до мутации сохранён.

### B. After-audit

После archive/apply обязательно повторить source hygiene audit уже на изменённом registry.

Ожидаемая команда:

```bash

  --source-prd PRD-046.0.7-HF2 \
  --output-json TO_DO_LIST/logs/PRD-046.0.7-HF2/source_hygiene_audit_after.json \
```

### D. Admin UI hygiene visibility

Обновить BotDB Admin UI так, чтобы после archive/apply пользователь видел чистую картину.

Минимальные требования:

1. Dashboard:
   - не показывает SD-блок;
   - показывает Chroma count;
   - показывает blocks count;
   - показывает source hygiene readiness summary, если уже есть поле `governance_readiness`.

2. Registry:
   - SD-колонки нет;
   - у источников есть статус `archived`;
   - archived sources либо скрыты по умолчанию, либо явно помечены badge `archived`;
   - `Кузница Духа` видна как единственный active/done источник с 229 blocks;
   - test/stale sources не должны выглядеть как активные рабочие источники.

3. Не добавлять destructive delete UI в этом PRD.

Допустимо добавить только:

```text
show archived / hide archived
```

если это маленькое и безопасное изменение.

### E. API / compatibility

Проверить, что `/api/registry/` после archive/apply:

- возвращает archived sources со статусом `archived`, если UI их запрашивает;
- не считает archived sources как active для readiness;
- не ломает upload/book/youtube forms.

Проверить, что `/api/registry/stats`:

- `legacy_sd_active=false`;
- `sd_distribution={}` или deprecated-compatible пустое значение;
- `governance_readiness.archive_candidate_sources` уменьшается после archive;
- `governance_readiness.focus_source_count == 1`.

## 11. Acceptance Criteria

PRD считается выполненным, если:

1. Registry cleanup выполнен через controlled archive, не hard delete.
2. `КУЗНИЦА ДУХА` осталась active/done и имеет 229 blocks.
3. Все zero-block test/stale sources переведены в `archived` или явно объяснены в blockers.
4. `active_source_count == 1`.
5. `multiple_active_sources_without_allowlist` исчез из readiness gate.
6. `legacy_sd_active == false`.
7. `all_blocks_merged.json` не изменился.
8. Chroma не переиндексировался и count остался 229.
9. Runtime bot path не изменился.
10. Все обязательные тесты прошли.
11. Docs обновлены:

```text
docs/PROJECT_STATE.md
docs/ROADMAP.md
docs/PRD_INDEX.md
docs/DECISIONS.md, если принято новое ADR
```

12. Сделан commit + push в `main`.
13. После push обновлён implementation report с commit hash.
