# PRD-046.0.7.1 — Controlled Review Decision Apply v1

## 1. Контекст

Проект находится в состоянии `post-PRD-046.0.9.3`.

Ключевые факты текущего состояния:

- `all_blocks_merged.json` согласован с post-reprocess baseline:
  - `blocks_total = 247`;
  - fresh RUN1 review queue coverage = `87/87`;
  - missing block ids = `0`.
- `PRD-046.0.9-RUN1` выполнил real provider enrichment по `247/247` blocks без production mutation.
- `PRD-046.0.9.3` сгенерировал валидный auto-decisions overlay:
  - `decisions_count = 87`;
  - `approved = 28`;
  - `needs_edit = 12`;
  - `rejected = 1`;
  - `defer = 46`;
  - `coverage_percent = 100.0`;
  - `apply_ready = true`.
- Production apply/reindex после `PRD-046.0.9.3` не выполнялись.
- Governance authority поля остаются deterministic authority и не должны мутироваться через LLM/review overlay.

Этот PRD является первым apply-шагом после автоматизированной review policy.

---

## 2. Цель PRD

Безопасно применить validated review decisions overlay к production KB metadata.

Главная цель:

```text
validated enrichment + validated review decisions
→ controlled apply plan
→ backup
→ production metadata mutation only for allowed advisory fields
→ Chroma refresh/reindex only if required
→ retrieval/admin/runtime smoke
→ updated reports/docs
```

Этот PRD должен аккуратно перевести часть validated enrichment из candidate/review артефактов в production `metadata.llm_enrichment`, не меняя смысловые/governance authority поля.

---

## 3. Принципиальное правило apply

Нельзя применять все 87 review items одинаково.

Решения применяются так:

```text
approved   → apply only approved advisory fields from RUN1 enrichment
needs_edit → apply edited_fields from decision overlay
rejected   → do not apply enrichment for this item
defer      → do not apply enrichment for this item
```

Дополнительно:

```text
safe non-review RUN1 items → may be applied if they passed RUN1 validation and are not in review queue
```

Итоговый apply должен различать:

1. `safe_non_review_apply`
2. `review_approved_apply`
3. `review_needs_edit_apply`
4. `review_rejected_skip`
5. `review_defer_skip`

Ожидаемая логика по количествам:

```text
total_blocks = 247
review_items = 87
review_apply_candidates = approved + needs_edit = 28 + 12 = 40
review_skip = rejected + defer = 1 + 46 = 47
safe_non_review_candidates = 247 - 87 = 160
max_expected_apply_candidates = 160 + 40 = 200
```

Фактический `updated_blocks` может быть меньше, если часть metadata уже совпадает с целевым состоянием.

## 7. Allowed metadata mutation

The only allowed production KB mutation is inside:

```text
block.metadata.llm_enrichment
```

Optional namespaced audit field is allowed only if it does not affect retrieval/runtime semantics:

```text
block.metadata.llm_enrichment_apply
```

or

```text
block.metadata.review_decision_apply
```

This audit field may include:

```json
{
  "source_prd": "PRD-046.0.7.1",
  "decision_source_prd": "PRD-046.0.9.3",
  "run_source_prd": "PRD-046.0.9-RUN1",
  "decision": "approved|needs_edit|skipped_defer|skipped_rejected|safe_non_review",
  "applied_at": "...",
  "applied_fields": ["summary", "tags", "use_when", "avoid_when"]
}
```

Do not write raw/full text into this audit field.

## 8. Field-level rules

### 8.1. Allowed advisory fields

Only these fields may be applied:

```text
summary
tags
lens_family_candidates
use_when
avoid_when
self_contained_score
self_contained_reason
split_merge_suggestion
confidence
```

### 8.2. Forbidden fields

Never mutate:

```text
text
content
content_full
full_text
raw_text
source_raw
chunk_type
allowed_use
safety_flags
source_id
block_id
governance
embedding
vector
not_for_direct_quote
source_style_not_user_facing
internal_only
do_not_use
```

### 8.3. Approved decisions

For `approved` decisions:

- apply only `approved_fields` from the RUN1 enrichment artifact;
- if `approved_fields` is empty, do not apply anything for that block and report it;
- do not infer additional fields.

### 8.4. Needs_edit decisions

For `needs_edit` decisions:

- apply only `edited_fields`;
- do not apply original RUN1 values for fields that were rejected;
- do not apply any field not listed in allowed advisory fields;
- preserve guardrail language exactly as validated unless it contains forbidden/private/raw values.

### 8.5. Rejected decisions

For `rejected` decisions:

- do not apply RUN1 enrichment;
- do not overwrite current production metadata;
- record skip in apply report only;
- optional production audit metadata should not be written unless explicitly justified and validated.

### 8.6. Defer decisions

For `defer` decisions:

- do not apply RUN1 enrichment;
- do not overwrite current production metadata;
- record skip in apply report only;
- keep deferred items available for future review cycle.
