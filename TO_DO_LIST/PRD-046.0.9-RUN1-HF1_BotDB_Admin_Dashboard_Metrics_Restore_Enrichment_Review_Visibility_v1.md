# PRD-046.0.9-RUN1-HF1 — BotDB Admin Dashboard Metrics Restore + Enrichment Review Visibility v1

## 1. Назначение PRD

Этот PRD является коротким hotfix после `PRD-046.0.9-RUN1`.

`PRD-046.0.9-RUN1` успешно выполнил real provider enrichment:

- provider called: `true`
- model: `gpt-4o-mini`
- blocks processed: `247`
- items completed: `247`
- validation errors: `0`
- validation warnings: `0`
- missing real provider output: `0`
- production KB mutated: `false`
- registry mutated: `false`
- Chroma mutated: `false`
- review queue after real enrichment: `87` items (`P0=0`, `P1=1`, `P2=86`)

Однако ручная проверка Web Admin показала UI-проблему:

- в Registry видно `Кузница Духа`, `247` blocks, protected source и archived/test sources;
- в Dashboard карточки показывают `—` вместо реальных чисел;
- блок “Последние источники” пустой;
- `Governance readiness` не показывает состояние.

Это не блокирует runtime/retrieval, но ухудшает управляемость проекта. Перед human review нужно, чтобы админка показывала реальное состояние базы, Chroma, governance и enrichment/review очереди.

Главная цель PRD — восстановить dashboard metrics и добавить compact visibility по post-reprocess enrichment/review state.

## 4. Scope

### 4.1. Dashboard metrics API contract

Нужно определить или восстановить единый read-only endpoint для dashboard summary.

Допустимые варианты:

- расширить существующий dashboard/status endpoint;
- добавить новый endpoint, например:
  - `GET /api/admin/dashboard`
  - или `GET /api/dashboard`
  - или другой путь, соответствующий текущей архитектуре.

### 4.2. Dashboard UI restore

Dashboard должен показывать:

1. Источников:
   - total;
   - active/protected желательно как subtext.

2. Блоков:
   - `247`.

3. ChromaDB:
   - `247`;
   - если Chroma недоступна — не `—`, а явный warning.

4. Governance readiness:
   - `ready`;
   - legacy SD inactive;
   - completeness rates.

5. Enrichment / Review:
   - provider called;
   - items completed `247`;
   - review queue `87`;
   - P0/P1/P2 counts.

6. Последние источники:
   - `Кузница Духа`;
   - protected badge;
   - blocks `247`;
   - status `done`.

Если endpoint/API падает:

- dashboard должен показать error banner;
- карточки могут показывать `—`, но рядом должна быть причина:
  - `API unavailable`
  - `Chroma unavailable`
  - `registry read failed`
  - `dashboard summary failed`
- нельзя молча показывать пустой dashboard без причины.

### 4.3. Registry page sanity check

Registry сейчас в целом работает. Нужно только проверить, что изменения dashboard не ломают registry.

Acceptance:

- Registry still shows `Кузница Духа` with `247` blocks.
- Protected source cannot be deleted.
- Archived/test sources remain visible.
- Delete buttons remain disabled/protected where required.
- No source mutation occurs during dashboard load.

### 4.4. Enrichment state visibility

Dashboard должен читать состояние из последних PRD artifacts или derived read-only summary:

Preferred artifacts:

- `TO_DO_LIST/logs/PRD-046.0.9-RUN1/real_enrichment_scorecard.json`
- `TO_DO_LIST/logs/PRD-046.0.9-RUN1/review_queue_after_real_enrichment.json`
- `TO_DO_LIST/reports/PRD-046.0.9-RUN1_IMPLEMENTATION_REPORT.md` only as fallback, not primary.

Если artifacts не найдены:

- dashboard должен показать `enrichment.status = unknown`;
- не падать;
- не блокировать базовые source/block/chroma stats.
