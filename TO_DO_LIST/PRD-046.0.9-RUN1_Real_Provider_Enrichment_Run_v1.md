# PRD-046.0.9-RUN1 — Real Provider Enrichment Run after Post-Reprocess Rebaseline v1

## 1. Назначение PRD

Этот PRD выполняет реальный LLM enrichment run после `PRD-046.0.9`.

`PRD-046.0.9` успешно построил post-reprocess baseline для текущих `247` production block ids, но не вызвал LLM provider:

- `provider_status = not_called`
- `mode = validate-existing`
- `enrichment_items_total = 247`
- `validated_items_total = 247`
- `validation_errors_count = 0`
- `validation_warnings_count = 562`
- `review_queue_items_count = 247`
- все `247` items имеют `needs_human_review=true`
- `summary_present_rate = 0.0`
- `tags_present_rate = 0.0`
- `use_when_present_rate = 0.0`
- `avoid_when_present_rate = 0.0`
- `lens_family_candidates_present_rate = 0.0`

Главная причина: overlay был создан как пустой/placeholder candidate с `review_reasons=["missing_real_provider_output"]`.

Цель этого PRD — запустить настоящий provider-backed enrichment в controlled `real` mode, получить реальные advisory summaries/tags/use_when/avoid_when/lens candidates для текущих `247` block ids и заново построить review queue уже на основе реального enrichment.

Этот PRD НЕ применяет enrichment в production.

---

## 2. Почему нужен отдельный RUN1

Нельзя переходить к `Human Review Decisions` поверх текущей очереди `247 P0`, потому что сейчас все P0 вызваны не качеством LLM-output, а отсутствием LLM-output.

Текущая review queue говорит:

```text
247 items need review because provider output is missing.
```

Это не предмет human review. Сначала нужно получить реальный enrichment.

Правильная последовательность:

```text
PRD-046.0.9
baseline/inventory/empty overlay/contracts

PRD-046.0.9-RUN1
real provider enrichment run

затем:
validator calibration или human review / controlled apply
```

LLM enrichment remains advisory.

- summary
- tags
- lens_family_candidates

- user_state_fit

## 7. Prompt and output safety

### 7.1. Prompt discipline

LLM prompt must instruct:

- do not quote source text directly;
- summarize in neutral Russian;
- do not imitate author’s voice;
- do not give direct clinical claims;
- do not classify user;
- do not produce therapy instructions as if for user directly;
- generate metadata for internal retrieval/context only;
- preserve “internal lens library, not quote source” principle.

### 7.2. Output limits

Per block:

- summary max 1–2 concise sentences;
- tags max 8;
- lens_family_candidates max 5;
- use_when max 5;
- avoid_when max 5;
- retrieval_queries max 5;
- writer_guidance max 350 chars;
- no raw passages copied over configured quote threshold.

- docs updated;
- commit and push completed.

### 13.2. Partial accepted state

Partial run can be accepted only if:

- provider called successfully;
- completed items are valid;
- failed/unprocessed items are explicitly listed;
- progress is resumable;
- next recommendation is continuation RUN2;
- no production mutation occurred.

### 13.3. Blocked state

If provider unavailable, budget missing, or credits unavailable:

- do not run fake enrichment;
- write `blocked_by_provider` or `blocked_by_budget`;
- keep production unchanged;
- next recommendation: rerun after provider/budget fix.
