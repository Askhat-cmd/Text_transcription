# PRD-046.1.11 — Prompt-Constraint Pilot Supervised Results Consolidation / Rollout Decision Gate v1

## 0. Краткое название

**Supervised Results Consolidation / Rollout Decision Gate v1**

## 1. Контекст

После `PRD-046.1.9` и `PRD-046.1.10` Prompt-Constraint Pilot прошёл два controlled supervised цикла.

Это даёт право **не на broad rollout**, а на consolidated rollout-decision gate: собрать evidence двух циклов, проверить повторяемость качества, сверить инварианты, подготовить решение о следующем безопасном шаге.

Текущий baseline остаётся:

```text
PROMPT_CONSTRAINT_PILOT_ENABLED=false
PROMPT_CONSTRAINT_PILOT_FORCE_DISABLED=true
PROMPT_CONSTRAINT_PILOT_MODE=shadow|test_apply
```

Архитектурная позиция не меняется:

```text
Diagnostic Center v1: trace-only shadow layer
Planner Bridge: candidate-only / shadow-eval lineage
planner_bridge_compliance_shadow: trace-only compare layer
planner_bridge_writer_contract_pilot: pilot_shadow_only
writer_prompt_replay: offline_replay_only
prompt_constraint_pilot_runtime: default-off limited allowlisted test path
```

## 2. Цель PRD

Создать **консолидационный decision gate** по двум supervised циклам.

Цель — ответить:

```text
достаточно ли стабильны результаты PRD-046.1.9 и PRD-046.1.10,
чтобы готовить следующий маленький production-limited rollout plan,
или нужно остаться в supervised режимах / сделать hotfix / остановить pilot.
```

PRD должен:

1. Прочитать артефакты PRD-046.1.9 и PRD-046.1.10.
2. Сконсолидировать метрики в единый scorecard.
3. Проверить повторяемость критичных инвариантов.
4. Сформировать risk register.
5. Проверить readiness к следующему этапу.
6. Сгенерировать rollout-decision report.
7. Не менять runtime defaults.
8. Не делать production rollout.
9. Не вызывать provider.
10. Не мутировать KB/registry/Chroma/config.

## 3. Главный принцип

```text
PRD-046.1.11 принимает решение о готовности к следующему плановому шагу,
но сам не включает production rollout.
```

После выполнения PRD обычный пользовательский путь должен остаться неизменным:

```text
normal_user_apply_count=0
default_off_user_path_effect_count=0
config_mutated=false
production_apply_performed=false
```

## 4. Non-goals

В рамках PRD запрещено:

1. Включать broad rollout.
2. Включать pilot для обычных пользователей.
3. Менять default flags в config/code.
4. Делать `PROMPT_CONSTRAINT_PILOT_ENABLED=true` новым baseline default.
5. Делать `PROMPT_CONSTRAINT_PILOT_FORCE_DISABLED=false` новым baseline default.
6. Расширять cohort или запускать новый execution cycle.
7. Использовать реальные production-users.
8. Делать provider-dependent acceptance.
9. Мутировать KB, registry, Chroma, config.
10. Менять deterministic governance authority.
11. Делать Diagnostic Center runtime authority.
12. Удалять supervised/rollback/no-mutation gates.
13. Считать consolidated decision автоматическим разрешением на rollout без отдельного PRD.

### 7.1. `prepare_production_limited_rollout_plan`

Разрешено только если:

```text
source_cycles_passed=true
both_cycles_continue_supervised=true

prompt_bloat_regression_total=0
constraint_conflict_regression_total=0
raw_kb_text_exposure_total=0
internal_only_exposure_total=0
not_for_direct_quote_violation_total=0
provider_called_total=0
production_mutation_detected_any=false
artifact_encoding_hygiene_all_passed=true
risk_register_has_blockers=false
```

Это решение не включает production rollout. Оно только разрешает следующий PRD:

```text
PRD-046.1.12 - Production-Limited Prompt-Constraint Pilot Rollout Plan v1
```

### 7.2. `stay_supervised`

Использовать, если:

```text
hard blockers отсутствуют,
но evidence недостаточно для production-limited planning
или risk register содержит medium risks, требующие ещё одного supervised cycle.
```

### 7.3. `hotfix_required`

Использовать, если:

```text
есть non-critical issue, который нужно исправить до следующей фазы,
но нет hard safety/privacy/rollback blocker.
```

### 7.4. `stop`

Использовать при hard abort:

```text
normal_user_apply_total > 0
rollback_failure_total > 0
raw_kb_text_exposure_total > 0
internal_only_exposure_total > 0
not_for_direct_quote_violation_total > 0
safety_regression_total > 0
kb_policy_regression_total > 0
production_mutation_detected_any=true
```

2. Не начинать кодинг до task list.
3. Не запускать новый supervised execution cycle.
4. Не менять baseline defaults.
5. Не включать broad rollout.
6. Не использовать реальных пользователей.
7. Не коммитить `.env`, ключи, `.venv`, `node_modules`, sqlite/db/cache.
8. Все artifacts должны быть sanitized.
9. Raw private prompts, raw KB text и raw user private logs не коммитить.
10. Все изменения должны быть provider-free.
11. Все mutation-sensitive files должны иметь no-mutation proof.
12. Запустить тесты и CLI.
13. Сохранить command output.
14. Запустить artifact encoding hygiene gate.
15. Обновить reports и living docs.
16. Сделать commit и push в `main`.
17. После push обновить implementation report: commit hash + push status.
