## 2. Цель PRD

Создать **строгий production-limited rollout plan** для Prompt-Constraint Pilot.

PRD должен подготовить план будущего ограниченного production execution, но не выполнять его.

План должен ответить:

```text
кого можно включать;
как включать;
какие флаги выставлять;
какие preflight gates обязательны;
как проверять normal-user no-effect;
как мониторить quality/safety;
как откатывать;
какие hard abort criteria;
какие артефакты обязательно сохранить;
какое решение будет принято после production-limited execution.
```

## 3. Главный принцип

```text
PRD-046.1.12 = production-limited rollout planning only.
PRD-046.1.12 не включает production-limited execution.
```

После выполнения PRD обычный пользовательский путь должен остаться неизменным:

```text
PROMPT_CONSTRAINT_PILOT_ENABLED=false
PROMPT_CONSTRAINT_PILOT_FORCE_DISABLED=true
normal_user_apply_count=0
default_off_user_path_effect_count=0
production_apply_performed=false
config_mutated=false
```

## 4. Non-goals

В рамках PRD запрещено:

1. Выполнять production-limited rollout.
2. Включать pilot для реальных пользователей.
3. Менять default flags в config/code.
4. Делать `PROMPT_CONSTRAINT_PILOT_ENABLED=true` новым default.
5. Делать `PROMPT_CONSTRAINT_PILOT_FORCE_DISABLED=false` новым default.
6. Расширять rollout автоматически.
7. Добавлять auto-enable логику.
8. Вызывать provider для acceptance.
9. Мутировать KB, registry, Chroma, config.
10. Менять deterministic governance authority.
11. Делать Diagnostic Center runtime authority.
12. Удалять или ослаблять rollback-first policy.
13. Считать planning PRD разрешением на execution без отдельного PRD.
