## 1. Контекст

Проект Bot Psychologist / Neo MindBot прошёл длинную цепочку подготовки Diagnostic Center v1:

```text
Diagnostic Center shadow
→ Planner Bridge shadow/eval
→ compliance shadow compare
→ Writer-Contract pilot shadow
→ Writer Prompt Replay offline
→ Prompt-Constraint Pilot limited runtime flag
→ runtime evidence/rollback/quality gate
→ supervised rollout planning
→ supervised cycles
→ supervised consolidation
→ production-limited planning
→ production-limited execution
→ post-run results/rollback/quality gate
→ stabilization / cleanup / eval harness consolidation
```

После `PRD-046.1.15` проект находится в состоянии:

```text
post-PRD-046.1.15-stabilization-cleanup-consolidation-passed
```

Текущий runtime-path:

```text
User message
  ↓
State Analyzer
  ↓
Thread Manager
  ↓
Memory Retrieval
  ↓
Context Assembly
  ↓
Diagnostic Card
  ↓
Diagnostic Center Shadow / Planner Bridge / Pilot trace layers
  ↓
Writer Move Compliance
  ↓
Writer
  ↓
Validator / Trace
  ↓
Memory Update
```

Diagnostic Center v1 всё ещё не должен становиться broad runtime authority.

Текущие статусы, которые нужно сохранить:

```text
Diagnostic Center v1: trace-only shadow layer
Planner Bridge: candidate-only / shadow-eval lineage
planner_bridge_compliance_shadow: trace-only compare layer
planner_bridge_writer_contract_pilot: pilot_shadow_only
writer_prompt_replay: offline_replay_only
prompt_constraint_pilot_runtime: default-off limited allowlisted test path
```

## 2. Главная цель PRD

Формально закрыть Diagnostic Center v1 как **принятый governed runtime-shadow layer**, подтвердить постоянные regression gates и зафиксировать runtime boundaries перед любым будущим расширением.

Ключевая формула:

```text
Diagnostic Center v1 accepted as governed internal map-making layer.
Broad runtime authority remains prohibited until separate future rollout PRD.
```
