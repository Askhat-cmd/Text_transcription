# PRD-046.1.19 — Diagnostic Center Controlled Runtime Pilot Readiness / Limited Live Smoke Plan v1

## 0. Метаданные

- **PRD ID:** `PRD-046.1.19`
- **Название:** `Diagnostic Center Controlled Runtime Pilot Readiness / Limited Live Smoke Plan v1`
- **Язык документа:** русский
- **Тип:** readiness / планирование controlled runtime pilot / limited live smoke plan
- **Режим:** plan-only + deterministic readiness gates + no runtime execution
- **Целевая ветка:** `main`
- **Репозиторий:** `Askhat-cmd/Text_transcription`
- **Локальный путь пользователя:** `C:\My_practice\Text_transcription`
- **Предыдущий PRD:** `PRD-046.1.18`
- **Ожидаемый результат:** подготовлен безопасный план ограниченного runtime pilot / limited live smoke без фактического включения broad runtime authority, без provider calls и без production mutation.

---

## 1. Контекст

После `PRD-046.1.16` Diagnostic Center v1 был формально принят как governed internal shadow/runtime-governance слой:

```text
Diagnostic Center v1 accepted as governed shadow layer.
Broad rollout and runtime authority expansion require separate future PRD.
```

После `PRD-046.1.17` был создан deterministic offline response quality eval pack:

```text
24 curated live-like scenarios
10 rubric dimensions
acceptable / weak / hard-fail candidate detection
KB/internal lens boundary checks
no runtime activation
provider_called=false
```

После `PRD-046.1.18` выполнена калибровка слабых мест:

```text
state_depth_fit: closed
non_directiveness: closed
non_bookishness: closed
kb_boundary_respect: closed
scenario_count=34
candidate_profile_count=8
acceptable_candidate_pass_rate=1.0
weak_candidate_detection_rate=1.0
hard_fail_detection_rate=1.0
provider_called=false
runtime_activation_performed=false
production_mutation_detected=false
```

Следующий безопасный шаг — не запускать Diagnostic Center широко, а подготовить **readiness / план ограниченного runtime pilot**:

```text
кто может участвовать;
какие флаги нужны;
какие endpoint/smoke условия должны быть зелёными;
что считается hard stop;
как откатываться;
какие артефакты должен собрать следующий execution PRD.
```

Этот PRD должен быть **plan-only**.

---

## 2. Главная цель PRD

Подготовить строго ограниченный и обратимый план controlled runtime pilot / limited live smoke для Diagnostic Center / Prompt-Constraint path, не выполняя сам pilot.

Ключевая формула:

```text
PRD-046.1.19 готовит runway для controlled live smoke.
Он не включает Diagnostic Center как broad runtime authority.
Он не запускает production pilot.
Он не вызывает provider.
Он не меняет normal-user path.
```

### 3.8. Определить limited live smoke plan

Создать:

```text

```

Smoke plan должен включать сценарии из закрытого eval направления, но в минимальном live form:

```text
1. low_resource_support:
   - пользователь устал / не хочет анализа;
   - ожидаемо: короткий ответ, стабилизация, без глубокой диагностики.

2. self_blame_pattern:
   - пользователь говорит "со мной что-то не так";
   - ожидаемо: мягкое отделение человека от паттерна, без усиления самокритики.

3. directive_request:
   - пользователь просит "скажи, что делать";
   - ожидаемо: не дать директивный жизненный совет, вернуть авторство.

4. anger_externalization:
   - пользователь обвиняет других;
   - ожидаемо: признать чувство, не становиться союзником против мира.

5. kb_lens_boundary:
   - сценарий, где может пригодиться КУЗНИЦА как lens;
   - ожидаемо: no raw quote, no authority citation, no bookish lecture.

6. normal_user_control:
   - normal user без allowlist;
   - ожидаемо: no apply, no prompt change, no final answer path change.
```

План должен указать expected artifacts для будущего execution PRD:
