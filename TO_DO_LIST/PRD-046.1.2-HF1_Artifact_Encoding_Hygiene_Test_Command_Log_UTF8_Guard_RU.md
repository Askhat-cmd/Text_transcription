# PRD-046.1.2-HF1 — PRD Artifact Encoding Hygiene / Test Command Log UTF-8 Guard v1

**Проект:** Bot Psychologist / Neo MindBot  
**Репозиторий:** `Askhat-cmd/Text_transcription`  
**Локальный путь пользователя:** `C:\My_practice\Text_transcription`  
**Язык PRD:** русский  
**Статус документа:** ready for IDE-agent implementation  
**Дата:** 2026-05-17  
**Предыдущий PRD:** `PRD-046.1.2 — Diagnostic Center v1 Shadow Divergence Calibration / Planner Bridge`  
**Следующий ожидаемый PRD после HF1:** `PRD-046.1.3 — Diagnostic Center Planner Bridge Shadow-to-Compliance Integration v1`

---

## 0. Краткое резюме для IDE-агента

`PRD-046.1.2` функционально прошёл: Planner Bridge contract готов, runtime/user-path не изменён, no-mutation proof чистый.

Но в audit GitHub найден hygiene-дефект:

```text
TO_DO_LIST/logs/PRD-046.1.2/test_command_output.txt
```

Файл содержит большое количество `NUL`-символов (`\x00`) и выглядит как UTF-16LE/неправильно записанный командный лог. Это не ломает runtime, но нарушает дисциплину артефактов проекта.

Этот HF1 должен:

```text
1. Нормализовать/перегенерировать test_command_output.txt в чистый UTF-8.
2. Добавить маленький reusable artifact encoding validator.
3. Прогнать validator на PRD-046.1.2 artifacts.
4. Зафиксировать proof, что production/runtime/KB/config/index не менялись.
5. Обновить отчёты и docs только в части hygiene state.
```

Главный принцип:

```text
Перед переходом к PRD-046.1.3 не оставлять в истории проекта грязные test logs.
```

## 2. Цель PRD

Цель — не менять бизнес-логику, Diagnostic Center, Planner Bridge или runtime.

Цель — закрыть маленький hygiene-хвост:

```text
PRD logs must be human-readable, UTF-8-safe, and validator-checked.
```

---

## 3. Goals

### 3.1. Исправить `test_command_output.txt`

Нужно привести файл:

```text
TO_DO_LIST/logs/PRD-046.1.2/test_command_output.txt
```

к нормальному UTF-8 виду.

Допустимые способы:

```text
вариант A: корректно перекодировать существующий файл из UTF-16LE/подобного состояния в UTF-8;
вариант B: заново выполнить те же test/eval commands и записать лог в UTF-8 через безопасный writer;
вариант C: если полное восстановление невозможно, заменить файл новым честным rerun-log с теми же командами и результатами.
```

Обязательно:

```text
- сохранить читаемые команды;
- сохранить exit codes;
- сохранить summaries passed;
- не фальсифицировать результаты;
- не удалять историю проблемы без mention в HF1 report.
```

### 3.2. Добавить validator для artifact encoding hygiene

Добавить reusable CLI, например:

```text
bot_psychologist/tools/validate_prd_artifact_encoding.py
```

или, если в проекте есть более подходящий общий tools-layout, использовать его.

CLI должен уметь:

```text
python -m bot_psychologist.tools.validate_prd_artifact_encoding \
  --prd PRD-046.1.2 \
  --logs-dir TO_DO_LIST/logs/PRD-046.1.2 \
  --reports-dir TO_DO_LIST/reports
```

Минимальные проверки:

```text
- файл читается как UTF-8;
- нет NUL bytes / NUL characters;
- нет массовых replacement chars `�`;
- нет типичных mojibake markers сверх порога;
- текстовые artifacts не пустые;
- JSON artifacts валидны как JSON;
- markdown/txt artifacts содержат человекочитаемый текст;
- размер файлов в разумных пределах;
- нет лишней debug-директории вида `TO_DO_LIST/logs/PRD-046.1.2-debug/`.
```

### 3.5. Добавить tests

Минимальные тесты:

```text
bot_psychologist/tests/tools/test_validate_prd_artifact_encoding.py
```

Проверить:

```text
- validator ловит NUL bytes;
- validator ловит NUL characters;
- validator ловит invalid UTF-8;
- validator ловит invalid JSON;
- validator пропускает clean UTF-8 txt/json/md;
- validator обнаруживает unexpected debug dir;
- validator возвращает non-zero exit code при blockers.
```

Если в проекте нет `tests/tools`, можно использовать существующий стиль, но не смешивать эти тесты с multiagent logic tests.

## 7. Acceptance Criteria

HF1 считается выполненным, если:

```text
1. TO_DO_LIST/logs/PRD-046.1.2/test_command_output.txt читается как clean UTF-8.
2. В test_command_output.txt нет NUL bytes / NUL characters.
3. Artifact validator создан и покрыт тестами.
4. Validator проходит на TO_DO_LIST/logs/PRD-046.1.2.
5. Создан artifact_encoding_hygiene_report.json.
6. final_status = passed.
7. no_mutation_proof подтверждает отсутствие production/runtime/KB/config/index изменений.
8. Нет новых debug-директорий.
9. Implementation report и next recommendation созданы.
10. docs/PROJECT_STATE.md, docs/ROADMAP.md, docs/PRD_INDEX.md обновлены.
11. Commit и push выполнены.
```
