## 2. Главная цель PRD

Провести первый контролируемый runtime smoke Diagnostic Center / Prompt-Constraint path на одном allowlisted operator user и минимум двух normal-user controls, собрать evidence и доказать:

```text
1. apply работает только для allowlisted operator user;
2. normal users не затронуты;
3. rollback-first работает до и после apply;
4. hard-stop criteria корректно блокируют опасные состояния;
5. KB/internal lens boundaries не нарушены;
6. trace artifacts sanitized;
7. production data не мутирована;
8. результат можно использовать для следующего results/rollback/quality gate PRD.
```


### 6.2. Planned execution state

Execution state допускается только внутри runner/harness и только для synthetic allowlisted operator user:

```text
PROMPT_CONSTRAINT_PILOT_ENABLED=true
PROMPT_CONSTRAINT_PILOT_FORCE_DISABLED=false
PROMPT_CONSTRAINT_PILOT_MODE=test_apply
PROMPT_CONSTRAINT_PILOT_ALLOWED_USER_IDS=pilot_runtime_operator_001
```

Важно:

- не записывать эти значения в `.env`;
- не менять committed defaults;
- не сохранять real private config;
- после execution обязательно вернуть rollback state.
