# PRD-046.1.21-HF1 — BotDB Admin Chroma / Registry Integrity Repair Gate v1

## 0. Метаданные

- **PRD ID:** `PRD-046.1.21-HF1`
- **Название:** `BotDB Admin Chroma / Registry Integrity Repair Gate v1`
- **Тип:** emergency hotfix / live BotDB repair / Chroma persistent store recovery / registry resilience
- **Язык документа:** русский
- **Целевая ветка:** `main`
- **Репозиторий:** `Askhat-cmd/Text_transcription`
- **Локальный путь пользователя:** `C:\My_practice\Text_transcription`
- **Причина:** после успешного PRD-046.1.21 в живом запуске обнаружена деградация BotDB/Chroma: Dashboard показывает `ChromaDB 0 / unavailable`, `/api/query/` возвращает 503, `/api/registry/stats` возвращает HTTP 500, бот уходит в `semantic_fallback`.
- **Статус Diagnostic Center:** продолжение PRD-046.1.22 временно заблокировано до восстановления live BotDB/Chroma readiness.

---

## 1. Контекст и диагноз

Формально PRD-046.1.21 прошёл:

```text
final_status=passed
runtime_activation_mode=deterministic_runtime_harness
provider_called_by_execution_count=0
evidence_scope=deterministic_runtime_harness_only
```

Это означает: PRD-046.1.21 подтвердил controlled harness, rollback, normal-user no-effect и quality gate, но **не проверял живую BotDB/Chroma как обязательную runtime-зависимость**.

После ручного запуска пользователя обнаружены реальные live-симптомы:

```text
Dashboard: ChromaDB = 0, status = unavailable
Registry page: Ошибка загрузки реестра: HTTP 500
/api/query/: 503 Service Unavailable
Bot retriever: BotDB circuit open -> semantic fallback
```

В логах есть конкретная ошибка Chroma:

```text
[QUERY] Chroma query failed: object of type 'int' has no len()
POST /api/query/ -> 503 Service Unavailable
GET /api/registry/stats -> 500 Internal Server Error
TypeError: object of type 'int' has no len()
chromadb/segment/impl/metadata/sqlite.py -> _decode_seq_id
```

Текущее состояние нужно трактовать так:

```text
registry/all_blocks видят production source 247 blocks,
но live Chroma collection health/count/query ломается.
```

Пока это не восстановлено, продолжать Diagnostic Center runtime pilot нельзя, потому что система тестируется в аварийном retrieval fallback, а не в штатном BotDB/Chroma режиме.

---

## 2. Главная цель PRD

Восстановить live BotDB/Chroma/Admin readiness безопасным контролируемым способом:

```text
1. Зафиксировать live-problem evidence.
2. Сделать /api/registry/stats устойчивым к Chroma crash.
3. Диагностировать persistent Chroma store.
4. Сделать backup перед любым repair/reindex/cleanup.
5. Выполнить controlled focus-source Chroma recovery/reindex при подтверждённом Chroma failure.
6. Подтвердить: registry focus blocks = 247, all_blocks_merged = 247, Chroma count = 247.
7. Подтвердить: /api/query/ больше не возвращает 503 из-за ChromaDB unavailable.
8. Подтвердить: Dashboard и Registry browser-visible live smoke зелёные.
9. После восстановления Chroma подготовить или выполнить строго безопасную cleanup-политику zero-block test/archive sources.
10. Не мутировать KB text/governance authority.
```

Ключевая формула:

```text
Сначала чиним live knowledge runtime.
Потом возвращаемся к Diagnostic Center provider-backed pilot.
```

## 7. Исправить устойчивость `/api/registry/stats`

Проблема: сейчас `/api/registry/stats` вызывает:

```python
chroma_stats = runner.chroma_manager.get_stats()
```

без защитного `try/except`. Если Chroma падает, endpoint возвращает HTTP 500 и frontend пишет `Реестр не загружен`, хотя `/api/registry/` уже вернул строки.

Изменить:

```text
Bot_data_base/api/routes/registry.py
```

Требования:

```text
- /api/registry/stats не должен падать HTTP 500 из-за Chroma exception;
- при Chroma exception endpoint должен вернуть 200 с явным degraded state;
- chroma_total должен быть 0 или null по принятому контракту;
- warnings должен включать причину `chroma_stats_unavailable`;
- error detail должен быть sanitized;
- registry statistics должны возвращаться даже при Chroma unavailable;
- frontend не должен объявлять весь реестр незагруженным, если только stats endpoint degraded.
```

Предложенный contract:

```json
{
  "chroma_total": 0,
  "chroma_status": "unavailable",
  "chroma_error_code": "chroma_stats_unavailable",
  "warnings": ["Chroma stats unavailable; registry stats returned in degraded mode"]
}
```

Создать тесты:

```text
Bot_data_base/tests/test_registry_stats_chroma_resilience_hf1.py
```

Acceptance:

```text
registry_stats_chroma_exception_returns_200=true
registry_stats_contains_degraded_chroma_status=true
registry_stats_preserves_registry_counts=true
registry_stats_does_not_expose_raw_traceback=true
```
