# PRD-046.1.21-HF2 — BotDB Live Query Recovery Closure / Registry Zero-Block Cleanup Gate v1

## 0. Метаданные

- **PRD ID:** `PRD-046.1.21-HF2`
- **Название:** `BotDB Live Query Recovery Closure / Registry Zero-Block Cleanup Gate v1`
- **Тип:** hotfix follow-up / live recovery verification / query retrieval smoke / registry hygiene gate
- **Режим:** live BotDB verification + guarded cleanup only if all safety gates pass
- **Репозиторий:** `Askhat-cmd/Text_transcription`
- **Локальный путь:** `C:\My_practice\Text_transcription`
- **Предыдущий PRD:** `PRD-046.1.21-HF1`
- **Ожидаемый результат:** live BotDB/Chroma/Admin подтверждены: Dashboard Chroma `247 ok`, `/api/registry/stats` `200`, `/api/query/` `200`, бот не уходит в `semantic_fallback`. После этого разрешена только безопасная очистка zero-block test/archive sources с backup и защитой focus source.

---

## 1. Контекст

`PRD-046.1.21-HF1` был выполнен после live-проблемы BotDB/Chroma:

```text
Dashboard показывал ChromaDB 0 / unavailable.
Registry stats падал HTTP 500.
POST /api/query/ возвращал 503 ChromaDB unavailable.
Бот уходил в semantic fallback.
```

HF1 внёс важные исправления:

```text
- добавлен degraded-mode fallback для /api/registry/stats;
- добавлена диагностика persistent Chroma store;
- добавлены live audit/smoke tools;
- Chroma count после recovery дошёл до 247;
- focus source защищён;
- governance mutation не обнаружена.
```

Но GitHub implementation report по HF1 формально остался `failed`:

```text
final_status=failed
decision=botdb_chroma_registry_repair_blocked
blockers=['query_still_503_chromadb_unavailable', 'botdb_circuit_open_after']
query_http_200=False
registry_stats_http_200=True
chroma_count_after=247
```

При этом пользователь после перезапуска админки увидел:

```text
Dashboard:
ChromaDB = 247
status = ok
Блоков = 247
```

Это значит: состояние могло восстановиться после restart/reload, но GitHub-артефакты всё ещё фиксируют незакрытый blocker. Нельзя продолжать Diagnostic Center PRD, пока `/api/query/` и retrieval path не подтверждены live-артефактами.

## 2. Главная цель PRD

Закрыть расхождение между:

```text
локально сейчас выглядит "Chroma ok"
и
GitHub HF1 report говорит "query still 503 / circuit open".
```

Нужно получить live proof:

```text
1. Dashboard Chroma = 247 / ok.
2. Registry stats = 200 and chroma_total=247.
3. /api/query/ = 200 and returns chunks.
4. Bot runtime retrieval uses BotDB API, not semantic fallback.
5. BotDB circuit is closed after recovery.
6. Focus source 123__кузница_духа protected.
7. Registry zero-block cleanup allowed only for safe test/archive rows.
8. No KB text/governance mutation.
9. No Chroma reindex unless live query is still broken and controlled rebuild is explicitly required.
```
