## 2. Главная цель PRD

Не запускать новый runtime pilot, а выполнить post-execution consolidation:

```text
1. проверить, что PRD-046.1.20 evidence действительно достаточно;
2. подтвердить rollback до/после execution;
3. подтвердить normal-user no-effect;
4. подтвердить quality delta без regressions;
5. подтвердить safety/KB/trace boundaries;
6. отдельно оценить encoding warning;
7. принять решение: continue_limited_candidate / fix_required / stop_pilot;
8. подготовить следующий маленький PRD на основании фактов.
```

Ключевая формула:


### 5.1. Условия для `continue_limited_candidate`

Разрешено только если:

```text
source_gate_passed=true
execution_evidence_status=passed
rollback_evidence_status=passed
normal_user_no_effect_status=passed
quality_gate_decision=passed
safety_kb_boundary_status=passed
trace_sanitization_status=passed
artifact_hygiene_status=passed
no_mutation_status=passed
hard_stop_triggered=false
blockers=[]
```

Важно:

```text
continue_limited_candidate != broad rollout.
continue_limited_candidate != production ready.
continue_limited_candidate значит: можно готовить следующий ограниченный шаг.
```

### 5.2. Условия для `fix_required`

Если есть:

```text
quality warning that affects acceptance
encoding warning escalated by policy
trace warning requiring cleanup
minor documentation mismatch
non-critical harness limitation requiring calibration
```

### 5.3. Условия для `stop_pilot`

Если есть:

```text
normal_user_apply_count > 0
rollback failed
stale apply after force-disabled
KB boundary violation
safety regression
raw KB quote exposure
КУЗНИЦА authority citation
trace private leak
production mutation
provider error leaked to output
```

Если `fix_required` — рекомендовать hotfix/calibration PRD.
