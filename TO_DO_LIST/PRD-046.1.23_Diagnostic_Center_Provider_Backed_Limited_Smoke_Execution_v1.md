## 2. Главная цель PRD

Выполнить первый реальный provider-backed smoke Diagnostic Center под строгими ограничениями:

```text
target_user_count=1
allowed_user_ids=["pilot_runtime_operator_001"]
normal_user_control_count>=2
max_provider_calls<=5
broad_rollout_allowed=false
normal_user_apply_allowed=false
production_ready=false
```

Цель — проверить не широкую готовность продукта, а только:

```text
1. provider-backed path вообще работает;
2. Diagnostic Center constraints применяются только к allowlisted operator;
3. normal users не затрагиваются;
4. BotDB/Chroma остаются healthy;
5. provider output не ломает safety/KB boundaries;
6. rollback выключает apply без stale effects;
7. raw provider/private payload не попадает в committed artifacts.
```

---

## 3. Что этот PRD НЕ делает

Запрещено:

```text
- broad rollout;
- включение для normal users;
- расширение cohort больше 1;
- production-ready decision;
- изменение default feature flags;
- изменение Writer prompt/contract для normal users;
- изменение KB text/governance;
- Chroma reindex;
- загрузка новых источников в BotDB;
- коммит raw provider payload;
- коммит real private user ids;
- коммит .env/secrets/private logs.
```

Важно:

```text
Даже если PRD-046.1.23 passed,
Diagnostic Center всё ещё НЕ считается production-ready.
```
