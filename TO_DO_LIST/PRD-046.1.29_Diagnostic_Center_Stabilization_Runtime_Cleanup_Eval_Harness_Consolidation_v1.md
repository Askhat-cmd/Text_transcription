# PRD-046.1.29 — Diagnostic Center Stabilization / Runtime Cleanup / Eval Harness Consolidation v1

## 0. Краткий обзор

`PRD-046.1.29` закрывает этап Diagnostic Center после финальной runtime-governance приёмки `PRD-046.1.28`.

Цель PRD — не расширять Diagnostic Center и не запускать новый provider-backed smoke, а привести проект в устойчивое рабочее состояние:

```text
production runtime — понятный и компактный;
permanent quality/eval gates — сохранены и воспроизводимы;
historical evidence — доступна, но не мешает навигации;
temporary/debug artifacts — классифицированы и вынесены из активной зоны через archive/manifest-first подход;
living docs — сжаты до актуальной operational map.
```

Этот PRD является **stabilization / cleanup PRD**, обязательным после завершения provider-backed Diagnostic Center acceptance.

### 1.2. Почему cleanup нужен сейчас

Diagnostic Center строился через безопасную цепочку:

```text
shadow contracts
→ planner bridge
→ writer contract pilot
→ prompt replay
→ limited runtime gates
→ provider-backed limited smoke
→ controlled cohort expansion
→ final runtime governance acceptance
```

В процессе накопились:

- временные transition modules;
- одноразовые smoke runners;
- debug/trace samples;
- большие append-only docs;
- дублирующиеся roadmap/project-state записи;
- множество PRD evidence artifacts;
- permanent gates, которые нужно сохранить отдельно от временного шума.

Если не провести стабилизацию, проект станет сложнее сопровождать: production path, eval harness и historical PRD evidence начнут смешиваться.

## 2. Цели PRD

### 2.1. Главная цель

Стабилизировать проект после Diagnostic Center provider-backed acceptance и разделить активные файлы на понятные зоны:

```text
Production runtime
Permanent quality/eval/regression tooling
Historical PRD evidence archive
Temporary/debug cleanup candidates
```

### 2.2. Практические цели

IDE-агент должен:

1. Провести инвентаризацию Diagnostic Center runtime/eval/report/log artifacts.
2. Классифицировать файлы и модули по зонам.
3. Создать archive/cleanup manifest без опасного удаления.
4. Сжать living docs (`docs/PROJECT_STATE.md`, `docs/ROADMAP.md`, при необходимости `docs/PRD_INDEX.md`) до актуальной карты проекта, сохранив pre-cleanup snapshots.
5. Создать постоянную карту Diagnostic Center runtime и permanent gates.
6. Перепроверить permanent regression gates после cleanup/docs compaction.
7. Закрыть non-blocking artifact hygiene warning из `PRD-046.1.28`, если оно связано с воспроизводимым mojibake/replacement-character источником в command log generation.
8. Зафиксировать, какие cleanup действия выполнены сейчас, а какие требуют отдельного deletion PRD.

## 3. Non-goals

Этот PRD **не должен**:

```text
- запускать новый provider-backed smoke;
- расширять cohort;
- включать Diagnostic Center для normal users;
- объявлять production-ready;
- включать broad rollout;
- менять production defaults;
- удалять safety/eval/no-mutation/encoding gates;
- удалять runtime governance contracts;
- удалять исторические evidence artifacts, если это ломает воспроизводимость;
- менять KB governance authority поля: chunk_type / allowed_use / safety_flags;
- делать Chroma reindex;
- мутировать BotDB production data;
- коммитить secrets, .env, raw provider payload, raw private logs, sqlite/db/cache/.venv/node_modules.
```
