# PRD-046.1.33 — Diagnostic Center Limited Runtime Activation Readiness / Normal-User Boundary Decision Gate v1

## 0. Назначение PRD

Этот PRD продолжает проект **Bot Psychologist / Neo MindBot** после `PRD-046.1.32`.

`PRD-046.1.32` завершил controlled rollout results / rollback / quality gate в режиме no-new-execution и выдал решение:

```text
ready_for_limited_runtime_activation_readiness_prd
```

Текущий следующий шаг — не широкое включение Diagnostic Center, а отдельный readiness / boundary decision gate перед возможной ограниченной live-активацией в allowlisted режиме.

Цель этого PRD — создать проверяемый механизм, который ответит на вопрос:

```text
Можно ли безопасно готовить следующий PRD для ограниченной allowlisted live-активации Diagnostic Center,
без broad rollout, без normal-user activation и без production-ready declaration?
```

---

## 1. Главная цель

Создать deterministic readiness gate для перехода от controlled synthetic/internal rollout evidence к следующему безопасному этапу — **ограниченному живому тестированию**, но только как будущему отдельному PRD.

Итоговое решение PRD должно быть одним из трёх:

```text
ready_for_allowlisted_limited_live_activation_prd
readiness_blocked_fix_required
stop_runtime_activation_path
```

---

## 2. Высокоуровневый контекст

Проект уже прошёл длинную цепочку:

```text
Diagnostic Center shadow
→ Planner Bridge / Prompt-Constraint stack
→ offline replay / response-quality eval
→ controlled runtime pilot
→ provider-backed limited smoke
→ controlled cohort expansion
→ final runtime governance acceptance
→ stabilization / cleanup
→ controlled rollout planning
→ controlled rollout execution
→ controlled rollout results / rollback / quality gate
```

После `PRD-046.1.32` текущие границы остаются строгими:

```text
broad_rollout_allowed=false
production_ready=false
normal_user_activation_allowed=false
```

Diagnostic Center остаётся governed limited-runtime candidate, а не полноценной production authority.

---

## 3. Что этот PRD должен сделать

IDE-агент должен реализовать отдельный readiness / boundary gate, который проверяет:

1. source evidence после `PRD-046.1.32`;
2. live BotDB/Admin/API readiness;
3. Diagnostic Center runtime boundary;
4. normal-user no-effect boundary;
5. allowlisted-only activation policy для следующего PRD;
6. rollback-first и hard-stop готовность;
7. safety / KB boundary;
8. trace / provider sanitization boundary;
9. runtime defaults no-effect;
10. provider budget policy для будущего limited live step;
11. no-mutation proof;
12. artifact encoding hygiene;
13. docs consistency;
14. final readiness scorecard;
15. next PRD recommendation.

---

## 4. Non-goals / строгие запреты

Этот PRD **не должен**:

```text
- запускать broad rollout;
- включать Diagnostic Center для всех normal users;
- объявлять production-ready;
- выполнять реальную live-активацию для пользователей;
- выполнять новый provider-backed execution;
- делать новые provider calls;
- менять Writer final-answer path для normal users;
- менять Writer prompt/contract defaults;
- менять runtime defaults;
- мутировать KB governance authority:
  chunk_type / allowed_use / safety_flags;
- делать Chroma reindex;
- менять BotDB production data;
- применять новые enrichment/review overlays;
- коммитить raw provider payload;
- коммитить raw private logs;
- коммитить .env, secrets, .venv, node_modules, sqlite/db/cache.
```

Этот PRD — **readiness-only decision gate**, а не execution PRD.

---

## 5. Обязательные архитектурные инварианты

IDE-агент обязан сохранить следующие инварианты:

```text
Diagnostic Center is an internal map-making layer, not user-facing agent.
Writer is not the whole diagnostic system.
Context Assembly remains the source of bounded context.
KB is an internal lens library, not a quote source.
Deterministic governance is authority; LLM enrichment is advisory.
Raw history is preserved; summaries are additive.
Normal-user no-effect must remain proven.
Rollback-first and hard-stop policy must remain available before any future activation.
Provider payloads must be sanitized and never committed raw.
```

Diagnostic Center может быть проверен как readiness candidate, но не должен получить новую runtime authority в этом PRD.

## 11. Allowlisted limited live activation policy gate

Этот PRD должен подготовить policy contract для следующего PRD, но не выполнять activation.

Будущий limited live PRD может быть рекомендован только при наличии policy:

```text
allowlist_required=true
allowlist_scope=explicit_users_only
max_live_users_recommended<=3
activation_window_required=true
provider_budget_required=true
hard_stop_required=true
rollback_first_required=true
normal_user_controls_required=true
raw_payload_commit_forbidden=true
```

Рекомендуемый минимальный будущий scope:

```text
1–3 allowlisted real users
ограниченное число сценариев
короткое окно тестирования
обязательные normal-user controls
обязательный post-run no-new-execution results gate
```

Gate должен заблокировать readiness, если будущая policy допускает:

```text
unbounded users
implicit normal-user activation
broad rollout
production-ready declaration
provider calls without budget
activation without rollback
activation without hard-stop
```
