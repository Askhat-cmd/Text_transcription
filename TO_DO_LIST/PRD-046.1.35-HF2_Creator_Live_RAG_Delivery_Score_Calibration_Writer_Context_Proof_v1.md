# PRD-046.1.35-HF2 — Creator Live RAG Delivery / Score Calibration / Writer Context Proof v1

## 0. Назначение PRD

`PRD-046.1.35-HF2` продолжает `PRD-046.1.35-HF1`.

HF1 доказал важное:

```text
actual_live_turn_evidence_count = 1
botdb_chunks_returned = 4
BotDB HTTP status = 200
```

Но HF1 strict runner завершился как:

```text
final_status = blocked
decision = hf1_blocked_rag_delivery_failed
```

Причина:

```text
BotDB отдаёт чанки,
но они не доходят до Memory / Knowledge Policy / Context Assembly / Writer Prompt.
```

Цель HF2 — сделать точечный repair цепочки доставки знаний:

```text
BotDB query
→ bot_psychologist retriever
→ MemoryRetrievalAgent
→ Knowledge Policy
→ Context Assembly
→ WriterContract
→ Writer prompt
→ Web Trace
```

HF2 должен не расширять rollout, не включать normal users и не менять стратегическую архитектуру. Он должен аккуратно устранить конкретный delivery blocker и доказать, что Writer действительно получает governance-safe KB context.

## 7. Score filtering / calibration policy v1

Нужно ввести безопасную source-aware политику, а не просто глобально снизить `RAG_MIN_SCORE`.

### 7.1. Требования

В `MemoryRetrievalAgent` или рядом создать маленький deterministic helper:

```text
rag_score_policy_v1
```

Он должен принимать:

```text
raw_hits: list[SemanticHit]
rag_min_score: float
retrieval_source_used: api|semantic_fallback|tfidf_fallback|unknown
query: str
```

И возвращать:

```json
{
  "filtered_hits": [],
  "trace": {
    "raw_hits_count": 0,
    "raw_scores": [],
    "filtered_hits_count": 0,
    "filtered_by_score_count": 0,
    "salvaged_hits_count": 0,
    "score_policy_mode": "standard_threshold|source_aware_salvage|empty|governance_only",
    "reasons": []
  }
}
```

### 7.2. Standard threshold

Обычный путь должен остаться:

```text
hit.score >= RAG_MIN_SCORE
```

### 7.3. Source-aware salvage

Если:

```text
raw_hits_count > 0
filtered_hits_count == 0
all scores < RAG_MIN_SCORE
top score > 0
retrieval_source_used in api|semantic_fallback|tfidf_fallback|unknown
```

то разрешить conservative salvage:

```text
include top 1–3 hits
but mark them:
  score_policy_mode = source_aware_salvage
  low_score_salvage = true
  original_score preserved
```

Ограничения salvage:

```text
- salvage не должен обходить Knowledge Policy;
- salvage не должен bypass governance;
- salvage не должен отдавать do_not_use/internal_only Writer'у;
- salvage должен быть trace-visible;
- salvage должен быть отключаем через feature flag.
```

Feature flags:

```text
MEMORY_RAG_SCORE_POLICY_V1_ENABLED=true
MEMORY_RAG_LOW_SCORE_SALVAGE_ENABLED=true
MEMORY_RAG_LOW_SCORE_SALVAGE_MAX_HITS=3
MEMORY_RAG_LOW_SCORE_SALVAGE_MIN_TOP_SCORE=0.03
```

Если feature flag отсутствует, defaults должны быть safe для dev-pilot:

```text
enabled=true
max_hits=3
min_top_score=0.03
```

### 7.4. Почему это безопасно

Это не делает KB authority сильнее. Это только разрешает top-ranked chunks дойти до уже существующего deterministic governance filter.

Финальное решение о Writer context по-прежнему принимает:

```text
apply_knowledge_policy_v1
```

### 15.1. Passed: creator live RAG delivery ready

Разрешено только если:

```text
source_gate=passed
botdb_query_gate=passed
rag_raw_hits_count >= 1
rag_filtered_hits_count + rag_salvaged_hits_count >= 1
knowledge_policy_included_writer_count >= 1
context_assembly_knowledge_hits_count >= 1
writer_prompt_knowledge_hits_count >= 1
creator_live_turn_gate=passed
trace_alignment_gate=passed
normal_user_no_effect_gate=passed
trace_sanitization_gate=passed
provider_budget_gate=passed
no_mutation_proof=passed
artifact_encoding_hygiene=passed
broad_rollout_allowed=false
production_ready=false
normal_user_activation_allowed=false
```

Decision:

```text
hf2_passed_creator_live_rag_delivery_ready
```

Next PRD:

```text
PRD-046.1.36 - Creator Live Pilot Acceptance / Minimal Admin Runtime Controls v1
```

### 15.2. Passed but governance blocks writer context

Если:

```text
rag_raw_hits_count >= 1
rag_filtered_hits_count + rag_salvaged_hits_count >= 1
knowledge_policy_input_hits_count >= 1
knowledge_policy_included_writer_count = 0
knowledge_policy_dropped_count/internal_only_count > 0
all safety/no-mutation gates passed
```

Decision:

```text
hf2_passed_rag_available_but_governance_blocks_writer_context
```

Next PRD:

```text
PRD-046.1.36-KB - Governance Readiness for Writer Context v1
```

Это не runtime blocker. Это значит, что retrieval работает, но KB governance не разрешает Writer context.

### 15.3. Blocked: score filtering unresolved

Если:

```text
botdb_chunks_returned >= 1
rag_raw_hits_count >= 1
rag_filtered_hits_count = 0
rag_salvaged_hits_count = 0
```

Decision:

