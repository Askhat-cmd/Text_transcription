# PRD-046.1.35-HF3 — Creator Live RAG Evidence Sync / Writer KB Context Boundary Audit v1

## 0. Назначение PRD

`PRD-046.1.35-HF3` продолжает `PRD-046.1.35-HF2`.

HF2 дал противоречивую картину:

```text
1. Кодовая часть RAG delivery repair появилась.
2. In-process memory-agent probe показал:
   rag_raw_hits_count=4
   rag_salvaged_hits_count=3
   knowledge_policy_included_writer_count=3

   Memory Agent: semantic_hits=3
   has_relevant_knowledge=true

   Writer prompt содержит KNOWLEDGE RAG HITS
4. Но HF2 strict scorecard всё ещё зафиксировал:
   final_status=evidence_incomplete

   rag_raw_hits_count=0
```

```text

```

## 9. Writer KB truncation boundary audit

HF3 должен отдельно проверить текущую проблему пользователя:

```text
Writer получает KB, но KB-фрагменты обрезаны.
```

Нужно создать artifact:

```text
TO_DO_LIST/logs/PRD-046.1.35-HF3/writer_kb_truncation_audit.json
```

Минимальная структура:

```json
{
  "schema_version": "writer_kb_truncation_audit_v1",
  "prd_id": "PRD-046.1.35-HF3",
  "truncation_detected": true,
  "truncation_sources": [
    {
      "path": "bot_psychologist/bot_agent/multiagent/knowledge_policy.py",
      "mechanism": "_POLICY_SANITIZED_MAX_CHARS",
      "current_limit": 240,
      "is_safety_boundary": true
    },
    {
      "path": "bot_psychologist/bot_agent/multiagent/agents/writer_agent.py",
      "mechanism": "_format_hits h[:300]",
      "current_limit": 300,
      "is_safety_boundary": false
    },
    {
      "path": "bot_psychologist/bot_agent/multiagent/agents/writer_agent.py",
      "mechanism": "conversation_context[:2000]",
      "current_limit": 2000,
      "is_safety_boundary": false
    }
  ],
  "hard_cut_mid_word_count": 0,
  "hard_cut_mid_sentence_count": 0,
  "duplicate_kb_payload_detected": true,
  "llm_enrichment_available": true,
  "recommendation": "defer_full_kb_context_v2_after_creator_live_acceptance"
}
```

### 9.1. Что считать blocker

Blocker только если:

```text
- KB snippet обрезается так, что Writer получает бессмысленный/опасный фрагмент;
- raw private/full KB content leaking into unsafe debug;
- hard-cut ломает UTF-8/слова массово;
- Writer не может использовать KB из-за обрезки вообще.
```

### 9.2. Что НЕ считать blocker

Не считать blocker для Diagnostic Center creator-pilot:

```text
- фрагменты короткие, но смысловые;
- Writer получает 2–3 safe snippets;
- ответ использует KB как внутреннюю линзу;
- проблема скорее про качество/богатство KB context, а не про работоспособность runtime.
```

```text

```

То есть основа для умной KB-подачи есть, но отдельная задача остаётся:

```text
перестроить Writer KB payload так, чтобы он получал не обрезанный raw fragment,
а компактный structured payload:
summary
use_when
avoid_when
lens_family
practice_candidate

source metadata
policy reasons
```

Это отдельный PRD после creator-live acceptance, а не blocker для HF3.
