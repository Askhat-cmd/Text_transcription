# PRD-046.1.35 — Diagnostic Center Creator Live Results / Rollback / Quality Gate v1

## 0. Краткое назначение

`PRD-046.1.35` — это **no-new-execution results gate** после `PRD-046.1.34`.

Задача PRD: не запускать новый live rollout и не расширять аудиторию, а внимательно разобрать уже созданные артефакты `PRD-046.1.34`, проверить качество доказательств, rollback/safety/trace/no-mutation, подтвердить или заблокировать дальнейшее движение Diagnostic Center.

Главный вопрос:

```text
Достаточно ли evidence из PRD-046.1.34, чтобы продолжать creator-only limited live track,
или нужно остановиться и сделать hotfix/evidence-repair перед следующими живыми шагами?
```

## 3. Цель PRD

Создать results/rollback/quality gate для `PRD-046.1.34`, который:

1. Собирает и проверяет evidence из `PRD-046.1.34`.
2. Проверяет качество доказательств live activation.
3. Отдельно классифицирует evidence как:
   - `actual_live_turn_evidence`;
   - `runtime_probe_evidence`;
   - `simulated_gate_evidence`;
   - `missing_evidence`.
4. Проверяет rollback/hard-stop.
5. Проверяет normal-user no-effect.
6. Проверяет provider budget.
7. Проверяет trace sanitization и отсутствие raw payload/private logs.
8. Проверяет no-mutation proof.
9. Проверяет docs consistency.
10. Выдаёт одно из решений:
    - `continue_creator_limited_observation`;
    - `evidence_incomplete_hotfix_required`;
    - `rollback_required`;
    - `blocked_fix_required`.

## 6. Главный новый слой: evidence strength audit

Нужно добавить явный audit поверх результатов `PRD-046.1.34`.

### 6.1. Evidence classes

Для каждого ключевого proof-фрагмента определить:

```json
{
  "evidence_class": "actual_live_turn_evidence | runtime_probe_evidence | simulated_gate_evidence | missing_evidence",
  "evidence_strength": "strong | medium | weak | missing",
  "reason": "...",
  "source_files": ["..."]
}
```

### 6.2. Критерии классов

#### actual_live_turn_evidence

Можно ставить только если есть evidence, что реально был выполнен пользовательский turn:

```text
- есть sanitized trace конкретного turn;
- есть turn_id / session_id / timestamp;
- есть indication input -> answer_received;
- есть diagnostic_center mode creator_only;
- есть writer/validator/trace fields;
- raw provider payload не сохранён;
- user text либо отсутствует, либо sanitized/hashed/preview-safe.
```

#### runtime_probe_evidence

Можно ставить, если была проверена доступность runtime/admin/web-ui, но нет доказательства реального turn:

```text
- endpoint reachable;
- admin status ok;
- web UI reachable;
- dashboard/registry ok;
- но нет turn trace / answer artifact.
```

#### simulated_gate_evidence

Ставить, если значение было получено детерминированно из gate logic без реального runtime action:

```text
- message_sent=true потому что runtime_reachable=true;
- answer_received=true потому что runtime_reachable=true;
- provider_calls inferred, not measured;
- no actual request/response artifact.
```

#### missing_evidence

Ставить, если файл отсутствует, JSON не парсится, поле пустое или нет источника.

### 8.2. Decision rules

#### continue_creator_limited_observation

Разрешено только если:

```text
source_artifacts_gate=passed
rollback_quality_gate=passed
normal_user_boundary_gate=passed
trace_sanitization_gate=passed
provider_budget_gate=passed
no_mutation_proof=passed
docs_consistency_gate=passed
artifact_encoding_hygiene=passed
actual_live_turn_evidence_count >= 1 OR evidence_strength_gate=warning_not_blocking_with_explicit_note
broad_rollout_allowed=false
production_ready=false
normal_user_activation_allowed=false
```

Важно: если actual live turn evidence отсутствует, но все safety gates зелёные, решение может быть не `continue_creator_limited_observation`, а:

```text
evidence_incomplete_hotfix_required
```

Архитектурно предпочтительно не продолжать к следующему живому шагу, пока не доказан реальный sanitized live turn.

#### evidence_incomplete_hotfix_required

Ставить, если:

```text
safety gates passed
rollback/no-mutation passed
но live evidence weak/missing/simulated
```

Рекомендуемый следующий PRD:

```text
PRD-046.1.35-HF1 - Creator Live Evidence Capture Repair / Real Sanitized Turn Proof v1
```

#### rollback_required

Ставить, если:

```text
normal_user_no_effect failed
или trace sanitization failed
или rollback/hard-stop failed
или raw provider payload/private logs/secrets committed
```

#### blocked_fix_required

Ставить, если:

```text
source artifacts missing/unparseable
docs inconsistent
provider budget unknown/failed
no-mutation proof failed
```

```text
- TASK_LIST создан до реализации;
- source artifacts manifest создан;
- evidence strength audit создан;
- все required JSON logs созданы;
- все required Markdown reports созданы;
- tests запущены и вывод сохранён;
- strict runner запущен;
- no-mutation proof создан;
- artifact encoding hygiene выполнен;
- docs обновлены согласно decision;
- implementation report обновлён после push;
- commit и push в main выполнены;
- нет raw provider payload, private logs, secrets, .env, sqlite/db/cache/.venv/node_modules.
```

## 21. Важная архитектурная позиция

Если `PRD-046.1.35` обнаружит, что `PRD-046.1.34` был безопасным, но evidence живого turn недостаточно сильный — это **не провал проекта**.

Это правильная работа gate.

В таком случае следующий шаг:

```text
PRD-046.1.35-HF1 - Creator Live Evidence Capture Repair / Real Sanitized Turn Proof v1
```

Он должен будет не расширять rollout, а аккуратно получить один реальный sanitized creator-only turn proof.
