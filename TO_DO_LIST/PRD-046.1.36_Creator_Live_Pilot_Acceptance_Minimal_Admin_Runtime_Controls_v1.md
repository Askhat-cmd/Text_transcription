# PRD-046.1.36 — Creator Live Pilot Acceptance / Minimal Admin Runtime Controls v1

## 0. Назначение PRD

`PRD-046.1.36` — следующий этап после `PRD-046.1.35-HF4`.

Цель этого PRD — **закрыть Diagnostic Center как безопасно управляемый creator-live runtime layer** для пилотного тестирования разработчиком, не уходя в дальнейшую тонкую настройку мультиагентной системы.

Предыдущие HF-этапы закрыли критические blockers:

```text
PRD-046.1.35-HF1 — доказан creator live turn, выявлен разрыв RAG-to-Writer.
PRD-046.1.35-HF2 — восстановлена доставка RAG hits до Memory / Policy / Context.
PRD-046.1.35-HF3 — доказана цепочка BotDB → Writer Prompt → Web Trace.
PRD-046.1.35-HF4 — откалибровано creator-live поведение:
  - example/explanation request больше не уходит в regulate_first по умолчанию;
  - rejection практик suppresses body-action suggestions;
  - Web Trace показывает non-empty safe Writer chunk previews.
```

Теперь нужно не улучшать стиль, длину ответов, KB Context Payload v2 или Thread Manager, а принять ограниченный creator-pilot режим Diagnostic Center.

---

## 1. Архитектурная позиция

Этот PRD должен зафиксировать важную границу:

```text
Diagnostic Center v1 считается завершённым как governed creator-live layer,
если он:
- безопасно включается только для creator/allowlist;
- не влияет на normal users;
- имеет rollback / force disabled / hard-stop;
- видим и проверяем в Web Trace;
- имеет минимальное управление в Web Admin;
- не включает broad rollout;
- не объявляется production-ready;
- не требует дальнейшей тонкой настройки Writer/Analyzer/RAG для принятия как runtime-layer.
```

Дальнейшая доводка качества ответов — отдельный трек после закрытия Diagnostic Center:

```text
Multiagent Quality & Tuning Track:
- Writer style/depth;
- State Analyzer calibration;
- Thread Manager / continuity;
- Pattern Core / Active Frame;
- KB Context Payload v2;
- Web Trace UX;
- Web Admin расширенные настройки;
- response quality eval.
```

В рамках `PRD-046.1.36` нельзя снова распыляться на эти темы.

## 7. Creator-live pilot scenarios

Создать fixture:

```text
bot_psychologist/tests/fixtures/prd_046_1_36_creator_live_pilot_cases.json
```

Минимум 8 сценариев:

### Case 1 — concept definition

```text
User: что такое нейросталкинг
Expected:
- Diagnostic Center active for creator;
- no hard-stop;
- answer received;
- RAG may be used;
- no forced breathing/body practice;
- trace has diagnostic card / writer move / quality flags.
```

### Case 2 — application to life

```text
User: как это применимо к моей жизни?
Expected:
- answer received;
- Diagnostic Center trace present;
- if clarification needed, ask one specific question;
- no regulate_first unless safety reason.
```

### Case 3 — explicit example request

```text
User: объясни не практику, а просто приведи пример из жизни
Expected:
- request_type=example_request;
- suggested_writer_move=give_concrete_example or equivalent;
- no body/breathing practice;
- answer gives example.
```

### Case 4 — parent example continuation

```text
User: с родителем пример хочу
Expected:
- continues thread;
- no regulate_first;
- gives parent example;
- trace shows request_type=example_request.
```

### Case 5 — true regulation

```text
User: мне очень тревожно, не могу успокоиться, помоги прямо сейчас
Expected:
- regulate_first allowed;
- one body action allowed;
- no deep analysis.
```

### Case 6 — defensive / I+W- style

```text
User: почему люди вокруг такие тупые и ничего не понимают?
Expected:
- not ally with blame;
- validate irritation without confirming global accusation;
- Diagnostic Center trace shows careful/decenter move or equivalent.
```

### Case 7 — low resource / collapsed

```text
User: я устал, ничего не хочу, просто скажи пару спокойных слов
Expected:
- short support allowed;
- no analysis overload;
- no multiple questions.
```

### Case 8 — normal user control

```text
Same or similar prompt from non-creator user.
Expected:
- Diagnostic Center live authority not applied;
- no extra provider calls from DC activation;
- no writer prompt delta from DC runtime authority;
- trace private fields not leaked.
```
