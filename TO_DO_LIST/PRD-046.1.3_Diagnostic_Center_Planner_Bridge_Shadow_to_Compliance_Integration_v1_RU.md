## 2. Почему нужен этот PRD

Сейчас Writer Move Compliance строит инструкции из `DiagnosticCard`:

```text
DiagnosticCard
→ build_writer_move_instructions_v1()
→ writer_move_instructions
→ Writer prompt context
→ quality_trace.writer_move_compliance
```

А Planner Bridge уже умеет формировать candidate constraints:

```text
DiagnosticCenterOutput + divergence
→ PlannerBridgeOutput
→ response_goal_candidate
→ response_mode_candidate
→ depth_limit
→ max_questions
→ max_concepts
→ must_do_candidates
→ must_not_do_candidates
→ safety_constraints
→ kb_constraints
```

Но пока эти два слоя не сравниваются в runtime-quality trace.

Из-за этого система ещё не знает:

```text
совпадает ли Planner Bridge с текущим Writer Move Compliance;
где он усиливает ограничения;
где конфликтует;
где должен быть заблокирован;
какие candidate constraints потенциально полезны для будущего WriterContract pilot.
```

Этот PRD должен создать **compare-mode integration**:

```text
existing Writer Move Compliance instructions
vs
Planner Bridge candidate constraints
→ compatibility report
→ compliance shadow trace
→ eval scorecard
```

Но без применения к Writer.

### 3.4. Создать compatibility taxonomy

Минимальные статусы:

```text
compatible
tightens_constraints
expected_divergence
needs_review
blocked
```

Смысл:

#### compatible

Planner Bridge candidate совпадает с Writer Move Compliance и не требует изменений.

#### tightens_constraints

Planner Bridge предлагает более строгие ограничения, но безопасные:

```text
max_questions меньше или равно;
depth_limit ниже;
добавлены must_not_do;
сохранены safety/kb boundaries.
```

Это не blocker.

#### expected_divergence

Расхождение ожидаемо:

```text
Diagnostic Card даёт мягкий writer move,
а Diagnostic Center видит safety / low-resource / branch continuity / KB-lens boundary.
```

Это не blocker, но должно попадать в trace.

#### needs_review

Есть сомнительное расхождение:

```text
phase mismatch;
relation mismatch;
low confidence;
pattern_core missing;
writer move and bridge goal расходятся без safety причины.
```

Это не blocker для shadow режима, но учитывается в scorecard.

#### blocked

Нельзя даже рассматривать candidate constraints как будущую основу:

```text
safety mismatch;
unsafe depth in safety context;
raw KB text exposed;
must_not_quote_source=false;
apply_to_writer=true;
writer_prompt_changed=true;
final_answer_changed=true;
hard divergence blocker.
```
