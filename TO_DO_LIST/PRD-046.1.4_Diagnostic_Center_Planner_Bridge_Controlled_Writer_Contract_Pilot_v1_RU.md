## 0. Краткое резюме для IDE-агента

`PRD-046.1.3` подтвердил, что Planner Bridge умеет сравниваться с текущим `Writer Move Compliance` в режиме `shadow_compare_only`.

Следующий шаг — сделать **контролируемый Writer-Contract Pilot**, но всё ещё **без влияния на финальный пользовательский ответ**.

В этом PRD нужно создать слой, который строит **candidate overlay для WriterContract**, но не применяет его к настоящему Writer path.

Главная формула:

```text
Собрать pilot writer-contract constraints
→ сравнить с текущим WriterContract / prompt context
→ доказать, что основной WriterContract не изменился
→ доказать, что Writer prompt и final answer не изменились
→ сохранить trace/eval
```

То есть это **не включение Diagnostic Center в Writer**, а безопасная подготовка к будущему включению через проверяемый контракт.

## 3. Архитектурный смысл

Сейчас система подошла к границе, где легко ошибиться.

Нельзя резко сделать так:

```text
Planner Bridge → WriterContract → Writer → final answer
```

Правильный шаг сейчас:

```text
Planner Bridge → Pilot Overlay → Compare/Audit → Trace
```

То есть мы не меняем боевой контракт Writer, а строим рядом **пилотный слой**, который можно измерить.

### 4.2. Создать builder слоя Pilot

Добавить:

```text
bot_psychologist/bot_agent/multiagent/planner_bridge_writer_contract_pilot.py
```

Builder должен:

```text
1. принять текущий WriterContract;
2. принять planner_bridge_compliance_shadow;
3. извлечь candidate constraints;
4. построить pilot overlay;
5. сравнить overlay с текущим prompt-context WriterContract;
6. вернуть result в trace-only формате;
7. доказать, что исходный WriterContract не мутировал.
```

Критически важно:

```text
Нельзя изменять сам WriterContract object.
Нельзя подменять diagnostic_card.
Нельзя менять thread_state.
Нельзя менять memory_bundle.
Нельзя менять context_package.
Нельзя менять writer prompt.
```

### 4.3. Добавить runtime trace-only insertion

