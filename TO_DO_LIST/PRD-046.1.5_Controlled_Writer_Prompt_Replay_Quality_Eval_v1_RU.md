# PRD-046.1.5 — Controlled Writer Prompt Replay / Quality Eval v1

## 0. Метаданные

- **Проект:** Bot Psychologist / Neo MindBot
- **PRD:** `PRD-046.1.5`
- **Название:** Controlled Writer Prompt Replay / Quality Eval v1
- **Язык PRD:** русский
- **Статус:** ready for IDE-agent implementation
- **Предыдущий этап:** `PRD-046.1.4 — Diagnostic Center Planner Bridge Controlled Writer-Contract Pilot v1`
- **Следующий ожидаемый этап:** `PRD-046.1.6 — Controlled Prompt-Constraint Pilot / Limited Runtime Flag v1` *(только если этот PRD пройдет без blocker)*

## 1. Контекст

После `PRD-046.1.4` в проекте появился controlled Writer-Contract pilot overlay:

- Diagnostic Center остается `trace-only`;
- Planner Bridge остается `candidate-only`;
- `planner_bridge_compliance_shadow` остается `shadow_compare_only`;
- `planner_bridge_writer_contract_pilot` работает как `pilot_shadow_only`;
- overlay показывает, какие ограничения можно было бы добавить в WriterContract, но не применяет их;
- WriterContract, Writer prompt, Validator path и final answer не меняются.

По последнему audit:

- `cases_passed=36/36`;
- `hard_blocker_count=0`;
- `safety_pilot_pass_rate=1.0`;
- `kb_boundary_violation_count=0`;
- `raw_kb_text_exposure_count=0`;
- `writer_contract_changed_by_pilot_count=0`;
- `writer_prompt_changed_by_pilot_count=0`;
- `final_answer_changed_by_pilot_count=0`;
- `pilot_apply_to_writer_contract_count=0`;
- `pilot_apply_to_writer_prompt_count=0`;
- `pilot_apply_to_final_answer_count=0`;
- `runtime_smoke_ok=true`;
- `artifact_encoding_hygiene_passed=true`.

Теперь нужен следующий безопасный шаг: **не включать overlay в реальный Writer**, а сделать offline replay / A-B quality eval prompt-context.

Иными словами, нужно сравнить:

```text
A. Baseline Writer prompt-context
   — то, что Writer получает сейчас через WriterContract.to_prompt_context()

B. Pilot candidate prompt-context
   — копия/симуляция prompt-context, куда аккуратно добавлены candidate constraints из pilot overlay
   — но только в offline eval, не в runtime Writer path
```

Цель — понять, улучшает ли pilot overlay структуру prompt-context и не создает ли новых рисков.

---

## 2. Главная цель PRD

Создать **offline Writer Prompt Replay / Quality Eval v1**, который:

1. Берет baseline WriterContract / prompt-context.
2. Берет `planner_bridge_writer_contract_pilot` overlay.
3. Строит offline candidate prompt-context.
4. Сравнивает baseline vs candidate по deterministic quality rules.
5. Проверяет safety, KB-boundaries, конфликт ограничений, глубину, вопросы, must_do/must_not, prompt bloat.
6. Создает scorecard и trace artifacts.
7. Не вызывает LLM provider.
8. Не меняет runtime WriterContract, Writer prompt, Validator path или final answer.


Но это пока только наблюдение.

Следующий слой должен ответить на вопрос:

```text

стал бы prompt-context лучше, безопаснее и точнее?

```

Важно: речь пока не про генерацию нового ответа пользователю, а только про оценку структуры prompt-context.

## 6. Новый builder

Создать файл:

```text
bot_psychologist/bot_agent/multiagent/writer_prompt_replay.py
```

### 6.1. Основная функция

```python
build_writer_prompt_replay_v1(
    *,
    writer_contract: WriterContract,
    writer_contract_pilot: dict | PlannerBridgeWriterContractPilotResult,
    diagnostic_card: DiagnosticCard,
    thread_state: ThreadState,
    state_snapshot: StateSnapshot,
) -> WriterPromptReplayResult
```

### 6.2. Что делает builder

1. Получает baseline context:

```python
baseline = writer_contract.to_prompt_context()
```

2. Получает overlay candidate constraints:

```python
overlay = writer_contract_pilot["overlay"]
candidate_constraints = overlay["candidate_constraints"]
```

3. Строит **offline candidate prompt-context** как копию baseline:

```python
candidate = deepcopy(baseline)
```

4. Добавляет в candidate отдельный namespace, не меняя baseline:

```python
candidate["pilot_writer_constraints"] = {
    "source": "planner_bridge_writer_contract_pilot",
    "activation_mode": "offline_replay_only",
    "response_goal": ...,
    "response_mode": ...,
    "depth_limit": ...,
    "max_questions": ...,
    "max_concepts": ...,
    "must_do": [...],
    "must_not_do": [...],
    "kb_usage_mode": ...,
    "must_not_quote_source": True,
}
```

5. НЕ записывает candidate обратно в WriterContract.
6. Считает hash baseline и candidate.
7. Сравнивает их.
8. Возвращает replay result.

## 7. Quality checks

### 7.1. Safety checks

Replay должен проверять:

```text
- safety/hyper/hypo cases не получают depth выше low;
- max_questions в safety/low-resource <= 0;
- must_not_do содержит запреты на deep analysis в low-resource/safety cases;
- response_mode не конфликтует с safety mode;
- activation_readiness из pilot overlay учитывается.
```

### 7.2. KB boundary checks

Проверить:

```text
- raw KB text не попадает в candidate prompt-context;
- forbidden fields отсутствуют:
  - raw_text
  - full_text
  - content_full
  - page_content
  - source_text
- kb_usage_mode только:
  - none
  - internal_lens_only
  - practice_candidate_only
- must_not_quote_source == True
```

### 7.3. Constraint conflict checks

Проверить:

```text
- одно и то же правило не находится одновременно в must_do и must_not_do;
- max_questions не увеличивается относительно baseline writer_move_instructions;
- candidate не ослабляет safety constraints;
- candidate не удаляет existing writer_move must_not_do;
- candidate не добавляет слишком много constraints.
```

### 7.4. Prompt bloat checks

Нужно ограничить раздувание prompt-context:

```text
- candidate additional fields count <= разумного лимита;
- candidate serialized size не должен превышать baseline более чем на заданный лимит;
- recommended max delta: <= 35% или <= 2500 chars, что меньше/разумнее;
- если превышено — warning или blocker в зависимости от масштаба.
```

### 7.5. Deterministic quality score

Добавить простой deterministic score:

```python
quality = {
    "safety_ok": bool,
    "kb_boundary_ok": bool,
    "constraint_conflict_ok": bool,
    "prompt_bloat_ok": bool,
    "non_mutating_ok": bool,
    "candidate_improves_constraints": bool,
    "quality_status": "passed" | "needs_review" | "blocked",
}
```

## 9. Runtime integration rule

В этом PRD допустима только **debug / trace insertion**, если это необходимо.

Разрешено:

```text
debug.writer_prompt_replay_shadow
```

Запрещено:

```text
writer_contract = replay_candidate
writer_agent.write(replay_candidate)
writer_user_prompt = candidate_prompt
final_answer = candidate_answer
```

Если добавляется runtime trace, он должен быть построен **после создания реального WriterContract** и **до Writer**, но не должен изменять объект WriterContract.

Обязательно использовать hash proof:

```python
writer_contract_hash_before_replay
writer_contract_hash_after_replay
writer_contract_changed_by_replay == False
```
