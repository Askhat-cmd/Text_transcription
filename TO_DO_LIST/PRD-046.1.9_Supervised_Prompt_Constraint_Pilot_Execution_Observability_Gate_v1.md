# PRD-046.1.9 — Supervised Prompt-Constraint Pilot Execution / Observability Gate v1

## 0. Краткое название

**Supervised Prompt-Constraint Pilot Execution / Observability Gate v1**

## 1. Контекст

После `PRD-046.1.8` проект получил plan-first supervised rollout readiness layer для Prompt-Constraint Pilot:

```text
final_status=passed
decision=ready_for_supervised_execution_prd
enabled_default_false=true
force_disabled_default_true=true
normal_users_allowed=false
allowlist_required=true
test_prefix_required=true
max_initial_cohort_size=3
rollback_first_policy_preserved=true
toggle_matrix_ready=true
production_apply_performed=false
provider_called_by_plan=false
```

Это означает, что можно выполнить **один контролируемый supervised execution cycle** на минимальном allowlisted/test cohort, но всё ещё нельзя делать broad rollout.

Текущий baseline остаётся:

```text
PROMPT_CONSTRAINT_PILOT_ENABLED=false
PROMPT_CONSTRAINT_PILOT_FORCE_DISABLED=true
PROMPT_CONSTRAINT_PILOT_MODE=shadow|test_apply
```

В runtime-архитектуре Diagnostic Center всё ещё не является user-facing authority:

```text
Diagnostic Center v1: trace-only shadow layer
Planner Bridge: candidate-only / shadow-eval lineage
planner_bridge_compliance_shadow: trace-only compare layer
planner_bridge_writer_contract_pilot: pilot_shadow_only
writer_prompt_replay: offline_replay_only
prompt_constraint_pilot_runtime: default-off limited allowlisted test path
```

## 2. Цель PRD

Провести **один controlled supervised execution / observability gate** для Prompt-Constraint Pilot.

Смысл PRD:

```text
не расширить rollout,
а доказать, что supervised execution можно безопасно провести,
измерить,
откатить,
сравнить с baseline,
и принять решение о следующем шаге.
```

PRD должен создать execution harness, который:

1. Временно моделирует supervised `test_apply` run только для минимального test/allowlisted cohort.
2. Захватывает runtime trace samples.
3. Сравнивает baseline vs supervised `test_apply`.
4. Проверяет normal-user no-effect.
5. Проверяет rollback после supervised run.
6. Не вызывает provider.
7. Не мутирует production KB/registry/Chroma/config.
8. Возвращает финальное решение:

```text
stay_limited
hotfix_required
continue_supervised
stop
```

## 3. Главный принцип

```text
PRD-046.1.9 выполняет один supervised execution cycle в контролируемом harness,
но не делает broad rollout и не меняет default runtime flags.
```

После выполнения PRD обычный пользовательский путь должен остаться неизменным:

```text
normal_user_apply_count=0
default_off_user_path_effect_count=0
config_mutated=false
production_apply_performed=false
```

## 4. Non-goals

В рамках PRD запрещено:

1. Включать pilot для всех пользователей.
2. Менять default flags в коде или config.
3. Делать `PROMPT_CONSTRAINT_PILOT_ENABLED=true` новым baseline.
4. Делать `PROMPT_CONSTRAINT_PILOT_FORCE_DISABLED=false` новым baseline.
5. Запускать execution на реальных production-пользователях без явного allowlist/test-prefix.
6. Делать provider-dependent acceptance.
7. Мутировать `all_blocks_merged.json`, registry, Chroma, config.
8. Менять deterministic governance authority.
9. Давать Diagnostic Center authority над Writer без guards.
10. Удалять rollback-first policy.
11. Считать один успешный supervised run разрешением на broad rollout.
12. Включать автопереход к следующей фазе без отдельного PRD.

### 5.10. Decision rules

Финальное решение:

#### `continue_supervised`

Разрешено только если:

```text
source_plan_gate_passed=true
cohort_size<=3
test_apply_applied_count>0
normal_user_apply_count=0
rollback_failure_count=0
stale_apply_after_force_disabled_count=0
safety_regression_count=0
kb_policy_regression_count=0
prompt_bloat_regression_count=0
constraint_conflict_regression_count=0
raw_kb_text_exposure_count=0
provider_called_by_execution_count=0
production_mutation_detected=false
artifact_encoding_hygiene_passed=true
```

#### `stay_limited`

Использовать, если run безопасен, но evidence недостаточно для продолжения или test_apply фактически не применился.

#### `hotfix_required`

Использовать при non-critical regression/warning, который требует HF перед продолжением.

#### `stop`

Использовать при hard abort:

```text
normal_user_apply_count > 0
allowlist_violation_count > 0
rollback_failure_count > 0
stale_apply_after_force_disabled_count > 0
raw_kb_text_exposure_count > 0
safety_regression_count > 0
kb_policy_regression_count > 0
production_mutation_detected=true
```
