# PRD-047.13-HF1 — Complete Cleanup Closure / Legacy Split Removal / Docs Truth Finalization v1

## 0. Краткое резюме

`PRD-047.13-HF1` — это завершающий cleanup-hotfix после `PRD-047.13`.

`PRD-047.13` создал карту мусора и доказал, что runtime не был изменён, но не закрыл cleanup полностью:

```text
- empty artifacts остались;
- historical encoding issues остались;
- часть root docs всё ещё содержит противоречие PRD-047.13 / PRD-047.14;
- markdown summaries слишком краткие;
- legacy terms обнаружены, но не доведены до чистого active/current состояния;
- физическая архивная уборка почти не выполнялась;
- раздвоенность старых систем safe_guided / mvp_free_dialogue снята в документах частично, но должна быть окончательно доказана по фактическому состоянию проекта.
```

Цель HF1:

```text
Итеративно довести cleanup до закрытого состояния:
- active/current docs без противоречий;
- legacy split safe_guided / mvp_free_dialogue полностью удалён из активной логики, UI-лейблов и living docs как отдельные системы;
- historical evidence сохранён, но вынесен/помечен как historical;
- empty artifacts либо удалены, либо архивированы, либо заменены честным placeholder explanation;
- root docs и bot_psychologist/docs полностью соответствуют текущему состоянию;
- любые оставшиеся legacy terms имеют явную классификацию и не вводят в заблуждение;
- no-runtime-mutation proof и regression tests подтверждают, что бот не сломан.
```

Ключевой принцип:

```text
Cleanup должен быть доведён до состояния "не мешает работе и аудиту".
Не просто inventory, а закрытие активного шума.
```

Остаточные проблемы:

```text

2. 95 historical encoding issues остались.
3. root docs содержат противоречие:
   - PRD-047.13 уже cleanup;
   - но местами следующий quality polish ошибочно указан как PRD-047.13 вместо PRD-047.14.

   - нет отдельного runtime path;

   - нет UI label, который создаёт впечатление двух систем;
   - active docs объясняют это только как presets/aliases единой policy.
```

## 7. Legacy split removal / safe_guided vs mvp_free_dialogue

Главная задача HF1 — окончательно убрать активную раздвоенность старых систем.

### 7.1. Что считается активной раздвоенностью

Активная раздвоенность есть, если найдено что-то из списка:

```text
- safe_guided и mvp_free_dialogue ведут к разным runtime paths;
- active docs описывают их как две разные системы;
- Web Admin показывает их как независимые режимы без пояснения preset/alias;
- Writer prompt assembly имеет конкурирующие hard command branches, не описанные как compatibility;
- API/admin effective state не показывает unified_dialogue_policy_v2 как главный policy surface;
- ROADMAP/PROJECT_STATE/README противоречат друг другу;
- current docs говорят "mvp_free_dialogue" как отдельный режим вместо alias/preset;
- старые legacy labels видны пользователю/админу без contextual explanation.
```

### 7.2. Целевое состояние

К концу PRD должно быть верно:

```text
safe_guided = preset or compatibility alias
mvp_free_dialogue = preset or compatibility alias
free_dialogue_default = preset
unified_dialogue_policy_v2 = only active policy surface
multiagent_adapter = only active runtime path
Diagnostic Center = advisory-only
Writer = final answer author under hard safety boundaries
```
