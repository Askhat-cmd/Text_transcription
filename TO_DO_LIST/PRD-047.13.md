# PRD-047.13 — Project Cleanup / Legacy Noise Reduction / Docs Truth Sync / Admin Surface Inventory v1

## 0. Краткое резюме

`PRD-047.13` — это отдельный cleanup-PRD на уборку технического и исторического шума после стабилизации базовой диалоговой архитектуры `PRD-047.12` и `PRD-047.12-HF1`.

Этот PRD **не должен менять живую логику бота**.

Главная цель:

```text
Сделать проект понятнее, чище и безопаснее для дальнейшей полировки NEO:
- отделить актуальные документы от исторических;
- убрать/архивировать технический мусор;
- классифицировать legacy-следы;
- синхронизировать root docs и bot_psychologist/docs;
- проверить, что cleanup не изменил runtime-поведение;
- подготовить почву для PRD-047.14 Live Dialogue Quality Polish.
```

Cleanup нужен не ради косметики, а чтобы снизить архитектурный шум, который мешает:

```text
- IDE-агенту понимать, какие документы актуальны;
- архитектору быстро проводить GitHub-аудит;
- не путать исторические PRD-артефакты с живой системой;
- не цеплять старые safe_guided/mvp_free_dialogue/legacy fallback следы в новых PRD;
- не маскировать runtime-изменения огромными случайными diff.
```

Главный принцип:

```text
Inventory first.
Classify second.
Archive before delete.
No runtime mutation.
No broad cleanup without manifest.
```

## 1. Контекст

После `PRD-047.12-HF1` проект стал значительно устойчивее:

```text
- Unified Dialogue Policy v2 введён;
- safe_guided / mvp_free_dialogue / free_dialogue_default сведены к единой policy surface;
- Final Answer Acceptance Gate добавлен;
- Stale Stub Quarantine добавлен;
- Web Chat markdown технически подключён;
- bot_psychologist/docs начал заполняться актуальной документацией.
```

Но в проекте накоплен исторический слой:

```text
- старые PRD-файлы и отчёты;
- старые логи и screenshots;
- пустые / нулевые artifacts;
- дублирующие docs;
- битые или исторически повреждённые markdown/txt/json;
- устаревшие названия режимов;
- legacy fallback упоминания;
- historical Diagnostic Center statuses;
- старые scripts/runners;
- Web Admin sections, где смешаны active runtime, observability, dev-only и legacy;
- root docs и bot_psychologist/docs пока не полностью описывают одно и то же текущее состояние.
```

Отдельно: предыдущий audit показал, что часть HF1-артефактов в GitHub может быть добавлена как 0-byte/empty evidence files. Это не обязательно ломает runtime, но снижает доверие к evidence trail. PRD-047.13 должен проверить это системно и дать manifest.

## 2. Цель

Сделать безопасную уборку проекта без изменения поведения бота.

Цели:

1. Создать полный cleanup inventory.
2. Классифицировать файлы и UI-surfaces по статусам.
3. Синхронизировать root docs и `bot_psychologist/docs`.
4. Исправить активные docs, где есть устаревшие сведения.
5. Найти пустые/битые/лишние cleanup candidates.
6. Не удалять historical evidence без manifest.
7. Архивировать безопасные historical artifacts, если это снижает шум.
8. Обновить Web Admin surface inventory без runtime-изменений.
9. Проверить, что runtime-поведение не изменилось.
10. Подготовить итоговый отчёт: что актуально, что legacy, что archived, что dangerous-do-not-touch.

## 8. Encoding and corruption scan

Сканировать текстовые файлы:

```text
*.md
*.txt
*.json
*.py
*.ts
*.tsx
*.css
*.html
*.yml
*.yaml
```

Искать:

```text
- mojibake / исторически битые кириллические последовательности;
- replacement char U+FFFD;
- control characters, кроме допустимых \n \r \t;
- 0-byte JSON/MD/TXT artifacts;
- invalid JSON;
- markdown files with broken heading structure;
- duplicate BOM / odd BOM usage.
```

Запрещённые literal markers в scan logic:

```text
Рџ
Рґ
Рё
РЅ
СЃ
С‚
СЊ
РЎ
Рќ
�
\x07
\x0c
```

Правила исправления:

```text
- active docs: исправлять;
- active JSON reports: исправлять, если это не искажает evidence;
- historical logs: не переписывать без необходимости;
- historical corrupted artifacts: classify and optionally archive;
- runtime files: не менять, кроме если найден секрет/опасность — тогда остановиться и запросить отдельный security PRD.
```

## 9. Empty / zero-byte artifact policy


Для каждого указать:

```json

  "path": "...",

  "classification": "empty_artifact_candidate | historical_evidence_keep | unknown_do_not_touch",
  "safe_to_delete": false,
  "reason": "...",
  "referenced_by": ["..."],
  "action": "keep | archive | delete | replace_with_stub_explanation"

```

Важно:

```text

```

Пример:

```text
Этот artifact был создан runner'ом, но raw trace не сохранялся из-за privacy/sanitization policy.
```

Но это допустимо только если соответствует правде.
