# PRD-047.19-HF1 — Evidence Integrity / Regression Rerun / Artifact Encoding Repair v1

**Проект:** Bot Psychologist / Neo MindBot  
**Дата:** 2026-06-16  
**Статус:** Hotfix PRD для IDE-агента  
**Основание:** аудит PRD-047.19 после push обнаружил mismatch между implementation report и `test_command_output.txt`  
**Главная цель:** восстановить достоверность evidence: пересобрать тестовые логи PRD-047.19 с чистым temp-dir, добиться прохождения regression subsets PRD-047.18 / PRD-047.17 / PRD-047.19, убрать replacement-character warning в command output и обновить отчёты после push.

---

## 0. Краткое резюме

PRD-047.19 функционально сделал правильную вещь:

```text
curated overlay dry-run preflight
→ fixture-only overlay честно заблокирован
→ live apply = false
→ Chroma reindex = false
→ runtime visibility = false
→ no mutation proof = passed
```

Но acceptance evidence не полностью чистое.

В `TO_DO_LIST/logs/PRD-047.19/test_command_output.txt` сохранены failed regression runs:

```text
PRD-047.18 regression subset: 10 passed, 4 errors, exit_code=1
PRD-047.17 regression subset: 7 passed, 4 errors, exit_code=1
PRD-047.19 tests: 12 passed, exit_code=0
```

Причина ошибок похожа на environment/temp permission issue:

```text
PermissionError: [WinError 5]
C:\Users\video\AppData\Local\Temp\pytest-of-video
```

Дополнительно encoding hygiene зафиксировал:

```text
replacement_char_warning_count=1
TO_DO_LIST/logs/PRD-047.19/test_command_output.txt:replacement_chars=128
```

Поэтому PRD-047.19 нельзя считать полностью закрытым как clean evidence, даже если функциональная часть выглядит корректной.

HF1 должен сделать не новую функциональность, а **evidence repair**.

## 1. Контекст

### 1.1. Что принято по PRD-047.19

Допустимо сохранить как passed-with-expected-blockers:

```text
apply_preflight_report.status = passed_with_expected_blockers
ready_for_live_apply = false
expected_blockers = [no_real_human_reviewed_decisions, overlay_fixture_only]
```

Это правильно и не требует функционального отката.

### 1.2. Что не принято

Нельзя оставить как clean acceptance:

```text
implementation report говорит passed_with_expected_blockers,
но test_command_output содержит regression failures.
```

Нужно:

```text
- явно зафиксировать previous evidence mismatch;
- пересобрать тесты с контролируемым temp-dir;
- сохранить clean final command output;
- обновить implementation report;
- обновить encoding hygiene;
- сделать post-push metadata sync.
```

## 5. Test temp-dir repair

HF1 должен запускать pytest не через системный temp:

```text
C:\Users\video\AppData\Local\Temp\pytest-of-video
```

а через controlled temp внутри проекта:

```text
C:\My_practice\Text_transcription\TO_DO_LIST\logs\PRD-047.19-HF1\pytest_tmp
```

Можно использовать один или оба подхода:

### Вариант A — pytest `--basetemp`

```powershell
python -m pytest <tests> -q --basetemp TO_DO_LIST\logs\PRD-047.19-HF1\pytest_tmp\prd_047_18
```

### Вариант B — env vars

```powershell
$env:TEMP="C:\My_practice\Text_transcription\TO_DO_LIST\logs\PRD-047.19-HF1\tmp"
$env:TMP="C:\My_practice\Text_transcription\TO_DO_LIST\logs\PRD-047.19-HF1\tmp"
$env:PYTEST_ADDOPTS="--basetemp=TO_DO_LIST\logs\PRD-047.19-HF1\pytest_tmp"
```

Не использовать `.venv`, `node_modules`, DB/cache dirs как temp.

## 7. Encoding repair

`test_command_output.txt` должен быть записан в UTF-8 без replacement characters.

Требования:

```text
encoding_hygiene.final_status=passed
replacement_char_warning_count=0
mojibake_warning_count=0
utf8_decode_error_count=0
json_parse_error_count=0
```

Если Windows console даёт нечитаемый текст ошибки, использовать:

```powershell
$OutputEncoding = [Console]::OutputEncoding = [System.Text.UTF8Encoding]::new()
```

или писать через Python wrapper, который сохраняет stdout/stderr как UTF-8.
