# PRD-047.19 — Curated Candidate Dry-Run Apply Plan / Preflight over Accepted Overlay v1

**Проект:** Bot Psychologist / Neo MindBot  
**Дата:** 2026-06-16  
**Статус:** PRD для IDE-агента  
**Предыдущий этап:** PRD-047.18 — Manual Review / Curated Candidate Acceptance Workflow v1  
**Главная цель:** создать offline preflight/dry-run слой, который умеет принимать curated overlay preview, строить безопасный план будущего применения accepted fields к metadata, но не изменяет live metadata, processed blocks, Chroma, retrieval или Writer runtime.

---

## 0. Краткое резюме

PRD-047.18 создал ручной workflow проверки enrichment candidates:

```text
80 PRD-047.17 candidates
→ review queue 80/80
→ all-pending decision template
→ fixture-only review decisions
→ curated_overlay_preview fixture_only=true
→ accepted_item_count=2 / accepted_field_count=5 только в fixture
```

Следующий шаг — не применять metadata, а создать **dry-run apply plan / preflight**:

```text
curated_overlay_preview
→ intake validation
→ source/candidate consistency check
→ field mapping plan
→ collision/diff preview
→ apply readiness report
→ explicit blockers
→ no mutation proof
```

PRD-047.19 должен доказать, что система может безопасно построить план применения accepted overlay, но при текущем состоянии проекта обязана честно сказать:

```text
real_apply_ready=false
reason=overlay_is_fixture_only_or_no_real_human_accepted_decisions
```

Это нормальный accepted outcome для PRD-047.19.

### 1.3. Почему нужен PRD-047.19

Сейчас есть curated overlay preview, но нет слоя, который отвечает на вопросы:

```text
Что именно изменилось бы, если когда-нибудь применить accepted fields?
К каким source blocks это относится?
Какие поля будут затронуты?
Есть ли конфликт с текущей metadata?
Есть ли риск плохого source grounding?
Является ли overlay fixture-only?
Есть ли реальные human accepted decisions?
Можно ли вообще переходить к apply PRD?
```

Без preflight следующая итерация может ошибочно перепрыгнуть от review к mutation.

Правильная траектория:

```text
candidate_only
→ manual review
→ curated overlay preview
→ dry-run apply plan / preflight
→ human curated batch
→ eval over accepted overlay
→ explicit apply/reindex PRD later
```

## 2. Цель

Создать offline preflight toolchain для curated overlay:

1. Прочитать PRD-047.18 curated overlay preview.
2. Проверить, fixture-only это overlay или реальный human-reviewed overlay.
3. Проверить, что overlay ссылается на существующие PRD-047.17 candidates и source blocks.
4. Построить `dry_run_apply_plan`, который показывает потенциальные изменения как diff preview.
5. Проверить field mapping между overlay fields и `MechanismAwareChunkMetadata` / future metadata overlay.
6. Сформировать readiness report:

```text
ready_for_live_apply=false
ready_for_eval=false|true depending on overlay type
blockers/warnings
```

7. Доказать no mutation.
8. Подготовить следующий шаг: либо human curated review batch, либо eval over real accepted overlay.
