# PRD-047.25 — Overlay + Writer KB Payload Live Evidence / Evaluation v1

**Проект:** Bot Psychologist / Neo MindBot  
**Дата:** 2026-06-18  
**Статус:** PRD для IDE-агента  
**Предыдущий этап:** PRD-047.24 — Retrieval Query Assembly / Current-Turn Focus Repair v1  
**Тип PRD:** live evidence / evaluation / no-mutation  
**Главная цель:** на очищенном retrieval baseline проверить связку `overlay_shadow_trace` + `writer_kb_payload_v1` в живых Web Chat/API сценариях, не меняя поведение Writer и не давая overlay authority. Дополнительно проверить, не появились ли параллельные runtime-системы, и зафиксировать план их устранения, если они остаются как временные fallback/compatibility paths.

---

## 0. Краткое резюме

После PRD-047.24 подтверждено:

```text
- current_turn_focus_v1 стал primary path для retrieval query assembly в local/dev/pilot/test;
- standalone knowledge-вопросы больше не тащат предыдущий user query;
- duplicate fragments и mid-word truncation устранены;
- Q24-002: relevance high_exact;
- Q24-003: relevance medium_related, warning-grade but no longer missing_expected_source;
- Writer KB Payload остаётся canonical;
- Chroma / Bot_data_base / embeddings / chunking не мутировались.
```

Теперь можно вернуться к отложенному вопросу:

```text
Overlay shadow + Writer KB Payload действительно помогают live/eval качеству или пока только создают trace-шум?
```

PRD-047.25 не должен включать overlay в apply-mode. Это evidence PRD:

```text
overlay = trace-only / shadow
writer_kb_payload = canonical structured delivery
retrieval_query_build_trace = current_turn_focus_v1 proof
Writer = финальный автор ответа
```

```text
- overlay apply mode is active;
- overlay changes retrieval execution or Writer authority;

- tests fail;
- encoding hygiene fails;
```
