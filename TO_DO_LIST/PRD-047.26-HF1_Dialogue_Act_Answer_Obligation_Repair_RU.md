## 2. Цель

Сделать минимальный targeted repair, который:

1. Для прямых knowledge questions не оставляет `dialogue_act=unknown`.
2. Для прямых knowledge questions не оставляет `answer_obligation=continue_thread`.
3. Для explicit short practice request задаёт bounded practice obligation.
4. Для support/direct question about panic/control не пропускает textbook answer как качественный.
5. Не меняет Writer prompt.
6. Не меняет DB/Chroma/metadata.
7. Не создаёт нового агента.
8. Не меняет overlay mode.
9. Сохраняет Writer как единственного автора финального ответа.
10. После repair делает DB-track минимум `ready_with_warning`, если останется только overlay noise без user-facing влияния.

## 5. Runtime requirements

### 5.1. Изменения должны быть минимальными

Разрешённый scope:

```text
bot_psychologist/bot_agent/multiagent/final_answer_directive.py
bot_psychologist/bot_agent/multiagent/dialogue_policy.py
bot_psychologist/bot_agent/multiagent/final_answer_acceptance_gate.py
```

Если фактическая логика `dialogue_act_resolution` / `answer_obligation_resolution` находится в других файлах, можно изменить именно эти файлы, но обязательно указать в source gate и implementation report:

```text
- почему выбран этот файл;
- почему изменение минимальное;
- почему оно не создаёт новый runtime path.
```

### 5.2. Принцип ремонта

Не добавлять большой rule engine.

Допустимая логика:

```text
explicit user wording → high-confidence answer obligation override
```

Только для очевидных случаев:

1. Knowledge/direct explanation request:
   - `что такое ...`
   - `расскажи о ...`
   - `объясни ...`
   - `что значит ...`
   - `какие ...`
   - `почему ...`
   - явный `?`
   - explicit “без встречного вопроса”

2. Explicit practice request:
   - `дай одну короткую практику`
   - `дай практику`
   - `упражнение`
   - `микропрактика`
   - `чтобы заметить ...`

3. Support/direct explanation:
   - `когда накрывает паникой, почему ...`
   - `почему контроль становится сильнее`
   - формально support + direct question.

Нельзя делать:

```text
if "контроль" → depth_level=3
if "паника" → medical route
if "драйвер" → force practice
if "мама" → childhood route
```

То есть чинится форма обязательства ответа, а не психологическая интерпретация.

## 6. Required behavior

### 6.1. Q26-002 — five drivers

Input:

```text
Расскажи о пяти драйверах выживания.
```

Expected trace:

```json
{
  "dialogue_act": "knowledge_question",
  "answer_obligation": "answer_knowledge_question",
  "answer_shape": "structured_explanation",
  "question_policy": "optional_none"
}
```

Allowed variants:

```text
dialogue_act может быть direct_question, если answer_obligation=answer_knowledge_question
```

Final answer shape:

```text
- отвечает прямо;
- перечисляет 5 драйверов;
- не уходит в самореализацию;
- не спрашивает “что ты имеешь в виду?”;
- не dump raw KB;
- живой тон, не методичка.
```

### 6.2. Q26-004 — control as safety, no question

Input:

```text
Объясни коротко, что значит «контроль как безопасность», без встречного вопроса.
```

Expected trace:

```json
{
  "dialogue_act": "knowledge_question|direct_question",
  "answer_obligation": "answer_knowledge_question|answer_direct_question",
  "question_policy": "none|optional_none",
  "depth": "short|medium"
}
```

Final answer:

```text
- коротко объясняет;
- не задаёт встречный вопрос;
- не превращает в лекцию;
- не открывает детство/травму без запроса.
```

### 6.3. Q26-007 — one bounded practice

Input:

```text
Дай одну короткую практику, чтобы заметить драйвер «Будь сильным».
```

Expected trace:

```json
{
  "dialogue_act": "practice_request",
  "answer_obligation": "provide_one_bounded_practice",
  "answer_shape": "one_short_practice",
  "practice_policy": "allowed_explicit_request",
  "question_policy": "none|optional_followup"
}
```

Если существующие enum values не поддерживают такие строки, можно использовать ближайшие существующие, но report должен зафиксировать mapping:

```json
{
  "practice_request": "...",
  "provide_one_bounded_practice": "...",
  "one_short_practice": "..."
}
```

Final answer:

```text
- одна практика, не несколько;
- короткая;
- безопасная;
- связана с «Будь сильным»;
- не превращает практику в плётку самосовершенствования;
- не даёт длинную теорию;
- не просит уточнять перед тем, как дать практику.
```

### 6.4. Q26-011 — panic/control support direct question

Input:

```text
Когда накрывает паникой, почему контроль становится сильнее?
```

Expected trace:

```json
{
  "dialogue_act": "concrete_situation_question|direct_question|knowledge_question",
  "answer_obligation": "answer_concrete_situation|answer_direct_question|answer_knowledge_question",
  "answer_shape": "direct_supportive_explanation",
  "question_policy": "optional_none|optional_followup"
}
```

Final answer:

```text
- отвечает на “почему”;
- звучит по-человечески;
- не textbook;
- не диагностирует;
- не заменяет медицинскую помощь;
- не уходит в raw KB;
- при необходимости мягко держит safety boundary: если симптомы новые/сильные/непонятные — лучше медицинская оценка.
```

## 7. Acceptance gate / evaluator repair

PRD-047.26 показал `evaluator_false_pass_count=2`.

HF1 должен усилить evaluator/acceptance gate так, чтобы слабые ответы по Q26-007 и Q26-011 не проходили как healthy context.

### 7.1. Q26-007 false-pass должен ловиться

Если user explicitly asks:

```text
дай одну короткую практику
```

а answer:

```text
- не содержит практического шага;
- даёт несколько практик вместо одной;
- уходит в теорию;
- не упоминает ключевой объект запроса;
- задаёт уточнение вместо выполнения;
```

то acceptance gate должен дать:

```json
{
  "status": "failed|warning",
  "failed_checks": ["explicit_one_practice_request_not_fulfilled"],
  "answer_considered_real": false,
  "can_save_as_healthy_context": false
}
```

Допустимо начать как warning, если блокировка слишком рискованна, но evaluator_false_pass_count по Q26-007 должен стать 0 в triage report.

### 7.2. Q26-011 false-pass должен ловиться

Если support/direct question asks why panic makes control stronger, а answer:

```text
- слишком textbook;
- не отвечает на why;
- не содержит “контроль”/“паника”/“тело” или близкие смысловые anchors;
- уходит в общий совет без связи с вопросом;
```

то acceptance gate должен дать:

```json
{
  "status": "failed|warning",
  "failed_checks": ["support_direct_question_answer_too_textbook"],
  "can_save_as_healthy_context": false
}
```

Важно: не блокировать хороший короткий supportive answer только потому, что он не длинный.
