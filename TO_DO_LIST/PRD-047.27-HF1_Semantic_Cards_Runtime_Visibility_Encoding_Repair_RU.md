## 0. Краткое резюме простыми словами

`PRD-047.27` добавил pilot semantic cards pack и отчёты показали, что карточки выбирались в тестовом live pilot.

Но ручная проверка владельца проекта в реальном Web UI показала другое:

```text
- в раскрытом trace чата не видно semantic_cards_pilot;
- в Admin Config Panel нет статуса semantic cards pilot;
- в Runtime/Admin нет card pack / card count / feature flag;
- в Writer KB Payload видны обычные book/general_text chunks, но не semantic_card:*;
- в trace/admin есть битая кодировка: Рњ..., в–ј, вЂ— и т.п.
```

Это значит, что перед расширением DB-track нужно не улучшать ответы и не расширять БД, а сделать видимость и доказуемость runtime-интеграции.

Главная формула HF1:

```text
semantic cards не считаются live-интегрированными,
пока они не видны в Web UI trace и Admin Runtime.
```

## 4. Required behavior

### 4.1. Admin Runtime / Config visibility

В Admin UI должен появиться read-only статус semantic cards pilot.

Минимально в `Runtime`, `Overview` или отдельном блоке `Knowledge / Semantic Cards`:

```text
semantic_cards_pilot_enabled: true/false
semantic_cards_pilot_enabled_source: env|config|default|admin
semantic_cards_pack_id: semantic_cards_pilot_v1|none
semantic_cards_loaded_count: 12|0
semantic_cards_adapter_enabled: true/false
semantic_cards_writer_payload_enabled: true/false
semantic_cards_last_selected_count: N
semantic_cards_last_selected_ids: [...]
semantic_cards_authority: advisory_only
writer_can_ignore_cards: true
applied_as_authority: false
```

Если pilot выключен, Admin должен честно показывать:

```text
semantic_cards_pilot_enabled: false
reason: disabled_by_config
```

Если pack не найден:

```text
semantic_cards_pilot_enabled: true
semantic_cards_pack_id: semantic_cards_pilot_v1
semantic_cards_loaded_count: 0
status: pack_not_loaded
error: readable short reason
```

### 4.2. Web Chat trace visibility

В раскрытом trace каждого хода должен появиться отдельный блок:

```text
Semantic Cards Pilot | enabled|disabled | cards N
```

При включённом pilot:

```json
{
  "trace_version": "semantic_cards_pilot_trace_v1",
  "enabled": true,
  "pack_id": "semantic_cards_pilot_v1",
  "loaded_card_count": 12,
  "selected_card_count": 1,
  "selected_card_ids": ["program_imperfect_self_v1"],
  "selection_reason": "current_turn_overlap",
  "suppressed_reason": null,
  "writer_can_ignore": true,
  "applied_as_authority": false,
  "writer_payload_enriched": true
}
```

Если карточки не выбраны:

```json
{
  "enabled": true,
  "pack_id": "semantic_cards_pilot_v1",
  "loaded_card_count": 12,
  "selected_card_count": 0,
  "selected_card_ids": [],
  "suppressed_reason": "no_relevant_card|explicit_no_theory|greeting|no_kb_need",
  "writer_payload_enriched": false
}
```

Если pilot выключен:

```json
{
  "enabled": false,
  "pack_id": null,
  "loaded_card_count": 0,
  "selected_card_count": 0,
  "suppressed_reason": "disabled_by_config"
}
```

### 4.3. Writer KB Payload visibility

Когда карточка попала в Writer KB Payload, в trace должно быть видно не только `book/general_text`, но и semantic card item:

```text
chunk_id=semantic_card:program_imperfect_self_v1
source_doc=semantic_cards_pilot_v1
chunk_type=concept
quote_policy=paraphrase_only
allowed_use=direct_to_writer
writer_can_ignore=true
applied_as_authority=false
core_thesis=...
writer_instruction=...
```

Если adapter превращает card в обычный payload item, он всё равно обязан сохранять marker:

```text
payload_item_origin=semantic_card
semantic_card_id=program_imperfect_self_v1
semantic_card_pack_id=semantic_cards_pilot_v1
```

### 4.4. Runtime API contract

Debug/API response должен явно включать semantic cards pilot status.

Проверить и при необходимости расширить:

```text
/api/debug/trace
/api/admin/runtime
/api/admin/config
/api/chat/debug
или актуальные существующие endpoints проекта
```

Не создавать новый runtime path, если существующие debug/admin endpoints уже есть.

### 4.5. Encoding repair

Исправить mojibake в Web UI/Admin/Trace.

Проблемные примеры, которые должны исчезнуть:

```text
РњСѓР»СЊС‚РёР°РіРµРЅС‚РЅС‹Р№
РџР°Р№РїР»Р°Р№РЅ
в–ј
в–¶
вЂ”
```

Должно отображаться:

```text
Мультиагентный пайплайн
Контекст памяти
Полотно LLM
Токены и стоимость
—
▾
▸
```

Проверить цепочку:

```text
backend JSON serialization
response headers charset=utf-8
frontend fetch/json handling
trace export text handling
log/file read/write encoding
React/Vue/Svelte rendering layer, если применимо
```

Требования:

```text
- JSON serialize with UTF-8 safe behavior;
- не делать double-decode;
- не кодировать уже готовый Unicode повторно;
- все trace/admin strings должны проходить как Unicode strings;
- exported .txt/.json/.md trace artifacts должны быть UTF-8;
- no replacement characters unless source content truly contains them.
```

## 9. Acceptance criteria

HF1 считается `passed`, если:

```text
- source_gate passed;
- semantic cards pilot visible in Admin Runtime/Config;
- semantic cards pilot visible in Web Chat trace;
- selected_card_ids visible on expected KB questions;
- disabled/suppressed reason visible when no cards selected;
- Writer KB Payload exposes semantic_card origin/id when cards enrich payload;
- semantic_cards_loaded_count = 12 or documented expected count;
- writer_can_ignore=true visible;
- applied_as_authority=false visible;
- mojibake fixed in trace/admin labels;
- tests passed;
- no DB schema change;
- no Chroma reindex;
- no Writer prompt rewrite;
- no retrieval ranking rewrite;
- no overlay apply;
- no new runtime path;
- no new LLM-agent.
```

`passed_with_warning`, если:

```text
- backend/API trace visible but Web UI visual block is incomplete;
- Admin shows status but not all details;
- screenshots unavailable but JSON proof is complete;
- minor non-blocking UI label remains untranslated but not mojibake;
- semantic cards visible only when explicit env flag enabled and report documents how to enable it.
```

`blocked`, если:

```text
- semantic_cards_pilot still absent from real Web UI trace;
- Admin Runtime still has no semantic cards status;
- selected_card_ids cannot be observed in live path;
- mojibake remains in core trace/admin labels;
- cards become authoritative instead of advisory;
- feature flag default is changed to on globally without explicit local/dev guard;
- DB/Chroma/registry/processed_blocks mutated;
- new runtime path or new LLM-agent added;
- tests fail.
```
