## 1. Что именно надо исправить

### 1.1. No-practice blocker

Для запроса вида:

```text
Только не давай мне опять практику. Просто объясни, почему я так реагирую.
```

бот должен:

```text
- объяснить механизм;
- не давать упражнение;
- не давать домашнее задание;
- не давать императивный микро-шаг;
- не маскировать практику под "один следующий шаг".
```

Разрешено:

```text
- объяснение;
- отражение;
- пример;
- один мягкий вопрос без практики.
```

### 1.2. Benign acceptance over-quarantine

Из PRD-047.36 известно, что benign turns могут иметь:

```text
writer_raw_vs_api_match=true
api_vs_memory_match=false
acceptance_gate_status=failed
must_quarantine_answer=true
```

Затронутые классы:

```text
S1 greeting / purpose
S2 ordinary support
S7 explicit practice request
S8 no-practice request
S12 thanks / close
```

После HF1:

```text
- нормальный безопасный ответ не должен quarantine-иться только из-за отсутствия repair-signal;
- если gate fail-ит, система должна либо отдать repaired final answer и сохранить его, либо не показывать его как обычный final answer;
- API-visible final answer не должен silently пропадать из saved memory.
```

### 1.3. Chat bubble / LLM canvas mismatch

Owner evidence показывает ситуацию:

```text
Видимый user turn: "Можешь на каком нибудь примере это объяснить?"
Видимый answer: пример про присутствие / диссоциацию.
Но раскрытое Полотно LLM показывает user prompt другого turn:
"привет! Как перестать наступать на одни и те же грабли?"
А LLM response начинается как ответ на первый turn.
```

Это нельзя считать косметикой. Нужно доказать, что:

```text
- каждый visible assistant bubble раскрывает canvas своего же turn;
- canvas не привязан к array index после reload;
- trace_id/canvas_id/message_id сохраняются стабильно;
- prompt canvas user message совпадает с visible user message текущего turn.
```

### 1.4. Reload history loss

Owner evidence:

```text
после перезапуска / reload web chat часть сообщений теряется;
разговор начинался с "Как не наступать на одни и те же грабли?",
но после reload виден только продолженный фрагмент.
```

После HF1:

```text
- все delivered user/assistant pairs активной сессии должны восстановиться после reload;
- если ответ был показан пользователю как final answer, он должен быть persisted;
- orphan trace/canvas не должен показываться под другим visible message.
```
