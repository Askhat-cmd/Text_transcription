## 0. Простыми словами

Владелец проекта принял решение:

```text
не делать следующий hotfix,
не застревать в бесконечной доводке частностей,
а перейти к большому этапу Cleanup / Freeze / Pilot Start Brief.
```

Последняя серия HF дала системе рабочий позвоночник:

```text
HF4: trace/reload для свежих чатов восстановлен;
HF5: найденное релевантное знание теперь доходит до Writer на direct concept follow-up;
HF6: no_internal_db и no_practice теперь доказуемы в owner/debug trace через boundary_trace_v1.
```

Да, остаются нюансы:

```text
- greeting местами звучит слишком терапевтически;
- старые debug traces после backend restart могут истекать;
- source coverage для отдельных концептов может быть слабой;
- shadow planner иногда может быть noisy/invalid при рабочем production path;
- full pytest всё ещё имеет исторический unrelated blocker `_build_llm_prompts`.
```

Но эти нюансы больше не должны удерживать проект в режиме бесконечных HF.

Цель PRD-047.37:

```text
остановить накопление хаотичных правок,
зафиксировать текущую систему как baseline,
собрать карту известных предупреждений,
описать pilot-start checklist,
подготовить cleanup/retirement backlog,
и дать проекту целостную точку опоры для следующего этапа.
```


This is accepted as product-owner decision.

### 1.2. What this means architecturally

PRD-047.37 не должен быть новым runtime-fix.

Он должен:

```text
- оформить текущий baseline;
- зафиксировать accepted invariants;
- отделить blockers от warnings;

- подготовить pilot-start brief;
```

```text

теперь нужна целостность, документация, freeze и управляемый backlog.
```

## 2. Current accepted baseline

Treat the following as current baseline unless audit proves otherwise:

### 2.1. Runtime path

```text
canonical user-facing runtime: multiagent_adapter / multiagent_v1
Writer: final answer author
knowledge-to-Writer path: writer_kb_payload_v1
trace: owner/debug observability, not user-facing content
semantic cards: advisory-only, writer-can-ignore
```

### 2.2. Accepted capabilities

```text
- latest-turn authority is the main answer target;
- hidden knowledge competence is active;
- public answer must not mention DB/chunks/cards/trace by default;
- direct knowledge questions can pull Writer KB Payload;
- direct concept follow-ups can admit selected relevant knowledge;
- no_internal_db suppresses Writer-visible knowledge and is trace-proven;
- no_practice is trace-proven and must suppress practice/homework/step routine;
- fresh trace/reload works for new chats;
- old debug traces after backend restart may expire if explicitly labelled;
- safety remains above depth, DB and style.
```

### 2.3. Accepted warnings

```text
- old pre-restart in-memory debug traces can expire after backend restart;
- greeting/contact wording may be too therapeutic;
- source exact match may be weak for concepts not represented by strong DB chunks;
- shadow planner may show invalid/noisy state while production path is correct;
- full pytest collection still has historical unrelated `_build_llm_prompts` import blocker;
- some UI/trace display labels may still need polish.
```
