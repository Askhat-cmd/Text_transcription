## 1. Три важные особенности — прочитать перед реализацией

```text
ОСОБЕННОСТЬ 1 — третья история с именем dialogue_profile: ПЕРЕПРИСВАИВАНИЕ.
Строка 548:
  dialogue_profile = normalize_dialogue_profile(ctx.get("dialogue_profile", dialogue_profile))
Здесь СТАРОЕ значение локальной dialogue_profile (из slice1) участвует
как дефолт, а результат ПЕРЕЗАПИСЫВАЕТ локальную переменную. Helper
принимает старое значение параметром, возвращает новое полем
датакласса, вызывающий код ОБЯЗАН перепривязать локальную переменную:
  dialogue_profile = slice11_result.dialogue_profile
Не пропускать перепривязку, даже если кажется, что дальше в _call_llm
переменная не используется — сохраняем семантику дословно, а не
"по анализу использований".
Сводка трёх случаев одного имени (для отчёта Исполнителя):
  APPLY-15 — kwarg = независимый ctx.get(), локальную НЕ подставлять;
  APPLY-16 — локальная как дефолт, передать параметром;
  APPLY-18 — переприсваивание, параметр внутрь + перепривязка снаружи.

ОСОБЕННОСТЬ 2 — первая self-зависимость в серии.
Строка 549 вызывает self._resolve_runtime_settings(...) — метод
определён в writer_agent_lifecycle_mixin.py (ЗАЩИЩЁННЫЙ файл, не
трогать). Helper НЕ должен знать про self: вызывающий код передаёт
bound-метод параметром resolve_runtime_settings, helper вызывает его
строго как в оригинале — с keyword-аргументом dialogue_profile= и
УЖЕ нормализованным (новым) значением.

ОСОБЕННОСТЬ 3 — порядок двух ключей патча и источники импортов.
Патч строго {"system_prompt": ..., "dialogue_profile": ...} — в этом
порядке (json.dumps без sort_keys, порядок попадает в байты snapshot).
Импорты в slice11 брать из ТЕХ ЖЕ модулей, что в writer_agent.py:
  from ..dialogue_policy import DIALOGUE_PROFILE_MVP_FREE, normalize_dialogue_profile
  from .writer_agent_prompts import WRITER_SYSTEM, WRITER_SYSTEM_MVP_FREE_DIALOGUE
В writer_agent.py: normalize_dialogue_profile и DIALOGUE_PROFILE_MVP_FREE
используются в других местах (строки ~604, ~887) — их импорты НЕ
удалять. WRITER_SYSTEM и WRITER_SYSTEM_MVP_FREE_DIALOGUE после переноса
предположительно не используются — удалить ТОЛЬКО эти два имени из
import-блока после zero-match grep (WRITER_USER_TEMPLATE — ОСТАЁТСЯ).
```

from dataclasses import dataclass

from ..dialogue_policy import (
    DIALOGUE_PROFILE_MVP_FREE,
    normalize_dialogue_profile,
)
from .writer_agent_prompts import (
    WRITER_SYSTEM,
    WRITER_SYSTEM_MVP_FREE_DIALOGUE,
)


@dataclass(frozen=True)
class CallLLMSlice11RuntimeSettingsAndSystemPromptResult:
    dialogue_profile: str
    runtime_settings: dict[str, Any]
    system_prompt: str
    last_debug_patch: dict[str, Any]


def _apply_call_llm_slice11_runtime_settings_and_system_prompt(
    ctx: dict[str, Any],
    *,
    dialogue_profile: str,
    resolve_runtime_settings: Callable[..., dict[str, Any]],
) -> CallLLMSlice11RuntimeSettingsAndSystemPromptResult:

        ctx.get("dialogue_profile", dialogue_profile)
    )
    runtime_settings = resolve_runtime_settings(dialogue_profile=dialogue_profile)

        else WRITER_SYSTEM

    last_debug_patch: dict[str, Any] = {
        "system_prompt": system_prompt,
        "dialogue_profile": dialogue_profile,
    }
    return CallLLMSlice11RuntimeSettingsAndSystemPromptResult(
        dialogue_profile=dialogue_profile,
        runtime_settings=runtime_settings,
        system_prompt=system_prompt,
        last_debug_patch=last_debug_patch,
    )
