## 1. Три важные особенности — прочитать перед реализацией

```text
ОСОБЕННОСТЬ 1 — тайминг duration_ms и подмена time.perf_counter.
Строка 576: duration_ms = int((time.perf_counter() - start_ts) * 1000).
Snapshot-харнесс делает результат детерминированным (duration_ms=123),
подменяя perf_counter КАК АТРИБУТ ОБЩЕГО МОДУЛЯ time
(writer_agent_module.time.perf_counter = _PerfCounter([100.0, 100.123])).
Подмена атрибута общего модуля видна из любого файла, который
обращается к нему как time.perf_counter(). Поэтому в slice12
ОБЯЗАТЕЛЬНО:
  import time            # именно так
  ... time.perf_counter() ...
ЗАПРЕЩЕНО: from time import perf_counter — прямая привязка функции
сделает её невидимой для подмены, snapshot перестанет быть
детерминированным и сравнение до/после развалится.
Количество вызовов perf_counter в потоке _call_llm должно остаться
ровно 2 (start_ts в методе + один в helper'е) — не добавлять лишних.

ОСОБЕННОСТЬ 2 — вторая self-зависимость: _estimate_cost.
Строка 575 вызывает self._estimate_cost(tokens_prompt=...,
tokens_completion=...) — метод из ЗАЩИЩЁННОГО
writer_agent_fallback_state_mixin.py (возвращает Optional[float]).
Как в APPLY-18: вызывающий код передаёт bound-метод параметром
estimate_cost, helper вызывает его строго keyword-аргументами
tokens_prompt= и tokens_completion=, ровно один раз.

ОСОБЕННОСТЬ 3 — порядок 13 ключей патча и точность значений.
json.dumps без sort_keys — порядок попадает в байты snapshot. Порядок
строго как в оригинале (см. код Шага 1): model, api_mode, temperature,
max_tokens, timeout, llm_response, tokens_prompt, tokens_completion,
tokens_total, estimated_cost_usd, duration_ms, error, fallback_used.
llm_response = result.text БЕЗ str()-обёртки (как в оригинале),
error=None и fallback_used=False — литералы. Кортежная распаковка
tokens_* переносится как есть.
```
