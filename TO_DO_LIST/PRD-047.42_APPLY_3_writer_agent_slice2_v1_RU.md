## 0. Короткое решение

```text
Slice 2: 8 методов класса WriterAgent, помеченных @staticmethod — то
есть формально уже не зависят от self, просто пока лежат внутри класса
как методы, а не как отдельные функции. Следующий по безопасности шаг
после slice 1.

Проверено архитектором напрямую в текущем коде (не по старой карте
PRD-047.42 -- она размечала строки ДО slice 1, номера сдвинулись):

  _build_gentle_close_reply()               -- строка 1910-1912
  _build_no_practice_fallback_text(...)     -- строка 1914-1923
  _strip_optional_followup_invitation(...)  -- строка 1977-1978+
  _detect_language(text)                    -- строка 2000-2001+
  _format_hits(hits)                        -- строка 2005-2006+
  _format_diagnostic_summary(summary)       -- строка 2011-2012+
  _static_fallback(contract)                -- строка 2032-2103 (~70 строк,
                                                самый крупный в этом срезе)
  _normalize_name(raw_name)                 -- строка 2139-2140+

Для каждого проверено: тело не содержит "self." (grep по диапазону
строк -- пусто), ни один из 8 не вызывается снаружи writer_agent.py
(grep по всему репозиторию -- пусто, кроме собственных тестов).

Важный нюанс, которого не было в slice 1: некоторые из этих 8 методов
вызываются ИЗНУТРИ других, НЕ входящих в этот срез методов класса --
через self. (например self._strip_optional_followup_invitation(text)
на строке 1937, self._build_no_practice_fallback_text(...) на строке
1948). Slice 1 такого не имел (те 4 функции уже были module-level).
Значит здесь придётся решить: либо оставить в классе тонкие
@staticmethod-обёртки, которые зовут вынесенную функцию (ноль риска
для вызывающих self.xxx() мест), либо поправить сами вызывающие места
на прямой вызов функции без self. Выбор -- за Исполнителем (раздел 4),
но требование одно: ни одно поведение не должно измениться.
```

## 7. Simple explanation

```text
Второй маленький, безопасный кусочек. В прошлый раз убрали 4 функции,
которые вообще не были частью класса. Сейчас убираем 8 методов класса,
которые технически уже помечены как "не нуждаются в самом объекте"
(@staticmethod) -- просто лежат внутри класса по привычке. Их можно
вынести так же безопасно. Есть только одна тонкость: пара мест в файле
дёргает их через "self." -- решаем аккуратно, не меняя, что происходит
на самом деле, только откуда код физически лежит.
```
