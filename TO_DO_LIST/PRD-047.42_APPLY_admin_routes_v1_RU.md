## 0. Короткое решение

```text
PRD-047.42 дал карты для 3 файлов (writer_agent.py, admin_routes.py,
writer_contract.py). Резать все три одним PRD — то же нарушение
"каждый PRD маленький", что уже разбирали. Рекомендация Исполнителя
в next_recommendation.md (PRD-047.42) была явной: "keep admin_routes.py
split separate from writer_agent.py; they have different blast radii".

Начинаем с admin_routes.py — не потому что он самый важный, а потому что
он объективно самый безопасный из трёх:
  - 0 production-импортёров (только 1 тест + сам инструмент-картограф),
  - блоки в карте — это независимые route-хендлеры, не один сросшийся
    метод с общим состоянием (как у writer_agent._call_llm или
    writer_contract.to_prompt_context);
  - если что-то пойдёт не так, откат для одного файла проще, чем для
    трёх сразу.

writer_agent.py и writer_contract.py — отдельные будущие PRD
(047.42-APPLY-2, 047.42-APPLY-3), после того как этот пройдёт чисто.

Публичный контракт файла — main.py делает
"from .admin_routes import admin_router, admin_router_v1" — это
единственное, что снаружи должно остаться работать один-в-один.
```

## 2. Scope

```text
A. Создать 10 новых файлов bot_agent-уровня по карте раздела 1 —
   перенести код БЕЗ изменений логики (только перемещение + правка

   и admin_router_v1 из route-объектов новых модулей, экспортирует их
   так же, как сейчас — main.py не должен менять свою строку импорта.
C. Расширить behavior-snapshot тесты до полного покрытия: 1 тест на
   каждый зарегистрированный route (все ~45), не выборочно 3.
D. Gate: snapshot "ДО" (текущий admin_routes.py) против snapshot
   "ПОСЛЕ" (10 файлов + тонкий агрегатор) — построчно идентичные HTTP
   ответы на одинаковые запросы.
```

## 7. Risk register

```text
Risk R — main.py:244 тихо перестаёт работать
  Symptom: рефактор меняет имя admin_router или структуру экспорта, но
           main.py не обновляют синхронно -- ImportError при старте.
  Response: external_contract_proof.md обязателен именно для этой одной
            строки, не общими словами "контракт сохранён".

Risk S — Порядок регистрации route'ов меняется при переносе
  Symptom: FastAPI матчит по первому совпадению; если порядок в
           агрегаторе отличается от исходного, конкурирующий route
           (например /admin/config/{key} vs /admin/config/schema)
           может начать перехватывать не те запросы.
  Response: module_extraction_log.md фиксирует порядок регистрации до
            и после явно, не только факт переноса кода.

Risk T — Snapshot "before" сам по себе неполный (не все ~45 route)
  Symptom: 3 route уже засняты в PRD-047.42, кто-то решает, что этого
           достаточно, и не досчитывает остальные ~42.
  Response: раздел 4 п.1 explicitly требует полный список через
            .routes на реальном router-объекте, не список "на глаз".
```

## 8. Simple explanation

```text
Карта уже готова (прошлый PRD). Этот PRD — первая настоящая резка,
и начинаем с самого безопасного из трёх файлов: админки. У неё почти
нет внешних "потребителей" (по сути только сам сервер при старте),
и её куски — это независимые обработчики запросов, не один запутанный
метод. Перед тем как резать, "фотографируем" ответ КАЖДОЙ админ-ручки
(не выборочно, а все ~45), режем на 10 файлов поменьше, фотографируем
снова, сверяем -- должно быть побайтово одинаково. writer_agent.py и
writer_contract.py (там, где формируется сам ответ бота) — трогаем
только после того, как этот, самый безопасный, пройдёт полностью чисто.
```
