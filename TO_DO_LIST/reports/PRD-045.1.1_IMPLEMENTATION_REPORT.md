# PRD-045.1.1 IMPLEMENTATION REPORT

## 1) Где терялся quality_trace
`quality_trace` терялся в API-доставке между runtime и response:
- В `bot_psychologist/api/routes/chat.py` брали `raw = result.get("debug_trace") or result.get("debug")`.
- Если был `debug_trace`, поля `quality_trace_*` из `debug` не попадали в `trace_payload`.
- В `bot_psychologist/api/models.py::DebugTrace` отсутствовали поля `quality_trace_version`, `quality_trace`, `quality_trace_error`, и Pydantic отбрасывал extra keys.

## 2) Что изменено
1. `bot_psychologist/api/models.py`
- В `DebugTrace` добавлены:
  - `quality_trace_version: Optional[str]`
  - `quality_trace: Optional[Dict[str, Any]]`
  - `quality_trace_error: Optional[str]`
- В `MultiAgentTraceResponse` добавлены те же три опциональных поля.

2. `bot_psychologist/api/routes/chat.py`
- В adaptive debug path введён merge `debug_trace + debug` с приоритетом непустых значений из `debug`.
- В extension keys добавлены:
  - `quality_trace_version`
  - `quality_trace`
  - `quality_trace_error`
- Та же merge-логика добавлена в streaming trace event path.

3. `bot_psychologist/api/debug_routes.py`
- В `/api/debug/session/{session_id}/multiagent-trace` добавлен проброс:
  - `quality_trace_version`
  - `quality_trace` (только dict)
  - `quality_trace_error`

4. `bot_psychologist/docs/trace_runtime.md`
- Добавлено требование доставки quality trace во внешние debug surfaces.
