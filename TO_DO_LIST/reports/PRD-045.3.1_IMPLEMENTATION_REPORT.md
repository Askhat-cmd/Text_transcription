## 3. Какие deterministic phrases добавлены
- `_HYPO_KEYWORDS`: расширен low-resource словарем (RU/EN: exhausted/drained/устал/нет ресурса и т.д.).
- `_CONTACT_PHRASES`: добавлены no-advice/no-analysis/support формулировки.
- `_SOLUTION_PHRASES`: добавлены concrete-step запросы (RU/EN).
- `_CLARIFY_PHRASES`: добавлены ambiguity/understand паттерны.
- `_DEFENSIVE_PHRASES`: добавлены defensive паттерны.
- Добавлен `_COLLAPSED_PHRASES` и branch `collapsed` в `_detect_openness` (порядок: collapsed -> defensive -> open).

## 4. Что изменено в routing
В `thread_manager._mode_and_goal(...)`:
- было: `solution -> practice` только если `phase == "explore"`;
- стало: `solution -> practice` всегда (после safety/contact/nervous checks), в том числе для нового thread.
