## 3. Как изменена phase transition logic
В `_resolve_phase_with_debug(...)` добавлен guard перед `clarify_to_explore`:
- если `current_phase=clarify`, `relation=continue`, и при этом
  - `nervous_state in {hyper, hypo}` или
  - `intent=contact`,
  то фаза остаётся `clarify`.
- reason code:
  - `low_resource_hold_phase` для `hyper/hypo`
  - `contact_hold_phase` для `contact` при `window`

Обычное правило `clarify -> explore` для `window + continue + turns>=2` сохранено.
