# PRD-045.6.3 TURN SUMMARY CONTRACT REPORT

## Contract File
- `bot_psychologist/bot_agent/multiagent/contracts/turn_llm_summary.py`

## Contract Shape
- `status`: `pending|ready|failed|disabled`
- `summary`
- `important_quote`
- `open_loop`
- `user_need`
- `assistant_move`
- `emotional_tone`
- `summary_version=turn_llm_summary_v1`
- `summary_method=llm_abstractive_v1`
- `model`
- `provider`
- `source_hash`
- `created_at`
- `error`

## Validator Policy
`validate_turn_llm_summary(summary, source_hash)` allows context usage only if:
1. status is `ready`
2. `summary_version == turn_llm_summary_v1`
3. `summary` is non-empty
4. `source_hash` matches current turn hash
5. `created_at` is present

Any other state/result triggers deterministic fallback.
