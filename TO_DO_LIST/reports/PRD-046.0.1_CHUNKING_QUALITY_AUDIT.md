# PRD-046.0.1 Chunking Quality Audit

## Current BookChunker behavior
- Chapter detection: `#` headers, `Глава/Chapter`, numbered headings, uppercase headings.
- Token policy: `target/min/max/overlap` with paragraph-first assembly and sentence/word fallback splits.
- Overlap behavior: tail overlap by token window (`_take_last_tokens`) after chunk flush.
- Practice splitting risk: practice steps can be split across neighboring chunks because chunking is token-budget-driven, not practice-boundary-aware.
- heading_path preservation status: only `chapter_title` is preserved; nested heading path is not stored as structured hierarchy.

## Current SemanticChunker behavior
- YouTube segmentation logic: split by pauses (`\n\n`, `...`) and then sentence/word fallback.
- risks for lecture/transcript material: semantic boundaries can drift; one chunk may mix lens/practice/safety statements.

## Current metadata limitations
- UniversalBlock fields: core source/chunk metadata exists, but governance and chunking quality were absent before this PRD.
- JSON export fields (before PRD): no `metadata.governance`, no `metadata.chunking_quality`.
- Chroma metadata fields (before PRD): flat source/sd fields only, no governance policy fields.
- Query response fields (before PRD): `ChunkResult` had no governance summary.

## Risks for Neo MindBot / Forge materials
- practice sections may split incorrectly across token boundaries.
- lens/theory/case can be mixed in one chunk.
- strong claims may be exposed without policy markers if governance is absent.
- direct quote risk for internal/architecture notes without explicit `not_for_direct_quote`.
- low-resource contraindication risk if practice chunks are not traced.

## Decision
- This PRD adds governance pass-through and chunking-quality trace.
- Deep chunker redesign and runtime filtering are deferred to follow-up PRDs.
