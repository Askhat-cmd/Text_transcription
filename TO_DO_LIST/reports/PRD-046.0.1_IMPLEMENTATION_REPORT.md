## Bot_data_base flow audit

### Upload path
1. Book upload start: `api/routes/books.py::ingest_book`.
2. YouTube upload start: `api/routes/youtube.py::ingest_youtube`.

### Chunking / block creation path
3. `UniversalBlock` создаётся в:
- `chunkers/book_chunker.py::_make_block`
- `chunkers/semantic_chunker.py::_make_block`

4. Где теряется структура heading:
- В `BookChunker` сохраняется только `chapter_title` (nested heading_path отсутствует).
- Для YouTube используется pause/sentence segmentation без иерархии заголовков.

5. Точка минимальной интеграции governance:
- После `BlockNormalizer` и до JSON export / Chroma indexing в `PipelineRunner`.

### Export / index / query
6. JSON export fields до PRD:
- `id,text,title,summary,sd_level,complexity,source,metadata` без governance policy.

7. Chroma metadata fields до PRD:
- `sd_level,author_id,author,source_type,language,complexity,source_id,title,source_title,chapter_title,chunk_index`.

8. `/api/query/` до PRD возвращал:
- chunk score/meta поля без governance summary.

9. `bot_psychologist DBApiClient` до PRD:
- принимал `RetrievedChunk` без governance поля.

## Implemented changes

### 1) Governance adapter in Bot_data_base
Added:
- `Bot_data_base/governance/governance_adapter.py`
- `Bot_data_base/governance/chunking_quality.py`
- `Bot_data_base/governance/__init__.py`

Key points:
- deterministic rule-based classification (`practice/lens/safety/theory/style`).
- profile defaults: `general_book`, `practice_manual`, `architecture_notes`, `transcript`.
- governance schema attached to each block.
- chunking quality trace attached to each block.

### 2) UniversalBlock extension (backward-compatible)
Updated:
- `Bot_data_base/models/universal_block.py`

Added fields:
- `governance: dict[str, Any] = {}`
- `chunking_quality: dict[str, Any] = {}`

`to_bot_format()` now includes:
- `metadata.governance`
- `metadata.chunking_quality`

### 3) Pipeline integration
Updated:
- `Bot_data_base/pipeline_runner.py`

Flow now:
- chunking -> sd_labeling -> normalizing -> governance adapter -> chunking quality -> JSON export -> Chroma indexing.

Both flows updated:
- `run_book(..., governance_profile, source_kind)`
- `run_youtube(..., governance_profile, source_kind)`

### 4) API and Web UI governance inputs
Updated:
- `Bot_data_base/api/schemas.py`
- `Bot_data_base/api/routes/books.py`
- `Bot_data_base/api/routes/youtube.py`
- `Bot_data_base/web_ui/books.html`
- `Bot_data_base/web_ui/youtube.html`
- `Bot_data_base/web_ui/static/books.js`
- `Bot_data_base/web_ui/static/youtube.js`

Behavior:
- optional governance profile/kind accepted.
- invalid profile falls back safely and warning is returned in job result.

### 5) Chroma flat governance metadata
Updated:
- `Bot_data_base/storage/chroma_manager.py`

Added flat fields:
- `governance_schema_version`
- `governance_chunk_type`
- `governance_allowed_use` (CSV)
- `governance_safety_flags` (CSV)
- `governance_lens_family` (CSV)
- `governance_low_resource_safe`
- `governance_not_for_direct_quote`
- `governance_source_style_not_user_facing`

### 6) Query API governance summary
Updated:
- `Bot_data_base/api/schemas.py`
- `Bot_data_base/api/routes/query.py`

`ChunkResult` now has optional:
- `governance: dict`

`_build_chunk_result` reconstructs governance summary from flat Chroma metadata.

### 7) bot_psychologist pass-through
Updated:
- `bot_psychologist/bot_agent/db_api_client.py`
- `bot_psychologist/bot_agent/data_loader.py`
- `bot_psychologist/bot_agent/retriever.py`

Changes:
- `RetrievedChunk.governance` added.
- `Block` model extended with `governance/chunking_quality`.
- API->Block conversion preserves governance when present.
- retriever `_chunk_to_block()` now passes governance through.
