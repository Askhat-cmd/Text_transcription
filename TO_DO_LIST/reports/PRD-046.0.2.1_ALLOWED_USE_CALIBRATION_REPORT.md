# PRD-046.0.2.1 Allowed-Use Calibration Report

## Before
`practice_manual` profile base `allowed_use` contained `practice_suggestion`.

Consequence:
- lens/safety chunks inside practice manuals could inherit a practice action permission.
- runtime filtering would be forced to compensate for ingestion-level over-permission.

## After
Allowed-use now built in two layers:

1. Profile base use:
- `practice_manual`: `writer_context`, `diagnostic_lens`
- `architecture_notes`: `internal_only`, `style_guidance`, `diagnostic_lens`
- `general_book`: `writer_context`
- `transcript`: `writer_context`, `diagnostic_lens`

2. Chunk-type add-ons:
- `practice` -> add `practice_suggestion`
- `safety` -> add `safety_protocol`, `writer_context`
- `lens` -> add `diagnostic_lens`
- `style` -> keep `style_guidance` (+ `internal_only` on architecture profile)
- `quote/case` handled without implicit `practice_suggestion`

Precision guard:
- for all non-practice chunks, `practice_suggestion` is removed.

## Assertions Verified
- practice_manual lens block:
  - no `practice_suggestion`
  - has `diagnostic_lens`
- practice_manual safety block:
  - has `safety_protocol`
  - no `practice_suggestion`
- practice block:
  - has `practice_suggestion`
- architecture non-practice block:
  - has `internal_only` and `style_guidance`
  - no `practice_suggestion`
