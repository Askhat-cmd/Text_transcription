## Allowed-Use Audit (Before)
Before calibration in `governance_adapter.py`:
- `practice_manual` profile base `allowed_use` included `practice_suggestion`.
- Risk: non-practice chunks from practice materials could inherit practice routing intent.

## Implemented Changes

### 1. Allowed-use precision guard
Updated `Bot_data_base/governance/governance_adapter.py`:
- profile-level base `allowed_use` now separated from chunk-type add-ons.
- `practice_manual` base now: `writer_context`, `diagnostic_lens`.
- `practice_suggestion` is added only when `chunk_type == practice`.
- `safety_protocol` added for `chunk_type == safety`.
- architecture/style chunks keep `internal_only` + `style_guidance` and never inherit `practice_suggestion` unless truly practice.

### 2. Mixed-intent calibration
Updated `Bot_data_base/governance/chunking_quality.py`:
- added `analyze_mixed_intent_v1` and new output fields:
  - `mixed_intent_severity`
  - `primary_role`
  - `secondary_role_markers`
  - `mixed_intent_reason`
- reduced false positives for:
  - practice blocks with light lens mention;
  - safety blocks with phrase like `эта практика не заменяет специалиста`.
- preserved warnings on true mixed chunks.

### 3. Query/governance summary pass-through
Updated:
- `Bot_data_base/storage/chroma_manager.py`
- `Bot_data_base/api/routes/query.py`

Added safe summary fields in query governance payload:
- `mixed_intent_severity`
- `primary_role`
- `secondary_role_markers`
- `mixed_intent_reason`

### 4. Bot_data_base query endpoint stability
Updated `Bot_data_base/tests/test_query_endpoint.py`:
- patched runtime loader in unit tests to avoid accidental real `PipelineRunner(config.yaml)` boot.
- endpoint tests now deterministic and green.

Updated `Bot_data_base/tests/integration/test_api_endpoints.py`:
- enforce `Bot_data_base` cwd during integration tests so `config.yaml` resolves consistently.
