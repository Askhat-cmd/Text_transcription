## Root Cause Audit (from PRD-046.0.1 baseline)
- Previous failure mode (`synthetic practice_manual -> theory=3`) was caused by token/chapter-driven chunking without reliable section boundaries.
- Governance adapter ran after chunking and often saw mixed/diluted intent markers in one chunk.
- Missing structural context: `heading_path`, explicit `section_role_hint`, split cause, and boundary confidence.

## Implemented Changes
- Added deterministic structure parser:
  - `Bot_data_base/chunkers/structure_parser.py`
- Extended `UniversalBlock` with backward-compatible section/boundary fields:
  - `heading_path`, `section_role_hint`, `boundary_confidence`, `split_reason`, `parent_section_id`
- Upgraded `BookChunker` to structure-aware flow with fallback:
  - parse markdown-like sections
  - split by section role
  - preserve practice where possible
  - split long practices by step boundaries
  - keep safety/lens/architecture boundaries
- Updated governance adapter to use structural hints and richer practice metadata:
  - role precedence by `section_role_hint`
  - architecture profile guard (`internal_only`, `style_guidance`, no implicit `practice_suggestion`)
  - mixed intent detection note
- Extended chunking quality trace:
  - `heading_path_present`, `section_role_hint`, `boundary_confidence`, `split_reason`, `mixed_intent_risk`, `practice_steps_preserved`, `role_marker_counts`
- Updated Chroma flat metadata and query pass-through summary:
  - added safe structure/boundary fields
  - added `chunking_quality` summary under returned `governance`
- Updated `bot_psychologist` loader compatibility for new metadata fields.
