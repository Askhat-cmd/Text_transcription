# PRD-046.0.2 STRUCTURE-AWARE CHUNKING REPORT

## Parser Coverage
`Bot_data_base/chunkers/structure_parser.py` (`parse_markdown_like_sections_v1`) now supports:
- Markdown headings: `#`, `##`, `###` (up to `######`)
- Chapter-like headings: `Глава ...`, `Chapter ...`
- Numbered headings: `1.`, `1.1`, `1.1.1`
- Marker headings: `Практика`, `Упражнение`, `Техника`, `Безопасность`, `Важно`, `Протокол`, `Кейс`, `Пример`, `Цитата`

Produced section contract:
- `heading_path`
- `title`
- `level`
- `text`
- `start_line` / `end_line`
- `section_role_hint`
- `boundary_confidence`

## Role-Aware Boundary Logic
`BookChunker` flow for structured sources:
- `clean_text`
- `parse_markdown_like_sections_v1`
- role-aware chunking per section
- `UniversalBlock` with structural metadata

Per-role handling:
- `practice`: preserve as single chunk when within budget; long sections split by step boundaries (`Шаг N`, numbered/bulleted items)
- `safety`: isolated boundary, no merge with theory chunks
- `lens`: isolated boundary; split only if oversized
- `architecture`: isolated boundary for internal/style notes
- `theory/unknown`: token-budget split fallback

## Fallback Behavior
Backward-compatible fallback remains active when structure is weak/unreliable:
- no valid multi-section structure
- parser fallback with `unknown` role
- non-markdown/plain books

In fallback mode, legacy chapter/token chunking behavior is preserved.

## Added Boundary Trace Fields
`UniversalBlock` now carries (optional, backward-compatible):
- `heading_path`
- `section_role_hint`
- `boundary_confidence`
- `split_reason`
- `parent_section_id`

## Known Limitations
- Heuristics are deterministic rule-based (no LLM semantic parser).
- Mixed sections that intentionally contain safety + practice markers are retained but flagged (`mixed_intent_risk`) rather than hard-rejected.
- `test_query_endpoint.py` local environment still depends on app-level config bootstrap path; this is outside PRD-046.0.2 scope.
