## Root Cause Summary
1. LLM client runtime mismatch after project migration:
- SDK exposes `chat.completions`, but `client.responses` is missing.
- Direct `responses.create` path crashed Writer/StateAnalyzer in live runtime.

2. BotDB query path unhealthy:
- `POST /api/query/` returns `503 {"detail":"ChromaDB unavailable"}`.
- Without fast guard, retrieval waited on API errors before fallback, inflating Memory stage latency.

## Implemented Changes

### A) LLM compatibility guard
Updated:
- `bot_psychologist/bot_agent/multiagent/agents/agent_llm_client.py`

Done:
- Runtime capability detection for:
  - `responses.create`
  - `chat.completions.create`
- New api modes:
  - `responses`
  - `chat_completions`
  - `chat_completions_compat`
- Explicit capability error:
  - `OpenAI client has neither responses.create nor chat.completions.create`
- Compat fallback logic:
  - if old SDK rejects `response_format` -> retry without it
  - if old SDK rejects `max_completion_tokens` -> retry without explicit token limit
- Safe diagnostics log:
  - `[AGENT_LLM] sdk_capabilities ... selected_mode=...`

### B) Writer/StateAnalyzer visibility
No contract changes required in agent code:
- both already expose `api_mode`/`error` via debug.
- after HF1, compat path correctly sets:
  - `writer_api_mode=chat_completions_compat`
  - `writer_error=None` on success
  - `state_analyzer_error=None` on success

### C) BotDB circuit breaker / fast fallback
Updated:
- `bot_psychologist/bot_agent/retriever.py`
- `bot_psychologist/bot_agent/config.py`
- `bot_psychologist/bot_agent/runtime_config.py`

Done:
- Added circuit fields and state tracking in retriever.
- Added config flags:
  - `BOT_DB_CIRCUIT_BREAKER_ENABLED` (default `True`)
  - `BOT_DB_CIRCUIT_BREAKER_TTL_SECONDS` (default `60`)
  - `BOT_DB_FAST_FAIL_ON_503` (default `True`)
- Circuit opens on:
  - `connect`
  - `timeout`
  - `http_status=503` (incl. `ChromaDB unavailable`)
- While circuit open:
  - skip BotDB API
  - go directly to semantic/TF-IDF fallback
  - emit short warning log
- Added retrieval debug snapshot (`retrieval_source_used`, circuit/error markers).
- Reduced DB client internal retries for retriever path (`retries=1`) to cut first-failure delay.
