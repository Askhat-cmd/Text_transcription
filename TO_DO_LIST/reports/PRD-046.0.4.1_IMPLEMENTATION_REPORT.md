## What Was Implemented
1. `ChromaManager` now accepts explicit `embedding_model_name` and uses config-driven model selection.
2. `PipelineRunner` passes `embedding.model` from `config.yaml` into `ChromaManager`.
3. Added robust collection health probe in `ChromaManager`:
- safe count with structured error capture
- embedding probe dimension
- collection dimension detection when available
- mismatch signal
4. Added governed index gate `governed_index_gate_v1` in `kb_quality_audit`:
- strict blocker thresholds for governance/allowed_use/safety/not_for_direct_quote/summary/boundary
- lens_family as non-blocking desired threshold
- explicit statuses: `ready | not_ready | degraded`
5. Added safe recovery CLI `Bot_data_base/tools/chroma_recovery.py` with modes:
- `--probe-only`
- `--dry-run`
- `--reset` / `--reindex-from-json`
- destructive operations require `--confirm`
6. Query smoke snapshot logs only safe metadata (`content_preview_len`), no raw chunk text.
