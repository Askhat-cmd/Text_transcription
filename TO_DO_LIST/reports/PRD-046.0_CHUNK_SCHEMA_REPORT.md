## 3) Allowed Use Policy v1
- `architecture_notes` и источники уровня `конспект` по умолчанию получают `internal_only`/`style_guidance`/`diagnostic_lens` и `source_style_not_user_facing`.
- `practice` chunks получают `practice_suggestion` только в governed form с safety flags.
- `safety/protocol` chunks маркируются для `safety_protocol`.
- `excluded` chunks получают `do_not_use`.
- High-risk утверждения помечаются `not_for_direct_quote`.

## 4) Practice Metadata v1
`practice_metadata` содержит:
- `goal`
- `duration`
- `steps_count`
- `requires_journaling`
- `body_based`
- `low_resource_safe`

Если `low_resource_safe=false`, добавляется `practice_requires_low_resource_check`.

## 5) Known Limitations
- Rule-based classification не покрывает все лингвистические формулировки.
- В runtime retrieval пока нет полного policy-filter по nested governance metadata.
- Dry-run preview intentionally redacts long raw text to avoid source leakage.
