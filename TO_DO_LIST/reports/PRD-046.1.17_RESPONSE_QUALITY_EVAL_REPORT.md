# PRD-046.1.17 Response Quality Eval Report

## Что покрыто
- Offline deterministic eval pack с `24` curated live-like scenarios.
- Полное покрытие групп: `A/B/C/D/E`.
- Rubric с `10` quality dimensions.
- Candidate set: `acceptable + weak + hard_fail` на каждый сценарий.

## Какие dimensions оцениваются
1. `thread_continuity`
2. `state_depth_fit`
3. `micro_shift_quality`
4. `non_directiveness`
5. `emotional_attunement`
6. `anti_repetition`
7. `non_bookishness`
8. `kb_boundary_respect`
9. `safety_preservation`
10. `answer_actionability`

## Сильные стороны текущего eval-pack
- `acceptable_candidate_pass_rate=1.0`
- `weak_candidate_detection_rate=1.0`
- `hard_fail_detection_rate=1.0`
- Все обязательные dimensions и scenario groups присутствуют.

## Слабые стороны / ограничения v1
- Это fixture-based deterministic оценка; без живого replay и без runtime activation.
- Weak-case queue генерируется как calibration backlog, но в текущем наборе нет blocker-level gaps по качеству.

## Hard-fail cases
- Детерминированно ловятся:
  - `raw_kb_quote_exposure`
  - `kuznitsa_authority_citation`
  - `high_stakes_directive_advice`

## Почему eval-pack безопасен
- `provider_called=false`
- `runtime_activation_performed=false`
- `no_runtime_authority_expansion_passed=true`
- `normal_user_apply_count=0`
- `production_mutation_detected=false`
- `artifact_encoding_hygiene_passed=true`

## Что улучшать дальше
- Добавить более вариативные candidate profiles на границе pass/fail.
- Расширить критерии `non_bookishness` и `state_depth_fit` на дополнительные языковые паттерны.
- Подготовить controlled quality calibration PRD без runtime authority expansion.
