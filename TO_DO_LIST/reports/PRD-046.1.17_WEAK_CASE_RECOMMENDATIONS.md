# PRD-046.1.17 Weak Case Recommendations

## Weak case queue summary
- Queue generated from failed weak/hard-fail candidates for all scenarios.
- Severity profile: mostly `medium`, hard-fail entries marked as `blocker` for detector validation.

## Grouped recommendations by dimension
- `state_depth_fit`
  - Риск: перегруз анализа в low-resource/hyper/hypo.
  - Рекомендуемое действие: усилить guidance по depth limiting и stabilize-first phrasing.
- `non_directiveness`
  - Риск: директивный тон в I-W+ defensive и high-stakes prompts.
  - Рекомендуемое действие: tighten rule-set по autonomy-support markers.
- `non_bookishness`
  - Риск: doctrinal/lecture tone вместо живого отражения.
  - Рекомендуемое действие: отдельные style constraints и дополнительные negative markers.
- `kb_boundary_respect`
  - Риск: authority цитирование и raw-quote framing.
  - Рекомендуемое действие: оставить hard-fail rules неизменными и расширить detector examples.

## Severity handling
- `blocker`: только для hard-fail классов (unsafe/directive/raw-quote/authority citation).
- `high`: >3 критических dimension fails в одном candidate.
- `medium`: базовые threshold fails без hard-fail.

## Нужно ли переходить к rollout
- Нет.
- Этот PRD предназначен для quality evidence/calibration и не открывает runtime authority expansion.
