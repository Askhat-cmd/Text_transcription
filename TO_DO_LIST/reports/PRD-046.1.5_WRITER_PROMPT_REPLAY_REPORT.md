# PRD-046.1.5 Writer Prompt Replay Report

## Scope
Implemented an offline-only replay layer for deterministic comparison of:
- baseline `WriterContract.to_prompt_context()`
- candidate prompt-context enriched by pilot constraints namespace

## Core Invariants
- `activation_mode = offline_replay_only`
- `apply_to_writer_contract = false`
- `apply_to_writer_prompt = false`
- `apply_to_final_answer = false`
- `provider_called = false`

## Replay Construction
- Baseline prompt-context is built from production `WriterContract`.
- Candidate prompt-context is a deep copy of baseline plus:
  - `pilot_writer_constraints` namespace
- Baseline object is not mutated.
- Candidate is not injected back into runtime writer path.

## Deterministic Checks
- Safety checks (depth/question tightening in safety contexts).
- KB boundary checks (forbidden fields, kb_usage_mode allowlist, no quote mode).
- Conflict checks (`must_do` vs `must_not_do`).
- Prompt bloat checks (delta ratio/size guard).
- Non-mutation hash proof for WriterContract and baseline context.

## Eval Outcome
- Fixture: `40` cases.
- Result: `cases_passed=40/40`, `final_status=passed`.
- Runtime smoke: passed (`offline_replay_only`, no apply flags, provider not called).
