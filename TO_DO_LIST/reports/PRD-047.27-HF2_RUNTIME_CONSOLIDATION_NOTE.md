# PRD-047.27-HF2 Runtime Consolidation Note

## Canonical
- multiagent_adapter
  Canonical user-facing runtime entrypoint today. Web Chat and API user traffic should go through this path.
- ull_multiagent
  Canonical execution core behind the adapter. Keep this as the single live multiagent runtime, not a separate user-facing branch.
- writer_kb_payload_v1
  Canonical knowledge-to-Writer path today. Legacy semantic hits fallback is emergency-only.
- current_turn_focus_v1
  Canonical retrieval query builder for current-turn knowledge focus.

## Temporary Pilot
- semantic_cards_pilot_v1
  Temporary advisory-only Writer enrichment pilot. Local/dev proof is accepted; expansion is deferred.
- overlay_shadow_trace
  Temporary trace-only observability surface. No runtime authority.
- Diagnostic Center mode
  Developer-governed advisory runtime surface. Do not expand broad activation.

## Compatibility Only
- legacy fallback path
  Compatibility/emergency surface only. Keep for bounded failure handling, not feature growth.
- legacy graph / compatibility surface
  Read-only compatibility lane. Not a primary runtime control surface.

## Answers
1. Canonical user-facing runtime today:
   multiagent_adapter over the single multiagent runtime.
2. Canonical knowledge path today:
   retrieval -> current_turn_focus_v1 -> writer_kb_payload_v1 -> optional advisory semantic cards.
3. Temporary pilot toggles:
   SEMANTIC_CARDS_PILOT_ENABLED, trace-only overlay, developer-local Diagnostic Center controls.
4. Legacy/fallback paths that remain and why:
   fallback payload and compatibility surfaces remain only for resilience and observability while cleanup stays explicit.
5. What can be retired later:
   duplicate compatibility labels, emergency fallback visibility noise, and any pilot-only toggles once PRD-047.28 defines a unified baseline.
6. What must not be expanded further:
   parallel runtime branches, owner-only runtime flags, admin-only semantic-card overrides, or a second knowledge delivery path beside writer_kb_payload_v1.
