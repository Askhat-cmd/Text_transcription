## Active Line Module Changes
- Added `ACTIVE_LINE_VERSION = "active_line_v1"`.
- Added `ActiveLineState` dataclass and serialization.
- Added deterministic intent classification and continuity/repair behavior builder.
- Added mechanical-revoicing opener detector.

## Intent Classifier Rules
- Handles intents:
  - `understand_mechanism`
  - `ask_for_practice`
  - `ask_for_direct_step`
  - `short_support`
  - `known_concept_question`
  - `correction_of_bot`
  - `thanks_close`
  - `unknown`
- `unknown` is normalized to `understand_mechanism` for continuity-first behavior in this PRD.
- Explicit `no practice` phrases are interpreted as mechanism intent.

## Writer Contract Changes
- Added optional `active_line` payload to `WriterContract`.
- Added active-line fields into prompt context map with backward-compatible defaults.

## Prompt Changes
- Added dedicated active-line section in Writer system/user templates.
- Enforced active-line policy fields in prompt (`should_offer_practice`, `revoicing_allowed`, `repair_mode`, continuity mode).

## Revoicing Policy
- Mechanical revoicing is suppressed by default in continuity and repair paths.
- Revoicing is allowed only for limited start/new-line practice-request path.
- Compliance layer strips/rewrites mechanical opener responses when disallowed.

## Practice Suppression Changes
- Suppresses unsolicited action/practice in mechanism-first and correction intents.
- Adds repair-specific correction fallback without step-offers.
- Tightens marker set for timer/step phrase suppression in compliance.

## Trace / Admin Changes

- Admin runtime effective now exposes `active_line` read-only state with calibration summary.
