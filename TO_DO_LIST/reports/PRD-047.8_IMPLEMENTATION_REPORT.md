## 2. Root Cause Summary
- Runtime was over-constrained for owner MVP dialogue: short-support/regulate often overrode explicit requests for detailed explanation.
- Planner lacked long/deep and expansion-specific shapes.
- Writer compliance layer could collapse responses into short canned outputs even when user asked for detailed clarification.

## 4. Dialogue Profile Implementation
- Added runtime profile `DIALOGUE_PROFILE` with allowed values `safe_guided | mvp_free_dialogue` in shared `RuntimeConfig`.
- Profile is consumed by orchestrator and passed as `dialogue_policy` to planner/writer contract.
- No duplicated orchestrator/planner/writer/API path; single runtime flow retained.

## 5. Writer Prompt and Behavior Changes
- Added `WRITER_SYSTEM_MVP_FREE_DIALOGUE` prompt variant.
- Writer selects prompt variant by `dialogue_profile` in the same writer agent.
- In `mvp_free_dialogue` increased writer max token budget (`>=2500`) and reduced hard post-compliance collapsing for expansion requests.
- Added repair-and-expand fallback behavior for "я не понял / объясни нормально".

## 6. Response Planner Changes
- Added response depths: `long`, `deep`.
- Added answer shapes: `concept_explanation_full`, `expanded_explanation`, `example_driven_explanation`, `repair_and_expand`.
- Added expansion/repair marker handling for MVP mode.
- Added MVP rule: hypo-state alone no longer forces short_support.

## 7. Concept Continuation
- Added active-concept continuation helper:
  - follow-up expansion requests in `mvp_free_dialogue` inherit previous `active_concept`.
- Orchestrator now stores/propagates `active_concept` through `thread_state.active_frame` and `dialogue_policy`.

## 8. Admin/Web Runtime Visibility
- `/api/admin/runtime/effective` now exposes `dialogue_profile` effective block with developer-local warning.
- Runtime payload includes writer token profile fields and planner advisory-mode flag.
- UI runtime tab now displays `Dialogue Profile: safe_guided / mvp_free_dialogue` and warning when active.

## 9. Drift Guard and Observability
- Drift guard remains observe-only.
- Added MVP expansion-aware length envelope for expanded shapes.
- Runtime payload exposes MVP expansion exceptions metadata.
