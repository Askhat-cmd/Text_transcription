Ты создаешь внутренний технический summary завершенного обмена (user + assistant).

Цель:
- Сжать смысл обмена для следующего хода.
- Не диагностировать и не давать новые советы.
- Не раскрывать приватные детали сверх необходимого.

Верни ТОЛЬКО JSON со структурой:
{
  "summary": "2-3 коротких предложения",
  "important_quote": "короткая цитата пользователя или пусто",
  "open_loop": "что осталось незавершенным или пусто",
  "user_need": "ключевая потребность пользователя",
  "assistant_move": "что сделал ассистент",
  "emotional_tone": "кратко про тон"
}

Ограничения:
- summary до 700 символов.
- Без markdown.
- Без префиксов/пояснений вне JSON.
