# Bot Psychologist Roadmap (Дорожная карта Bot Psychologist)

## Already Implemented (Уже реализовано)

- Адаптивный runtime в проде: `answer_adaptive.py` работает как фасад-оркестратор, основная логика вынесена в `adaptive_runtime/`.
- Завершена модуляризация `answer_adaptive.py`: 144 волны, архитектура стабилизирована.
- Web UI в рабочем состоянии: чат, сессии, inline trace, dev-режим диагностики.
- Trace contract `v2` внедрен и используется в API и Web UI.
- Knowledge Graph и retrieval-пайплайн работают в рабочем контуре.
- Rerank в проде: используется `VoyageReranker`.
- Тестовая база после рефакторинга стабильна: `501 passed, 13 skipped`.

## Short Term 1-3 Months (Краткосрочно, 1-3 месяца)

- Устранить остатки проблем кодировок в документации и комментариях.
- Нормализовать документационный контур: единый стиль, единые ссылки, убрать дубли.
- Ввести минимальный smoke-набор автопроверок перед релизом (API + Web UI + trace).
- Финализировать операционные runbook-документы для локального запуска и восстановления после сбоев.

## Medium Term 3-6 Months (Среднесрочно, 3-6 месяцев)

- Подготовить и внедрить базовый CI pipeline (линтеры, тесты, сборка Web UI).
- Добавить централизованный runtime-мониторинг (latency, error-rate, token/cost метрики).
- Подготовить инфраструктуру к масштабированию API (очереди, пул воркеров, ограничения нагрузки).
- Укрепить тестовый контур: больше контрактных и интеграционных тестов для trace/API.

## Long Term (Долгосрочно)

- Production-grade CI/CD с контролируемыми окружениями и rollback-процедурами.
- Полноценная observability-платформа: метрики, трассировки, алерты, аудит изменений конфигурации.
- Горизонтальное масштабирование сервиса и изоляция критичных компонентов.
- Развитие мультиканальности (например, Telegram) после стабилизации текущего Web-контура.

## Current Technical Debt (Технический долг, актуальный)

- CI/CD отсутствует как обязательный релизный контур.
- Мониторинг и алертинг частично ручные, нет единой панели наблюдаемости.
- Масштабируемость ограничена текущим single-host сценарием.
- Нужна унификация документации и контроль кодировок в текстовых артефактах.
