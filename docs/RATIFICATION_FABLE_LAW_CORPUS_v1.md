# РАТИФИКАЦИЯ КОРПУСА FABLE — v1

**Назначение:** по прямому требованию Response 07 (Часть 2, находка А) — корпус
законов, с которым владелец ни разу не спорил, decoration, не конституция.
Это не формальность. Отклонить хотя бы пару пунктов — здоровый сигнал,
не провал.

**Как заполнять:** напротив каждого пункта — одно из трёх:
`[П]` принято · `[ПП]` принято с правкой (написать, с какой) · `[О]` отвергнуто.
Ответ дольше десяти секунд на пункт — сам по себе находка (это же правило
Fable использовала во всех owner-чек-листах серии).

Возвращайте мне результат в любом виде — построчно или списком отвергнутых.
Отвергнутые/исправленные пункты становятся приоритетным материалом для
такта 1, а не пропускаются молча.

```text

П25 одна психика как ground truth · П4 гравитация именованного
```

## Четыре корня

```text
[ ] Закон формы — "артефакт, не орган": сущности живут как данные
    и контракты, не как именованные модули с папками.

[ ] Закон власти — "один автор, остальные советуют или свидетельствуют":
    Writer единственный автор; суждения о человеке — только рамка;
    всё, что пересекает границу, декларативно.

[ ] Закон роста — "рост только по записанному провалу, с потребителем
    и сроком": новая сущность рождается против провала, имеет
    названного потребителя и условие смерти.

[ ] Закон истины — "свидетельство несёт сорт; сорт повышает только
    человек": замкнутый контур производит [внутр.] всегда.
```

## Узел

```text
[ ] Fixture-закон (данные + один прогонщик по требованию, срок
    годности, никогда runtime, никогда постоянный модуль) — как
    практическое воплощение корней Формы/Роста/Истины разом.
```

## Закон знаний (И1–И5, Response 03)

```text
[ ] И1 — закрытый словарь-посредник как единственная среда встречи
    запросной и знаниевой сторон.
[ ] И2 — всё, что пересекает границу Writer'а, декларативно.
[ ] И3 — каждое поле имеет названного потребителя.
[ ] И4 — валюта развития — записанные провалы, не воображение.
[ ] И5 (если отдельно нумеровался в R03 — сверить по документу) —
    писательский пакет: 4 отсека, бюджет дефицита.
```

## Закон восприятия (В1–В4, Response 04)

```text
[ ] В1 — одна рамка, один производитель, ни одного второго судьи.
[ ] В2 — рамка — уценённая гипотеза, не диагноз; переписывается
    каждый ход; latest turn authority неприкосновенна.
[ ] В3 — словари восприятия — тот же закрытый/малый/версионируемый
    режим, что словарь механизмов.
[ ] В4 — рамка декларативна и удаляема; ответ без рамки теряет
    точность, но остаётся валидным.
```

## Закон лаборатории (Л1–Л5, Response 05)

```text
[ ] Л1 — свидетельствует, не решает; выход — диффы, не вердикты.
[ ] Л2 — видимый ответ — артефакт первого класса, наравне с trace
    и пакетом (ИСПРАВЛЕНО Response 07/Н1: согласованность — по
    границе доставки, не использования).
[ ] Л3 — свидетельство несёт сорт и происхождение; сорт повышает
    только человек.
[ ] Л4 — мера успеха темы — цена человеческого взгляда, не число
    прогонов.
[ ] Л5 — контракт о поведении на видимой границе, никогда о
    внутренних стадиях.
```

## Закон голоса (Г1–Г3, Response 06)

```text
[ ] Г1 — право на механизм выдаёт рамка; без права — грамматика
    присутствия (ИСПРАВЛЕНО Response 07/Н3: лицензия асимметрична,
    отсутствие рамки не ломает валидность ответа).
[ ] Г2 — тест переносимости как структурное определение штампа
    (ИСПРАВЛЕНО Response 07/Н2: происхожденческая оговорка — след
    этого человека, не эхо, которое ему вернул бот).
[ ] Г3 — у живости нет эталонного стиля; любой образец — шаблон
    при первом копировании.
```
